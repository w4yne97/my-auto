# learn-marketing — 知识变现产出

## 触发

用户输入 `/learn-marketing <产出类型> <主题/概念>`

产出类型：
- `blog` — 技术博客大纲
- `talk` — 演讲/技术分享提纲
- `thread` — Twitter/X 技术 thread
- `overview` — 领域综述框架

## 工作流

### Step 1: 评估知识基础

1. 读取相关概念的 knowledge-map 状态
2. 确认有足够的 L2+ 概念支撑产出
3. 读取 knowledge-vault 中的相关笔记
4. 如果知识不足，提醒用户先学习

### Step 2: 搜索市场定位

1. 使用 WebSearch 搜索类似主题的已有内容
2. 分析已有内容的角度和深度
3. 找出差异化点——你的独特视角是什么

### Step 3: 生成产出框架

根据产出类型生成不同格式的框架：

**blog 大纲**:
```
# [标题]

## Hook（为什么读者应该关心）
## 背景与问题（简洁铺垫）
## 核心观点 1（含数据/图表建议）
## 核心观点 2
## 核心观点 3
## 实践启示（对从业者的建议）
## 结论与展望
## 参考文献
```

**talk 提纲**:
```
# [演讲标题] (X 分钟)

## Opening (2min)
## 问题定义 (3min)
## 技术深入 (15min)
  - 关键 slide 建议
  - 演示/代码示例建议
## 实践案例 (5min)
## Q&A 预判
```

**thread**:
```
1/ [Hook tweet]
2/ [背景]
3-7/ [核心观点，每条一个]
8/ [总结 + CTA]
```

### Step 4: 丰富内容

- 为每个章节标注支撑的知识笔记来源
- 建议可以引用的数据和图表
- 标注需要进一步研究的空白点
