L2: Policy Iteration (策略迭代)

§1 Why this lesson

L1 的 VI 在每轮里"同时" 更新 \(v\) 和 \(\pi\),中途的 \(v_k\) 不是 state value(只是 BOE 算子的迭代点)。这有个工程缺陷:如果你想在 \(k = 100\) 停下来"看看 \(\pi_{100}\) 到底有多好",你拿到的 \(v_{100}\) 不能严格 当作 \(\pi_{100}\) 的 value —— 它还在路上。L1 末段的 Q2 已经埋下伏笔:能不能在每轮都把 \(\pi_k\) 评估到位,让中途的 \(v\) 真正具有"\(\pi_k\) 的 state value" 含义?

本节(教材 §4.2,pp. 62-70)给出另一条路径:Policy Iteration (PI)。每轮外循环干两件事:(1) PE 解 Bellman 方程 \(v_{\pi_k} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}\)(Ch.2 L4 的闭式或迭代);(2) PI step 贪心 \(\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_{\pi_k})\)。三件核心结论:(a) 单调改进 (Lemma 4.1) \(v_{\pi_{k+1}}(s) \ge v_{\pi_k}(s) \forall s\);(b) 收敛 (Theorem 4.1) \(\{v_{\pi_k}\}\) 单调上升且被 \(v^*\) 上界 \(\Rightarrow\) 收敛到 \(v^*\);(c) 5×5 grid 上只需 \(\sim 10\) 外循环 —— 远比 VI 的 \(\sim 65\) 步少,代价是每外步要解一个完整 Bellman 方程。PI 是"少而贵"、VI 是"多而便宜"。这套"两层循环" 是现代 RL 算法的祖型 —— Ch.5 MC Basic、Ch.7 SARSA / Q-learning、Ch.10 actor-critic 都在 GPI 框架内活动。

§2 核心定义(直觉先行)

  • Policy Iteration (PI;Algorithm 4.2,p. 66):输入任意初始策略 \(\pi_0\)。每轮外循环 \(k\) 做(PE) 解 \(v_{\pi_k} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}\) + (PIstep) \(\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_{\pi_k})\),直到 \(\pi_{k+1} = \pi_k\) 或 \(\|v_{\pi_{k+1}} - v_{\pi_k}\| < \epsilon\)。
  • Policy Evaluation (PE):有两个等价做法。闭式 \(v_{\pi_k} = (I - \gamma P_{\pi_k})^{-1} r_{\pi_k}\)(理论用,\(O(n^3)\));迭代 \(v_{\pi_k}^{(j+1)} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}^{(j)}\)(实践用,从 \(v^{(0)}\) 任取出发,\(j \to \infty\) 收敛)。这是 Ch.2 L4 的迭代法被嵌入到 PI 的内循环。
  • Policy Improvement (PI step):\(\pi_{k+1}(s) = \arg\max_a q_{\pi_k}(s, a)\),其中 \(q_{\pi_k}(s, a) = \sum_r p(r\mid s,a) r + \gamma \sum_{s'} p(s'\mid s,a) v_{\pi_k}(s')\)。注意这里的 \(q_{\pi_k}\) 是真正的 action value(因为 \(v_{\pi_k}\) 是真正的 state value),不像 VI 中的 \(q_k\) 只是中间量。
  • Lemma 4.1 (Policy improvement,p. 63):若 \(\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_{\pi_k})\),则 \(v_{\pi_{k+1}} \ge v_{\pi_k}\)(逐元素)。翻译:贪心改进出来的新策略,严格意义上每个状态 都不比旧策略差。这是 PI 的"monotonic improvement" 保证。
  • Theorem 4.1 (Convergence of PI,p. 64):PI 生成的 \(\{v_{\pi_k}\}\) 收敛到 \(v^*\),\(\{\pi_k\}\) 收敛到一个最优策略。证明思路:Lemma 4.1 给单调上升,\(v_{\pi_k} \le v^*\) 给上界,单调有界数列必收敛(monotone convergence theorem);极限恰是 \(v^*\) 因为 PE+PI step 一起就是 BOE 算子的"加速版"(Box 4.2)。
  • PI 收敛不慢于 VI(Box 4.2):用同样的 \(v_0 = v_{\pi_0}\) 起点,PI 的 \(v_{\pi_k}\) 和 VI 的 \(v_k\) 满足 \(v_k \le v_{\pi_k} \le v^*\) —— 因为 PI 的内循环把 PE 跑到 \(\infty\) 步,而 VI 只跑 1 步。实操观察:对 5×5 grid,VI 需 \(\sim 65\) 外步,PI 只需 \(\sim 10\) 外步;但 PI 每外步代价远大,总壁钟时间不必更少。
  • PI 的两个空间观察(教材 §4.2.3 末段):(a) 信息从 target 向外辐射:近 target 格子先在早期外循环里找到最优朝向(因为 PE 立即把"近格" 的 \(v_\pi\) 算到位),远 target 格子要等近格有了正向 v 之后才能算出有意义的 q-table;(b) \(v^*\) 空间分布:近 target 值大、远 target 值小(\(\gamma\) 折扣使远期 reward 缩水)。Figure 4.4 给完整 11 帧演化。

§3 关键公式与拆解

本节核心两个证明:Lemma 4.1 单调改进(\(v_{\pi_{k+1}} \ge v_{\pi_k}\))和 Theorem 4.1 PI 收敛。这两个是 PI 算法能用 的全部理论保障。

公式 1:PI 的两步拆解(Algorithm 4.2 / Box 4.1 / Box 4.2)

Step 1. Lemma 4.1 单调改进证明(教材 Box 4.1)。两个 Bellman 方程:\(v_{\pi_k} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}\),\(v_{\pi_{k+1}} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_{\pi_{k+1}}\)。由 \(\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_{\pi_k})\),取 \(\pi = \pi_k\) 作竞争者,\(r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_{\pi_k} \ge r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k} = v_{\pi_k}\)。所以 \(v_{\pi_k} - v_{\pi_{k+1}} \le (r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_{\pi_k}) - v_{\pi_{k+1}} = \gamma P_{\pi_{k+1}} (v_{\pi_k} - v_{\pi_{k+1}})\)。
Step 2. Lemma 4.1 收尾。由 Step 1 的不等式迭代:\(v_{\pi_k} - v_{\pi_{k+1}} \le \gamma^n P_{\pi_{k+1}}^n (v_{\pi_k} - v_{\pi_{k+1}})\)。\(P_{\pi_{k+1}}^n\) 是非负行随机矩阵(每行和 \(\le 1\)),\(\gamma^n \to 0\),所以右端 \(\to 0\),从而 \(v_{\pi_k} - v_{\pi_{k+1}} \le 0\),即 \(v_{\pi_{k+1}} \ge v_{\pi_k}\)。关键洞察:贪心改进借助"同一个 \(v_{\pi_k}\) 当评判标尺" 在 PE 与 PI 之间架桥,使 PI 步在该 \(v_{\pi_k}\) 度量下"无可挑剔",再用 Bellman 方程把"PI 评判标尺下的不等式" 翻译成"\(v_\pi\) 自己的不等式"。
Step 3. Theorem 4.1 收敛性。Lemma 4.1 给 \(v_{\pi_0} \le v_{\pi_1} \le \dots \le v^*\) 单调上升 + 被 \(v^*\) 上界,monotone convergence theorem 给收敛到某 \(v_\infty\)。再证 \(v_\infty = v^*\):取 \(k \to \infty\),Bellman + 改进步给 \(v_\infty\) 满足 BOE,Ch.3 BOE 解唯一 \(\Rightarrow v_\infty = v^*\)。
Step 4. PI vs VI 收敛速率(Box 4.2 核心洞察)。同样从 \(v_0 = v_{\pi_0}\) 起步,可以证 \(v_k \le v_{\pi_k} \le v^*\) for all \(k\)(归纳:\(k = 0\) 取等;假设第 \(k\) 步成立,推第 \(k+1\) 步用 Lemma 4.1 + \(\arg\max\) 性质)。这告诉你PI 比 VI 在数值上更"靠近" \(v^*\) —— 因为 PI 的 PE 内循环跑了 \(\infty\) 步,而 VI 只跑 1 步。直觉:PI 充分利用了"已知 \(\pi_k\) 后 \(v_\pi\) 是个线性系统能直接解" 这个事实;VI 假装不知道、只走一步。代价:PE 解线性系统贵。

§4 Grid-world 实例化

用 5×5 grid 看 PI 怎样从一个"糟糕" 的 \(\pi_0\)(全部'stay',\(v_{\pi_0}\) 全 0)演化到 \(\pi^*\)。target = \((3, 2)\),forbidden = \(\{(1,1),(2,1),(2,2),(1,3),(3,3)\}\),\(r_\text{target} = +1\)、\(r_\text{forbidden} = -10\)、\(r_\text{boundary} = -1\)、\(r_\text{other} = 0\)、\(\gamma = 0.9\)(参考教材 §4.2.3 "more complicated example",Figure 4.4)。

PI 的两层循环结构 —— 这是本节最关键的可视化。我们用两个 iteration-stepper:外循环 显示 \(\pi_k \to \pi_{k+1}\) 的 grid 演化(箭头 + 热图同步动);内循环 单独演示一次 PE 的 \(v_{\pi_k}^{(j)} \to v_{\pi_k}\) 收敛过程(固定 \(\pi_k = \pi_3\),展示 \(j = 0, 1, \dots\) 直到收敛)。

外循环:\(\pi_k\) 的演化(Figure 4.4 (a)-(h))

外循环只用 \(\sim 10\) 步就到 \(\pi^*\) —— 远比 VI 的 \(\sim 65\) 步少。代价是每外步要解一个 25 维线性系统(或跑很多内循环步)。

内循环:一次 PE 的迭代过程(以 \(\pi_3\) 为例)

PE 是 PI 的嵌套迭代。固定 \(\pi_3\),从 \(v^{(0)} = 0\) 起跑迭代法 \(v^{(j+1)} = r_{\pi_3} + \gamma P_{\pi_3} v^{(j)}\)(Ch.2 L4 公式 (2.11)),看 \(v^{(j)}\) 怎样在固定 \(\pi_3\) 下收敛到 \(v_{\pi_3}\)。本动画在 5×5 grid 上展示 \(j = 0, 1, 2, 5, 20\) 的 \(v^{(j)}\) 热图。

PI vs VI 收敛对比。下图把同一 5×5 grid 上的 VI 和 PI 画在一起。x 轴是外循环步数,y 轴是 \(\|v_k - v^*\|_\infty\) 的 log-y。注意 PI 的"外循环" 与 VI 的"迭代轮次" 含义不同 —— 一个 PI 外步 = 一次完整 PE(可能跑 50 个内步) + 一次 PI step,代价远大于一个 VI 步。但在外步数 维度上 PI 显著更陡(更少步数到收敛)。Box 4.2 给的理论保证:同一起点下,\(v_k^\text{VI} \le v_{\pi_k}^\text{PI} \le v^*\)。

§5 常见误解 / 易混概念

很多人以为:"Lemma 4.1 保证单调,PI 一定收敛得比 VI 快。"
实际上:"外循环步数" 上 PI 快,总壁钟 不一定。每 PI 外步要解 \(n\) 维 Bellman:闭式 \(O(n^3)\),迭代跑 \(\sim 50-100\) 内步、每步 \(O(n^2)\) = \(O(n^2 \cdot 100)\)。VI 一步 \(O(n^2)\)。PI 一外步 ≈ 50-100 个 VI 步,PI 用 10 外步、VI 用 65 步 —— PI 总成本反而 5-10 倍 VI。
因为:PI 显式解线性系统,获得"v 与 π 自洽";VI 隐式做这件事,牺牲"v 是 state value" 但每步便宜。L3 truncated PI 把"PE 跑多少内步" 当作 dial 在两者之间找平衡。
很多人以为:"PI 中途的 \(v_{\pi_k}\) 也跟 VI 中途的 \(v_k\) 一样'不是 state value'。"
实际上:正好相反 —— PI 中途的 \(v_{\pi_k}\) 是 真正的 state value(假设 PE 解到收敛)。具体:Bellman 方程 \(v_{\pi_k} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}\) 在 PE 步后严格成立,所以可以合法地把 \(v_{\pi_k}\) 解读为"\(s\) 出发按 \(\pi_k\) 走的折扣 return 期望"。VI 的 \(v_k\) 不享受这特权(L1 §5 已详谈)。
因为:这是 PI 与 VI 设计哲学的根本差异:PI 在每外步保证"值与策略自洽"(Bellman 方程精确成立);VI 不要求中途自洽,只要极限自洽(\(v^*\) 满足 BOE)。"自洽" 这件事是 RL 工程上很多 trick(如 fitted Q-iteration 用 PE 当子例程)的理论根。L3 truncated PI 把"PE 跑多少步" 设为 dial 后,中途的 \(v\) 既不是 \(v_{\pi_k}\) 也不是 BOE 中间量,变成不完全自洽 —— 但仍能收敛。
很多人以为:"Lemma 4.1 说 \(v_{\pi_{k+1}} \ge v_{\pi_k}\),所以 PI 每个状态 严格单调上升、最终到 \(v^*\)。"
实际上:"≥" 不是 ">"。若 \(\pi_k\) 已最优,Lemma 4.1 取等、PI step 给 \(\pi_{k+1} = \pi_k\)、迭代停在原地 —— 这就是停机条件 \(\|v_{\pi_{k+1}} - v_{\pi_k}\| < \epsilon\) 的来源。精确说法:"新策略每个状态都不变差,且 strict improvement 至少在一个状态发生 —— 除非已最优"。
因为:有限状态 + 有限动作 \(\Rightarrow\) deterministic policy 总数 \(|\mathcal A|^{|\mathcal S|}\) 有限,strict improvement 不能无限发生,PI 有限步内必到 \(\pi^*\) —— 比 Box 4.2 的"单调有界" 更直接的另一种证法。

§6 自检 + 视频 + 下一节

Q1. §4 教材简单例子(2 状态 \(\{s_1, s_2\}\),3 动作 \(\{a_\ell, a_0, a_r\}\),target = \(s_2\),\(\gamma = 0.9\))。\(\pi_0(s_1) = a_\ell\)、\(\pi_0(s_2) = a_0\)。手算 PE 给 \(v_{\pi_0}(s_1) = -10\)、\(v_{\pi_0}(s_2) = -9\)。验证:这两个值满足 \(v_{\pi_0} = r_{\pi_0} + \gamma P_{\pi_0} v_{\pi_0}\) 吗?

A. \(\pi_0\) 在 \(s_1\) 选 \(a_\ell\)(向左,撞 boundary,\(r = -1\),保持在 \(s_1\)):\(v_{\pi_0}(s_1) = -1 + 0.9 v_{\pi_0}(s_1)\),解出 \(v_{\pi_0}(s_1) = -10\)。\(\pi_0\) 在 \(s_2\) 选 \(a_0\)(stay,\(r = 0\),保持 \(s_2\)):应该是 \(v_{\pi_0}(s_2) = 0 + 0.9 v_{\pi_0}(s_2) \Rightarrow v_{\pi_0}(s_2) = 0\)。但教材给 \(-9\)?注意:教材这里 \(s_2\) 上的 \(a_0\) 转移到 \(s_1\)(看图 4.3a 里 \(s_2\) 那个箭头其实指向 \(s_1\),不是 stay),所以 \(v_{\pi_0}(s_2) = 0 + 0.9 \cdot v_{\pi_0}(s_1) = 0.9 \cdot (-10) = -9\) ✓。验证通过。这是 PE 的本质:解线性系统给"遵从该策略下的真 return 期望"。

Q2. Lemma 4.1 给"\(v_{\pi_{k+1}} \ge v_{\pi_k}\) 逐元素"。如果 \(v_{\pi_{k+1}}(s_0) = v_{\pi_k}(s_0)\) 但 \(v_{\pi_{k+1}}(s_1) > v_{\pi_k}(s_1)\),还能算"严格改进" 吗?

A. 算。"严格改进" 的定义就是"至少一个状态 strict,其他状态不变差"。PI 不要求所有状态都严格上升 —— 比如 target 自身的 \(v^*(s_\text{target}) = 1/(1-\gamma) = 10\) 在 \(\gamma = 0.9\) 下,如果 \(\pi_k\) 已经在 target 上选'stay' 拿 reward,那 \(v_{\pi_k}(s_\text{target}) = 10\) 已经到顶,后续外循环 target 自身 \(v\) 不动了,但其他状态可能还在上升。这种"局部已饱和、整体未收敛" 的情形完全合规。

Q3. 为什么 PI 在 5×5 grid 上"近 target 格子先收敛"(教材 §4.2.3 末段第一个观察)?

A. 因为 \(v_{\pi_0} = 0\),第 1 外步 PI 的 q-table 只能在"一步可达 target" 的格子上拿到非零 q(直接吃 target reward),其他格子全 0、\(\arg\max\) 任选。所以第 1 外步只确定target 4-邻居的方向。第 2 外步 PE 把这些邻居的 \(v_{\pi_1}\) 算到位(他们能拿 +1 经折扣),才轮到"2 步可达 target" 的格子在 PI step 拿到非零 q、确定方向。信息从 target 向外辐射,辐射半径每外步增加 1 —— 这跟 L1 §4 的 VI 是同一物理。区别仅在 PI 每外步把当前已激活区的 \(v\) 算到精确,VI 每步把 \(v\) 推一小格。

下一节预告

到这里有了两个算法:VI 在值空间每步走一小格(每轮便宜),PI 在策略空间每外步把 \(v_{\pi_k}\) 解到位(每轮昂贵但少)。问题:PI 的 PE 真要跑到 \(j = \infty\) 吗?\(j = 5\)、\(j = 50\) 行不行?这种"PE 截断" 会破坏 Theorem 4.1 的收敛保证吗?下一节(L3,Book §4.3)给出统一答案 —— Truncated PI:PE 跑 \(j_\text{truncate}\) 步而不是 \(\infty\),得到一个连续可调的算法谱。\(j_\text{truncate} = 1\) ≡ VI,\(j_\text{truncate} = \infty\) ≡ PI,中间是实操甜点。"VI 低端、PI 高端" 的二分法被替换成"同一族算法的连续光谱" + 一个工程可调 dial。
