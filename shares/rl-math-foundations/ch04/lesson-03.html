L3: Truncated Policy Iteration (截断策略迭代)

§1 Why this lesson

L1 给我们 VI("多便宜外步"),L2 给我们 PI("少昂贵外步"),两者看起来是两套独立算法。但 L2 §6 已经埋下伏笔:PI 的 PE 内循环真的需要跑到 \(j = \infty\) 吗?如果只跑 \(j = 5\)、\(j = 50\),会发生什么?本节(教材 §4.3-4.5,pp. 70-76)给出统一答案 —— VI 与 PI 不是两套算法,而是同一族算法的两个极端,中间是一片连续光谱叫 Truncated Policy Iteration。

关键洞察(教材 Table 4.6 + §4.3.1 中段):若让两个算法同初始(\(v_0 = v_{\pi_0}\)),写出 PE 迭代过程 \(v_{\pi_1}^{(0)} = v_0,\ v_{\pi_1}^{(j+1)} = r_{\pi_1} + \gamma P_{\pi_1} v_{\pi_1}^{(j)},\ j = 0, 1, 2, \dots\)。VI 第 \(k\) 轮的 \(v_1\) 恰好是 \(v_{\pi_1}^{(1)}\)(PE 跑 1 步),PI 的 \(v_{\pi_1}\) 恰好是 \(v_{\pi_1}^{(\infty)}\)(PE 跑 \(\infty\) 步)。中间任何有限 \(j_\text{truncate}\) 步停下都得到一个新算法 —— Truncated PI。核心信息:(a) 算法谱: \(j_\text{truncate} = 1\) ≡ VI,\(j_\text{truncate} = \infty\) ≡ PI,中间 5-50 是实操甜点;(b) Proposition 4.1 单调改进:用 \(v_{\pi_k}^{(0)} = v_{\pi_{k-1}}\) warm start 时,内循环 \(v_{\pi_k}^{(j)}\) 在 \(j\) 上单调上升 —— 即"多跑一步内循环不会变差";(c) Figure 4.5 直觉:三条曲线 \(v_k\)(VI,慢)、\(\bar v_k\)(Truncated,中)、\(v_{\pi_k}\)(PI,快)在外步数维度上呈"上下夹挤" 的形态,Truncated 在中间。这把 L1/L2 的"VI 低端、PI 高端" 二分法替换成"同一族算法的连续光谱 + 一个工程可调 dial \(j_\text{truncate}\)"。这是 Generalized Policy Iteration (GPI,§4.4) 的具体实例,贯穿 Ch.5-Ch.10 所有 RL 算法。

§2 核心定义(直觉先行)

  • Truncated PI (Algorithm 4.3,p. 72):输入 \(\pi_0\)。每外循环 \(k\):PE 用 \(v_k^{(0)} = v_{k-1}\) 当 warm start,跑 \(j_\text{truncate}\) 次迭代 \(v_k^{(j+1)} = r_{\pi_k} + \gamma P_{\pi_k} v_k^{(j)}\),取 \(v_k = v_k^{(j_\text{truncate})}\);PI step \(\pi_{k+1} = \arg\max_a q_k(s, a)\)。注:\(v_k\) 这里既不是 VI 中的 \(v_k\)(没跑到收敛)也不是 PI 中的 \(v_{\pi_k}\)(也没跑到收敛),只是近似 state value。
  • 三个算法的 dial 视角:\(j_\text{truncate} = 1\) → VI(每外步 PE 只走一格);\(j_\text{truncate} = \infty\) → PI(每外步 PE 解到精确);\(j_\text{truncate} \in [2, \infty)\) → Truncated PI(实操中典型 \(j = 5-50\))。
  • Proposition 4.1 (Value improvement,Box 4.3,p. 73):在 PE 内循环里,若取初始 \(v_{\pi_k}^{(0)} = v_{\pi_{k-1}}\),则 \(v_{\pi_k}^{(j+1)} \ge v_{\pi_k}^{(j)}\) 对所有 \(j\) 成立 —— 内循环每多跑一步,值不变差。这是 Truncated PI 的"多跑无害" 定理。
  • VI 与 PI 的"同初始" 对比表 (Table 4.6,p. 71):取 \(v_0 = v_{\pi_0}\)。第 1-3 步两算法生成相同结果(\(v_0, \pi_1, v_1 / v_{\pi_1}^{(1)}\) 相等)。第 4 步开始分叉:VI 算 \(v_1 = r_{\pi_1} + \gamma P_{\pi_1} v_0\)(一步);PI 算 \(v_{\pi_1} = (I - \gamma P_{\pi_1})^{-1} r_{\pi_1}\)(无穷步)。Truncated 在中间停。
  • Generalized Policy Iteration (GPI,§4.4):不是具体算法,而是"值更新与策略更新交替" 的设计哲学。Ch.4 的 VI/PI/Truncated PI 都是 GPI 实例,Ch.5 MC、Ch.7 TD、Ch.10 actor-critic 也都是。这是把 RL 算法世界统一起来 的一个上层抽象。
  • Figure 4.5 三曲线直觉:同一外步数 \(k\) 下,\(v_k^\text{VI} \le \bar v_k^\text{Truncated} \le v_{\pi_k}^\text{PI} \le v^*\)。VI 最慢爬升,PI 最快,Truncated 居中。为什么 PI 最快:每外步把"已知 \(\pi_k\) 后值是线性系统能精确解" 这件事吃干。VI 最慢:这条信息只用一格的更新。
  • "为什么 \(j_\text{truncate}\) 大些能加速"(教材 §4.3.2 末段):多跑几步 PE 把 \(v_k\) 推得更接近 \(v_{\pi_k}\) ⇒ 下一外步 PI step 的 \(\arg\max\) 用的"未来承诺" 更准确 ⇒ 选出的 \(\pi_{k+1}\) 更接近 \(\pi^*\)。但收益边际递减:\(j = 1 \to 5\) 提升大,\(j = 50 \to \infty\) 提升微小。所以工程甜点在 \(j = 5-50\)。

Step 1. VI 与 PI 的"同初始" 表(教材 Table 4.6,p. 71)。取 \(v_0 = v_{\pi_0}\) 作公共起点。Step 1(政策):\(\pi_1^\text{VI} = \arg\max_\pi (r_\pi + \gamma P_\pi v_0) = \arg\max_\pi (r_\pi + \gamma P_\pi v_{\pi_0}) = \pi_1^\text{PI}\) ✓ 两策略相同。Step 2(值):VI 算 \(v_1 = r_{\pi_1} + \gamma P_{\pi_1} v_0\)(一步);PI 算 \(v_{\pi_1} = r_{\pi_1} + \gamma P_{\pi_1} v_{\pi_1}\)(无穷步)。前者 = \(v_{\pi_1}^{(1)}\),后者 = \(v_{\pi_1}^{(\infty)}\),Truncated 取 \(v_{\pi_1}^{(j_\text{truncate})}\) 在中间。
Step 2. VI ≡ Truncated PI with \(j_\text{truncate} = 1\) 的精确证明。VI 第 \(k\) 步 \(v_k = r_{\pi_k} + \gamma P_{\pi_k} v_{k-1}\),正是 PE 内循环 \(v_{\pi_k}^{(j+1)} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}^{(j)}\) 在 \(v_{\pi_k}^{(0)} = v_{k-1}\)、\(j = 0\) 时的结果 \(v_{\pi_k}^{(1)}\)。所以 VI 是 Truncated PI 在 \(j_\text{truncate} = 1\) 时的退化。同理 PI 是 \(j_\text{truncate} = \infty\) 的退化。关键:三个算法共享同一个外循环骨架,只在"PE 跑多少步" 上不同 —— 这就是"同族算法" 的精确说法。
Step 3. Proposition 4.1 证明(Box 4.3,p. 73)。两个等式:\(v_{\pi_k}^{(j+1)} - v_{\pi_k}^{(j)} = \gamma P_{\pi_k} (v_{\pi_k}^{(j)} - v_{\pi_k}^{(j-1)})\)(逐次代入),迭代得 \(v_{\pi_k}^{(j+1)} - v_{\pi_k}^{(j)} = \gamma^j P_{\pi_k}^j (v_{\pi_k}^{(1)} - v_{\pi_k}^{(0)})\)。再用 warm start \(v_{\pi_k}^{(0)} = v_{\pi_{k-1}}\):\(v_{\pi_k}^{(1)} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_{k-1}} \ge r_{\pi_{k-1}} + \gamma P_{\pi_{k-1}} v_{\pi_{k-1}} = v_{\pi_{k-1}} = v_{\pi_k}^{(0)}\)(由 \(\pi_k = \arg\max_\pi(r_\pi + \gamma P_\pi v_{\pi_{k-1}})\))。所以 \(v_{\pi_k}^{(1)} \ge v_{\pi_k}^{(0)}\),\(\gamma^j P^j\) 非负 \(\Rightarrow\) 全部 \(v_{\pi_k}^{(j+1)} \ge v_{\pi_k}^{(j)}\)。
Step 4. "\(v_k\) 不是 state value" 的延续。Truncated PI 中的 \(v_k\) 既不是 \(v_{\pi_k}\)(PE 没跑到 \(\infty\))也不是 \(v_{\pi_{k-1}}\)(PE 已经跑了 \(j_\text{truncate}\) 步),是"j 步逼近"。但这不影响 算法收敛 —— 教材 §4.3.2 末段直观:Truncated PI 的 \(v_k\) 仍夹在 VI 与 PI 之间,所以 \(v_k \to v^*\) 仍成立。严格证明需要 [2, §6.5] 给的论证,本节不展开。

§5 常见误解 / 易混概念

很多人以为:"Truncated PI 是个'妥协' 算法,理论上不如 PI 干净 —— 既然 PI 收敛性证明严格(Theorem 4.1),Truncated PI 的 \(v_k\) 不是 state value、应该收敛性更弱。"
实际上:Truncated PI 的收敛性同样有保障(教材 §4.3.2 末段 + [2, §6.5] 严证)。直觉:Truncated PI 的 \(v_k\) 夹在 VI 的 \(v_k\) 和 PI 的 \(v_{\pi_k}\) 之间,两个端点都收敛到 \(v^*\),夹挤定理给中间也收敛。Proposition 4.1 进一步说"warm start 下 PE 内循环单调上升" —— 这保证 Truncated 不会因为"PE 截断" 而陷入震荡。
因为:"\(v_k\) 是不是 state value" 是"中途自洽" 的性质,跟"极限收敛" 是两件事。VI 中途也不自洽(L1 §5),但极限收敛。Truncated 同理。RL 工程上很多实现(尤其是函数近似 Ch.8 起)都是事实上的 truncated —— 如 Q-learning 每步只更新一个 \((s, a)\) 而不是解整个线性系统,本质上 \(j_\text{truncate} = 1\) 的 sample 版。
很多人以为:"既然 \(j_\text{truncate}\) 越大越快收敛,实操上应该选尽量大的 \(j_\text{truncate}\)(比如 1000),逼近 PI。"
实际上:边际递减 + 总壁钟权衡。\(j = 1 \to 5\) 收益巨大(\(k = 10\) 时 ‖·‖ 从 3.49 降到 0.018),\(j = 50 \to \infty\) 收益微小(从 \(4.5 \times 10^{-5}\) 降到 \(1 \times 10^{-5}\))。但每外步代价随 \(j\) 线性 增长。所以最优 \(j_\text{truncate}\) 是"减速曲线斜率 = 单步代价" 的拐点,经验值在 \(5-50\) 之间。教材 Q&A 直接说"run a few iterations but not too many"。
因为:这是计算优化里"截断 SGD" 同类的权衡 —— 每内步带来的"信息" 在指数衰减(\(\gamma^j\)),而单步成本 \(O(n^2)\) 不变,所以总有一个 sweet spot。Ch.6 RM/SGD 会把这个想法形式化 到一般 stochastic approximation 框架。
很多人以为:"GPI(generalized policy iteration)是 Ch.4 的小附录,不重要。"
实际上:GPI 是 RL 算法世界最重要的统一抽象之一。教材 §4.4 末段直接说"most reinforcement learning algorithms in this book fall into the scope of generalized policy iteration"。具体:Ch.5 MC Basic 是 PI 在"用样本均值代替 PE 闭式解" 下的 GPI 实例;Ch.7 SARSA / Q-learning 是 \(j_\text{truncate} = 1\) 的 GPI 实例(每步 sample 一次只做一次 PE 更新);Ch.10 actor-critic 是把 PI 的两步 PE/PI 拆给两个网络的 GPI 实例。理解 GPI = 拿到 Ch.5-Ch.10 整个阅读路线图的钥匙。
因为:RL 算法的动力学结构可以浓缩成"值估计 ↔ 策略改进 互相驱动" 这一个 box-arrow 图。具体怎样估值、怎样改进、估到多准、改到多严格,是设计选择 但不改变骨架。GPI 这个抽象给我们看待 RL 算法的正确视角 —— 不要只看公式细节,要先看"两个箭头" 怎么闭环。

§6 自检 + 视频 + 下一节

Q1. 教材 Table 4.6 (p. 71) 给"VI 与 PI 同初始 \(v_0 = v_{\pi_0}\),前 3 步生成相同结果,第 4 步分叉"。第 4 步具体哪里不同?

A. 第 4 步两算法都做"用 \(\pi_1\) 更新 \(v\)" 这件事,但跑多少步 不同。VI:\(v_1 = r_{\pi_1} + \gamma P_{\pi_1} v_0\),用 \(v_0\) 一步到 \(v_1\)(= \(v_{\pi_1}^{(1)}\))。PI:解 \(v_{\pi_1} = r_{\pi_1} + \gamma P_{\pi_1} v_{\pi_1}\),无穷步迭代到 \(v_{\pi_1}^{(\infty)}\)。所以 VI 的 \(v_1 \ne\) PI 的 \(v_{\pi_1}\),前者 = 后者 PE 的"第一步" 而已。Truncated PI 在中间任意 \(j_\text{truncate}\) 处停下。这就是"三个算法是同族不同 dial" 的精确数学体现。

Q2. Proposition 4.1 要求 \(v_{\pi_k}^{(0)} = v_{\pi_{k-1}}\)(warm start)。如果实际算法用 \(v_{\pi_k}^{(0)} = v_{k-1}\)(Truncated PI 的 \(v_{k-1}\) 不是 \(v_{\pi_{k-1}}\),只是近似),Proposition 4.1 还成立吗?

A. 严格意义上不成立(教材 Box 4.3 末段直接说"\(v_{\pi_{k-1}}\) is unavailable in practice, only \(v_{k-1}\) is available"),但近似成立。直觉:\(v_{k-1} \approx v_{\pi_{k-1}}\)(误差 \(O(\gamma^{j_\text{truncate}})\)),所以 \(v_{\pi_k}^{(1)} \ge v_{\pi_k}^{(0)} - O(\gamma^{j_\text{truncate}})\)。当 \(j_\text{truncate}\) 充分大,Proposition 4.1 几乎严格成立。这是 Truncated PI 实操中"大致单调,偶尔小回退" 的根 —— 不是 bug,是 \(j_\text{truncate}\) 有限带来的固有近似误差。详细分析在 [2, §6.5]。

Q3. 把 VI / PI / Truncated PI 三个算法的"per-iteration" 计算量对比。哪个最贵?哪个最便宜?为什么 Truncated PI 是工程实操最常用的?

A. 设 \(n = |\mathcal S|\)。VI:每外步 \(O(n^2)\)(一次矩阵-向量乘 + 一次 \(\arg\max\))。PI:每外步 \(O(n^3)\)(闭式 PE)或 \(O(n^2 \cdot j_\text{evaluation})\)(迭代 PE 跑到收敛,典型 \(j \approx 100\))。Truncated PI (j=10):每外步 \(O(n^2 \cdot 10)\)。排序:VI < Truncated < PI(per-iteration)。外步数 排序反过来:VI > Truncated > PI。乘起来后,Truncated 通常总壁钟最少。这就是教材 §4.5 Q&A 推荐"a few iterations but not too many" 的根。

到这一节为止,Ch.4 完成:VI / PI / Truncated PI 三个算法被统一进 GPI 框架,每个都假设已知模型(\(p(r\mid s, a), p(s'\mid s, a)\) 显式给定)。这是 dynamic programming 的最后一站。但真实 RL 问题中,模型几乎永远是未知的 —— 没人会先告诉你"Atari 游戏在按 A 键后下一帧的概率分布"。下一章(Ch.5 Monte Carlo Methods)开始拆"模型已知" 这个假设:没有 \(p\),怎么估期望?最朴素答案 —— 采样多次取平均。MC Basic 把 PI 的 PE 步("解 \(v_{\pi_k}\) 用 \(P_\pi\)")换成"用样本均值代替期望":跑 \(N\) 条 trajectories,用 \(\hat q_\pi(s, a) = \frac{1}{N}\sum_i G_t^{(i)}\) 估 q-value。这把"规划"(planning)这件事变成"学习"(learning),GPI 框架不变 —— PE 用样本、PI step 仍是 \(\arg\max_a \hat q\)。"GPI 是 RL 算法骨架" 的论断,从 Ch.5 开始反复验证。
