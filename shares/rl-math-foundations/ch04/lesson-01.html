L1: Value Iteration (值迭代)

§1 Why this lesson

Ch.3 末尾我们立起了一个抽象结构:BOE \(v = \max_\pi (r_\pi + \gamma P_\pi v)\) 的右端是 \(\gamma\)-压缩映射,Banach 不动点定理保证它有唯一不动点 \(v^*\),并保证从任意 \(v_0\) 出发迭代 \(v_{k+1} = T v_k\) 几何收敛到 \(v^*\)。Ch.3 L4 又把"调 \(\gamma\) / reward 仿射"这两类参数变换的影响讲清。但 Ch.3 一直停留在谈论 "迭代会收敛" 的层面,从没正面落地:具体到一个 grid-world,我每一轮干什么?q-table 怎么算?policy 怎么更新?

本节(教材 §4.1,pp. 57-62)就是把 BOE 落地成第一个具体算法 —— Value Iteration (VI)。它直接照抄 Banach 给的迭代规则 \(v_{k+1} = \max_\pi (r_\pi + \gamma P_\pi v_k)\),不引入新数学,只把它"展开成实现细节"。跟 Ch.2 L4 的迭代法对比:Ch.2 L4 的迭代是 \(v_{k+1} = R_\pi + \gamma P_\pi v_k\)(固定 \(\pi\),做策略评估);VI 多了一个 \(\max_\pi\),每轮 同时更新 策略 —— 值更新与策略更新交织。每轮两步:(1) policy update \(\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_k)\);(2) value update \(v_{k+1} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_k\)。代入后 \(v_{k+1}(s) = \max_a q_k(s,a)\)。本节核心信息:VI 不评估任何具体策略;\(v_k\) 只是 BOE 的中间迭代量,既不是某个 \(\pi\) 的 \(v_\pi\),也不是 \(v^*\),要等 \(k \to \infty\) 才有 \(v_k \to v^*\)。这一"\(v_k\) 不是 state value" 的观察,在 L2 PI 对比下尤其重要。

§2 核心定义(直觉先行)

  • 值迭代算法 (value iteration,VI;教材 §4.1 公式 (4.1) 上方):\(v_{k+1} = \max_{\pi \in \Pi} (r_\pi + \gamma P_\pi v_k), k = 0, 1, 2, \dots\),从任意 \(v_0\) 出发。这正是 Ch.3 Banach contraction 给的迭代规则,只是从"抽象映射"变成"每轮显式展开两步"。
  • 第一步:policy update (PU) \(\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_k)\)。翻译:把当前 \(v_k\) 当作"未来的承诺",在每个状态上贪心选一步即时 reward + 折扣未来 \(v_k\) 最大的动作。元素形式 \(\pi_{k+1}(a\mid s) = \mathbb{1}[a = a_k^*(s)]\),其中 \(a_k^*(s) = \arg\max_a q_k(s, a)\)。
  • 第二步:value update (VU,教材 (4.1)) \(v_{k+1} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_k\)。代入 \(\pi_{k+1}\) 后元素形式坍缩成 \(v_{k+1}(s) = \max_a q_k(s, a)\) —— 每个状态取该状态下 q-table 行的最大值。这一行是 VI 的实操灵魂。
  • q-table 中间量 \(q_k(s,a)\):\(q_k(s,a) \doteq \sum_r p(r\mid s,a) r + \gamma \sum_{s'} p(s'\mid s,a) v_k(s')\)。注意它不是真正的 action value \(q_\pi(s,a)\) —— 因为 \(v_k\) 不是任何 \(\pi\) 的 state value,所以 \(q_k\) 也不是任何 \(\pi\) 的 q-value,只是中间计算变量。这点在 §5 会详谈。
  • VI 算法骨架 (Algorithm 4.1):Initialize \(v_0\);while \(\|v_k - v_{k-1}\| > \epsilon\): for 每个 \(s\): for 每个 \(a\): 算 \(q_k(s,a)\);\(a_k^*(s) = \arg\max_a q_k(s,a)\);\(\pi_{k+1}(a\mid s) = \mathbb{1}[a = a_k^*(s)]\);\(v_{k+1}(s) = \max_a q_k(s,a)\)。停机条件:相邻两轮 \(v\) 差小于阈值。
  • "\(v_k\) 不是 state value"(教材 §4.1.1 末段):VI 中的 \(v_k\) 既不满足 \(v = r_{\pi_k} + \gamma P_{\pi_k} v\) 也不满足 \(v = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v\),只是 BOE 算子的迭代点列。它会收敛到 \(v^*\),但路上每一帧都不是 v_π 的快照。这跟 Ch.2 L4 的迭代法本质不同:那里 \(v_k\) 沿途都在逼近同一个 \(v_\pi\),这里 \(v_k\) 是逼近"一个被同时优化的目标 \(v^*\)"。
  • VI vs Ch.2 L4 迭代法的一行差别:Ch.2 L4: \(v_{k+1} = r_\pi + \gamma P_\pi v_k\)(\(\pi\) 固定,evaluate);本节 VI: \(v_{k+1} = \max_\pi (r_\pi + \gamma P_\pi v_k)\)(\(\pi\) 每轮重选,solve BOE)。差一个 \(\max_\pi\),但物理意义跨越:从"评估"到"求解最优"。

§3 关键公式与拆解

本节核心是把 BOE 的"一行抽象" \(v_{k+1} = T v_k = \max_\pi (r_\pi + \gamma P_\pi v_k)\) 展开成两步实现细节。展开后第二步会自动坍缩成 \(v_{k+1}(s) = \max_a q_k(s, a)\),这是 VI 算法的"灵魂一行"。

公式 1:VI 的两步拆解(教材 §4.1 公式 (4.1) 与上下文)

Step 1. 从 BOE 算子到'两步展开'。Ch.3 把 BOE 写成 \(v = T v\),其中 \((Tv)(s) = \max_a [\sum_r p(r\mid s,a)r + \gamma \sum_{s'} p(s'\mid s,a) v(s')]\)。Banach 告诉我们迭代 \(v_{k+1} = T v_k\) 收敛到 \(v^*\)。本节把这一行 \(T\) 展开成两个子步:先从 \(v_k\) 算 q-table \(q_k(s,a)\),再对 \(a\) 取 max。这没引入新数学,只是把矢量化的 max 拆成"先列表再选行内最大值"的实现细节。
Step 2. 为什么 \(\arg\max_\pi\) 等价于 element-wise \(\arg\max_a\)(Ch.3 §3.3 已证)。\(\sum_a \pi(a\mid s) q_k(s,a) \le \max_a q_k(s,a)\) 对任意分布 \(\pi(\cdot\mid s)\) 成立(凸组合 \(\le\) max);取等当且仅当 \(\pi\) 把全部权重压在 \(a_k^*(s) = \arg\max_a q_k(s,a)\) 上。所以 BOE 中"在 \(\pi\) 上 max"必然给出greedy 确定策略(不存在 ties 时唯一,有 ties 时任选)。这把 \(\arg\max_\pi\) 这个看似复杂的对象坍缩成"每个状态独立做 \(\arg\max_a\)"。
Step 3. VU 自动坍缩成 \(\max_a q_k(s,a)\)。代入 \(\pi_{k+1}(a\mid s) = \mathbb{1}[a = a_k^*(s)]\) 到 \(v_{k+1}(s) = \sum_a \pi_{k+1}(a\mid s) q_k(s,a)\):求和里只有一项非零(\(a = a_k^*(s)\)),所以 \(v_{k+1}(s) = q_k(s, a_k^*(s)) = \max_a q_k(s,a)\)。关键:VU 不需要"代入 \(\pi_{k+1}\) 后再算一遍 q-table" —— 它直接读 PU 选出的那行最大值。这就是 Algorithm 4.1 在每轮里只算一次 q-table 的原因。
Step 4. 收敛性免费继承。VI 等价于 \(v_{k+1} = T v_k\),Ch.3 已证 \(T\) 是 \(\gamma\)-压缩映射,所以 \(\|v_k - v^*\|_\infty \le \gamma^k \|v_0 - v^*\|_\infty\) —— 几何收敛,速率 = \(\gamma\)。\(\gamma = 0.9\) 时大约 65 轮能让误差 \(< 10^{-3}\)。Ch.3 L3 已把证明做完,这里只是给"算法收敛性"贴价签。

§4 Grid-world 实例化

用作者 5×5 grid 跑一遍 VI:target = \((3, 2)\),forbidden = \(\{(1,1),(2,1),(2,2),(1,3),(3,3)\}\),\(r_\text{target} = +1\)、\(r_\text{forbidden} = r_\text{boundary} = -1\)、\(r_\text{other} = 0\)、\(\gamma = 0.9\)。从 \(v_0 = 0\)(全零)起,看 \(v_k\) 的热图怎样从"target 一个 +1 像素"逐步扩散成完整 \(v^*\)。这是本章 iteration-stepper 的第一次正面亮相 —— Ch.2 L4 用过类似动画(那里是固定 \(\pi\) 的策略评估),本节给出"π 也在动"的版本:每按一次"下一步",grid 上的箭头(\(\pi_k\) 的贪心选择)与颜色(\(v_k\))同步更新。

怎么读这个动画?每按"下一步",信息同时更新两件事:(a) 颜色(\(v_k\) 热图)从 target 一个像素扩散成完整曲面;(b) 箭头(\(\pi_k\) 的贪心方向)从"零信息" 逐渐站定为 \(\pi^*\)。关键观察:\(\pi_k\) 在 \(k = 5\) 左右就基本稳定(箭头不再变),但 \(v_k\) 还在继续涨 —— 这是 VI 的一个经验现象:"策略先收敛、值后收敛"。L2 PI 会反过来:每轮内部把 \(v_{\pi_k}\) 算到位,所以每轮外循环数值就变得很大。

VI 的收敛速率验证。Banach 给 \(\|v_k - v^*\|_\infty \le \gamma^k \|v_0 - v^*\|_\infty\),所以 log-y 上误差曲线斜率 = \(\log_{10} \gamma\)。下图把 \(\gamma = 0.9 / 0.99\) 跑出来对比,验证斜率随 \(\gamma\) 接近 1 而变平。\(\gamma = 0.9\) 时大约 65 轮误差降到 \(10^{-3}\),\(\gamma = 0.99\) 时约 685 轮 —— 差一个数量级。L2 PI 的对应曲线会外循环步数明显少(往往 < 10),但每步代价更高,这是 L3 truncated PI 要平衡的核心权衡。

§5 常见误解 / 易混概念

很多人以为:"VI 中间过程的 \(v_k\) 就是当前策略 \(\pi_k\) 的 state value。"
实际上:\(v_k\) 不是任何策略的 state value(教材 §4.1.1 末段)。具体:\(v_k\) 既不满足 \(v = r_{\pi_k} + \gamma P_{\pi_k} v\)(\(\pi_k\) 是上一轮选的、当时用的是 \(v_{k-1}\)),也不满足 \(v = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v\)(\(\pi_{k+1}\) 才刚选,\(v_k\) 是选 \(\pi_{k+1}\) 之前的值)。它只是 BOE 算子 \(T\) 的迭代点列,朝 \(v^*\) 走但路上每帧都不是 v_π。
因为:VI 的核心精神是"值与策略同时逼近",中途不强求 \(v\) 跟 \(\pi\) 自洽。这和 L2 的 PI 形成鲜明对比 —— PI 在每轮外循环里把 \(v_{\pi_k}\) 算到位(用 \(v_{\pi_k} = (I - \gamma P_{\pi_k})^{-1} r_{\pi_k}\) 或迭代),所以 PI 中途的 \(v_{\pi_k}\) 是真正的 state value,可以拿出来作策略评估。
很多人以为:"既然 VI 每轮算一次 q-table,那它跟 Ch.2 L5 引入的 q_π 是同一个东西。"
实际上:差一个微妙的"是不是 state value"。\(q_\pi(s, a)\) 严格定义是 "\(s\) 下强制 \(a\),之后按 \(\pi\) 走的折扣 return 期望",依赖一个具体 \(\pi\)。VI 中的 \(q_k(s, a) = \sum_r p(r\mid s,a) r + \gamma \sum_{s'} p(s'\mid s,a) v_k(s')\) 把未来值 用 \(v_k\) 代,但 \(v_k\) 不是 state value,所以 \(q_k\) 也不是 q-value,只是计算用的中间表(教材在 (4.2) 下方明确说 "\(q_k\) is not an action value")。
因为:RL 文献里"q" 这个符号被多处复用,\(q_\pi\)、\(q^*\)、VI 的中间 \(q_k\)、Q-learning 的估计 \(\hat Q\) 各有微妙差别。本节 \(q_k\) 是"用 \(v_k\) 当承诺时,在 \((s, a)\) 上一步 lookahead 的值",仅仅是 \(\arg\max\) 计算的便利量;只有 \(k \to \infty\) 时 \(q_k \to q^*\)、才能合法叫 action value。
很多人以为:"VI 收敛后的 \(\pi^*\) 是从 \(\pi_0\) 不断改进的结果,\(\pi_0\) 选好能省迭代。"
实际上:VI 不需要 \(\pi_0\) —— 它的初始化只有 \(v_0\)(典型 \(v_0 = 0\)),\(\pi_1\) 是从 \(v_0\) 现场 \(\arg\max\) 算出来的。这跟 L2 PI 形成对比:PI 必须给 \(\pi_0\),因为要先 evaluate \(\pi_0\) 拿 \(v_{\pi_0}\) 再 improve。VI 输入 \(v_0\)、PI 输入 \(\pi_0\) —— L3 truncated PI 串起来时这个差别尤其重要。
因为:VI 直接把 BOE 当不动点方程解,起点是值空间的某个 \(v_0\);PI 是"策略空间走梯度",起点是策略空间的某个 \(\pi_0\) —— 设计哲学不同。

§6 自检 + 视频 + 下一节

Q1. §4 的 5×5 grid 里,\(v_0 = 0\) 起步,VI 第 1 轮(\(k = 1\))之后 \(v_1\) 在哪些格子上非零?为什么?

A. 只有 target 自身 \((3, 2)\) 上 \(v_1 = +1\)(其'stay-on-target' 动作的 \(q_0 = 1 + 0.9 \cdot 0 = 1\)),其他格子 \(v_1 = 0\)。因为:\(q_0(s, a) = r(s, a) + 0.9 \cdot v_0(s') = r(s, a)\)(\(v_0 = 0\)),非 target / 非 boundary / 非 forbidden 转移的 \(r(s, a) = 0\),target 上'stay' 的 \(r = +1\),其他动作要么 \(r = 0\)(进 \(r_\text{other}\) 格) 要么 \(r = -1\)(撞 boundary / forbidden);\(\max_a q_0(s, a)\) 大多数格子上是 0,只有 target 上是 +1。这说明VI 第 1 轮信息只能"原地" 出现,要"扩散" 到邻居得等第 2 轮 γ·v_1 起作用 —— 这就是 §4 动画里"信息一格一格扩散" 的本质。

Q2. 教材 §4.1.1 末段说"\(v_k\) 不是 state value"。如果 VI 在 \(k = 100\) 停机,我得到的 \(v_{100}\) 是谁的 state value?\(\pi_{100}\) 的吗?

A. 谁的也不是,严格来说;但近似是 \(\pi_{100}\) 的。具体:\(v_{100}\) 是 BOE 算子从 \(v_0\) 起跑 100 次的结果,理论上不满足任何 \(\pi\) 的 Bellman 方程。但 \(v_{100} \approx v^*\) (误差 \(\approx \gamma^{100} \cdot 10 \approx 2.6 \times 10^{-4}\),工程上可以忽略),\(\pi_{100}\) 是\(v_{100}\) 的贪心策略;在工程上常把 \(v_{100}\) 当作 \(v_{\pi_{100}}\) 的近似(也是 \(v^*\) 的近似)。严格的 \(v_{\pi_{100}}\) 需要再做一次策略评估(用 Ch.2 L4 闭式或迭代解 \(v = r_{\pi_{100}} + \gamma P_{\pi_{100}} v\))。这正是 L2 PI 在每轮外循环干的事 —— "把 \(\pi_k\) 评估到位",用更精准 的 \(v_{\pi_k}\) 做下一轮 \(\arg\max\)。

Q3. 把 \(\gamma\) 从 0.9 改成 0.5,\(v_0 = 0\) 起,VI 大约多少轮能让 \(\|v_k - v^*\|_\infty < 10^{-3}\)?

A. Banach: \(\|v_k - v^*\|_\infty \le \gamma^k \|v_0 - v^*\|_\infty\)。\(\gamma = 0.5\) 时 \(v^*\) 量级也变小(因为 \(R_\text{max}/(1-\gamma) = 1/0.5 = 2\)),设 \(\|v_0 - v^*\|_\infty \approx 2\)。需 \(0.5^k \cdot 2 < 10^{-3}\) 即 \(0.5^k < 5 \times 10^{-4}\),\(k > \log_{0.5} (5 \times 10^{-4}) \approx 11\) 次。大约 11 轮(对比 \(\gamma = 0.9\) 需要 65 次)。γ 越小、收敛越快,但代价是 \(\pi^*\) 变'近视'(Ch.3 L4 §4 给的反例)。这是 RL 调参里"γ 不只是 hyperparameter,是问题定义的一部分" 的又一面体现。

下一节预告

本节 VI 的核心精神是"值与策略同时逼近",中途 \(v_k\) 不是任何 \(\pi\) 的 state value。下一节(L2,Book §4.2)介绍 Policy Iteration (PI):走另一条路 —— "值与策略交替逼近",每一轮先把 \(\pi_k\) 评估到位(\(v_{\pi_k}\) 是 state value)、再用它 improve 出 \(\pi_{k+1}\)。关键问题:既然 PI 每轮要解一个完整 Bellman 方程(闭式 \(O(n^3)\) 或迭代 \(O(n^2)\) ⋅ many steps),为什么人们还用它?答:PI 的外循环步数极少(教材 5×5 例只要 10 步外循环就到 \(\pi^*\)),且每步保证单调改进 \(v_{\pi_{k+1}}(s) \ge v_{\pi_k}(s) \forall s\)(Lemma 4.1)。所以 VI 与 PI 是"多便宜外步" vs "少昂贵外步" 的两端 —— L3 会把它们统一到 truncated PI。
