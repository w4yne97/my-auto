L6: Deep Q-Network (DQN) — Basic Idea (DQN 基本思想 + 神经网络替换线性 FA)

§1 Why this lesson

L5 的 Sarsa-FA / Q-learning-FA 都是线性 形态:\(\hat q(s, a; w) = \phi(s, a)^\top w\)。线性简单可分析(L4 给的 \(A \succ 0\)、收敛证、误差界 (8.32) 都依赖线性),但表达力 受手工 \(\phi\) 限制 —— Atari 那种"输入是 84×84 像素" 的任务,没人能 设计出有效的 \(\phi(s, a)\)。本节(教材 §8.4.1, pp. 181-183)用神经网络 替线性 FA:\(\hat q(s, a; w) = \text{NN}_w(s, a)\),把特征学习 从"人工设计" 推到"NN 自动学"。这是 RL 真正进入deep 时代的转折点(Mnih et al., 2015,DeepMind Atari)。

核心 takeaway:(a) 目标函数 \(J(w) = \mathbb E[(R + \gamma \max_{a'} \hat q(S', a'; w) - \hat q(S, A; w))^2]\)(教材 (8.37)),即squared Bellman optimality error。(b) 难点:\(w\) 既出现在 \(\hat q(S, A; w)\) 也出现在 \(\max_{a'}\hat q(S', a'; w)\) ⇒ 真梯度难算。教材 §8.4.1 的第一关键 trick:引入target network \(\hat q(s, a; w_T)\),把 max 中的 \(w\) 替成 \(w_T\)(短期 frozen),目标变 \(J = \mathbb E[(R + \gamma \max_{a'}\hat q(S', a'; w_T) - \hat q(S, A; w))^2]\),梯度只对 \(w\) 求(\(w_T\) 当常数)⇒ 半梯度 update (8.38)。(c) NN 的输入设计:把 \((s, a)\) 的位置 + 动作索引 归一化后concat 成 3 维输入(教材实验);或"state-only 输入,每动作一个 output"(常见 Atari 设计)。(d) NN 训练用mini-batch + 现成框架(PyTorch / TF),不再"每步一个 SGD",而是"每步对一批样本算 MSE loss + Adam"。这是 deep RL 与 tabular / 线性 FA 的计算流程差异。

§2 核心定义(直觉先行)

  • Deep Q-network (DQN):用神经网络 \(\hat q(s, a; w)\) 逼近最优 动作值 \(q^*(s, a)\)。"deep" 仅指"有隐藏层"(教材实验用 1 层 100 神经元),不是"必须很深"。
  • (8.37) 目标函数:\(J(w) = \mathbb E[(R + \gamma \max_{a'\in\mathcal A(S')}\hat q(S', a'; w) - \hat q(S, A; w))^2]\)。这是squared Bellman optimality error:\(q^*\) 满足 \(q^*(s, a) = \mathbb E[R_{t+1} + \gamma \max_{a'} q^*(S_{t+1}, a')\,|\,S_t=s, A_t=a]\),所以 \(R + \gamma \max_{a'}\hat q(S', a'; w) - \hat q(S, A; w)\) 在 \(\hat q = q^*\) 时期望为 0。
  • 主网络 (main network) \(\hat q(s, a; w)\):每步 update 的网络。提供"当前估" \(\hat q(s, a; w)\)。
  • 目标网络 (target network) \(\hat q(s, a; w_T)\):与主网络同结构 但参数 \(w_T\) 短期 frozen。每隔 \(C\) 步("每 C iterations",教材 Algorithm 8.3)同步 \(w_T \leftarrow w\)。它的唯一作用:在算 target \(y_T = R + \gamma \max_{a'}\hat q(S', a'; w_T)\) 时把 \(w\) 替成 \(w_T\),让 target 在 \(C\) 步内是常数(不依赖当前 \(w\))。
  • (8.38) 半梯度:\(\nabla_w J = -\mathbb E[(R + \gamma \max_{a'}\hat q(S', a'; w_T) - \hat q(S, A; w))\nabla_w \hat q(S, A; w)]\)(\(w_T\) frozen,梯度只对 \(w\) 求)。这是 L3 半梯度概念在 NN 下的具体化。
  • NN 训练流程:不直接用 (8.38) 单样本 update;而用mini-batch + MSE loss:从 buffer 抽 \(B\) 条样本 \(\{(s_i, a_i, r_i, s'_i)\}\),算 \(y_{T,i} = r_i + \gamma \max_{a'}\hat q(s'_i, a'; w_T)\),loss \(= \frac{1}{B}\sum_i (y_{T,i} - \hat q(s_i, a_i; w))^2\),用 PyTorch / TF 自动梯度 + Adam update \(w\)。
  • NN 输入设计(教材实验):三种常见(i) \((\text{row}, \text{col}, a) \to \text{scalar}\)(教材 grid 实验,3 输入 1 输出);(ii) \((\text{row}, \text{col}) \to (\hat q(s, a_1), \ldots, \hat q(s, a_5))\)(2 输入 5 输出,Atari 标准);(iii) \(\text{image} \to \text{action values}\)(Atari conv-net)。设计依任务状态结构。

Step 1. "NN 替线性 FA" 的三层差异:(i) 表达力:NN 是 universal approximator(给足神经元能拟合任何连续函数),线性 FA 受 \(\phi\) 限制;(ii) 可分析性:线性 FA 有谱条件 + \(A \succ 0\) + 收敛证,NN 完全没有(损失面非凸);(iii) 算力:线性 \(\nabla_w \hat q = \phi\) 是常向量,NN \(\nabla_w \hat q\) 需 backward pass。表达力↑、可分析↓、算力↑。RL 工程"用算力 + 数据换可分析性" 是核心交换。
Step 2. target network 的引入动机。如果不用 target net,梯度 \(\nabla_w (R + \gamma \max_{a'}\hat q(S', a'; w))^2\) 含两部分:\(-2\delta \nabla_w \hat q(S, A; w) + 2\delta \cdot \gamma \nabla_w \max_{a'}\hat q(S', a'; w)\)。第二部分让 target 跟着 \(w\) 移动 ⇒ "追自己影子" ⇒ 实测发散(尤其 NN 谱条件差时)。Target net 把第二部分断掉(\(w_T\) 当常数 ⇒ 梯度为 0),只剩第一部分 ⇒ 主网络追固定 target。每 \(C\) 步同步 \(w_T \leftarrow w\) ⇒ "固定 target" 不会永远 stale,会缓慢跟进。这是 L3 半梯度概念的工程极致版:不只扔掉项,还冻住整个网络。
Step 3. (8.38) 不 直接被 NN 训练用。教材明确:NN update 用mini-batch + MSE loss + 自动微分,把 (8.38) 当数学等价描述。原因:(i) 单样本 SGD 在 NN 上方差极大,几乎不收敛;(ii) mini-batch 平滑 + Adam 自适应步长是工业标准;(iii) PyTorch / TF 自动给 \(\nabla_w \hat q\),手写 (8.38) 没必要。所以"实操 DQN" 与"(8.38) 数学" 在形式上 不一致 —— (8.38) 是解释为什么 mini-batch MSE 在做对的事,不是计算指令。

§5 常见误解 / 易混概念

很多人以为:"DQN = Q-learning + 神经网络,所以 DQN 在纯 Q-learning 公式 上跑就行。"
实际上:DQN 必须配 target network + experience replay 才稳。Mnih 2015 的 ablation 实验明确显示:去掉 target net,Atari 训练曲线剧烈震荡 甚至发散;去掉 replay,样本相关性 + off-policy 加权失配让多数任务训不起来。"NN + Q-learning" 在 1990s 已被尝试(Tesauro TD-Gammon 用了类似想法,但是 on-policy + 表式扩展),没人 在 Atari 这种规模上跑通,直到 DeepMind 2013/2015 加了两件 trick。所以"DQN" 这个名字 = "NN + Q-learning + target net + replay" 一组合,不是单 NN-Q-learning。
因为:L4 deadly triad 三联(FA + bootstrap + off-policy)在 NN 下真 触发,不是理论 paranoid。Target net 削弱 bootstrap 自反馈;replay 让加权近 均匀(削弱 off-policy 失配);两者合力把"三联可发散" 缓解到"三联实操稳"。L7 详谈两者的数学动机。
很多人以为:"target network 同步频率 C 越大越稳(target 更不动)。"
实际上:C 太大也有害。\(C\) 太小("每 1 步同步")⇒ 退化为无 target net,追自己影子 ⇒ 不稳。\(C\) 太大("每 10⁵ 步同步")⇒ \(w_T\) 长期 stale,target 远离最新 估计 ⇒ 主网络追旧 target,信息延迟 ⇒ 收敛慢且学到旧 \(q^*\)(若任务非平稳更糟)。Atari 标准 \(C \in [10^3, 10^4]\),教材 grid 实验 \(C \approx 100\)。这是"稳 vs 速" 的工程 trade-off,无理论最优值,凭实测调。
因为:target net 引入了"w 与 \(w_T\) 同步周期" 这一新超参。\(C\) 控制"主网络追的 target 有多旧"。物理上像低通滤波:\(C\) 越大,target 越平滑越慢;\(C\) 越小,target 越响应但越噪。Atari 论文(Mnih 2015)选 \(C=10^4\) 是大量调参后的值。
很多人以为:"既然 (8.38) 是 DQN 的核心梯度公式,代码实现就该直接 写它。"
实际上:实操代码不 直接写 (8.38),用MSE loss + autograd 替。流程是:(i) 算 \(y_T = r + \gamma \max_{a'}\hat q(s', a'; w_T)\)(detach w_T);(ii) 算 \(\hat q(s, a; w)\)(从主网络);(iii) loss = \((y_T - \hat q(s, a; w))^2\)(对 batch 求平均);(iv) loss.backward() + optimizer.step()。autograd 自动只对主网络的 \(w\) 求梯度,target net 因 detach 不传播 ⇒ 数学上等价 于 (8.38),但代码上更通用(任何 NN 框架都支持 MSE + autograd)。
因为:(8.38) 是"用一阶展开理解 SGD on MSE 在做什么",不是"怎么实现"。NN 训练框架已封装"给 loss 自动算梯度",不需要手写梯度。"(8.38) 等价于 MSE backward" 这条洞察 让 RL 与 supervised learning 共用同一套 NN 训练栈,这是 DQN 工程价值的一半(另一半是 trick)。

§6 自检 + 视频 + 下一节

Q1. 写出 DQN 的目标函数 (8.37) 与半梯度 (8.38),并解释 \(w_T\) 起什么作用。

A. (8.37): \(J(w) = \mathbb E[(R + \gamma \max_{a'}\hat q(S', a'; w) - \hat q(S, A; w))^2]\)。直接对 \(w\) 求梯度难:\(w\) 出现两次,链式法则给两项,target 跟着 \(w\) 移 ⇒ "追自己影子" 不稳。引入 target net \(w_T\)(短期 frozen)替 max 中的 \(w\):\(J = \mathbb E[(R + \gamma \max_{a'}\hat q(S', a'; w_T) - \hat q(S, A; w))^2]\),梯度只对 \(w\) 求 ⇒ (8.38)。\(w_T\) 的作用:截断 bootstrap 反馈,让主网络追固定 target,稳定收敛;每 \(C\) 步 \(w_T \leftarrow w\) 同步,target 缓慢跟进新估。

Q2. NN 替线性 FA 带来什么实际 好处,代价是什么?

A. 好处:(i) 表达力:NN universal approximator,可拟合任意连续函数 ⇒ 高维状态(图像、连续 control)可学;(ii) 自动特征:不再手工设计 \(\phi(s, a)\),NN 自学;(iii) 工业框架:PyTorch/TF 提供成熟训练栈,RL 与 SL 共用。代价:(i) 算力:每步 forward + backward,~10^2-10^4× 线性 FA;(ii) 可分析性:线性 FA 有 \(A \succ 0\) + 收敛证 + 误差界 (8.32);NN 完全没有,损失面非凸,只有实证;(iii) 调参:lr / batch / target sync \(C\) / replay size 都需调,且互相耦合。这是"表达力换可分析性" 的 deep RL 工程哲学。

Q3. 为什么实操 DQN 用mini-batch MSE + autograd,不直接用 (8.38)?

A. 三个原因:(i) 单样本 SGD 在 NN 上方差极大,几乎不收敛;mini-batch 平滑梯度估,稳定 backprop。(ii) Adam / RMSProp 这类自适应优化器对 NN 训练必要(SGD 朴素步长在 NN 上调不动);它们与 MSE + autograd 无缝集成。(iii) PyTorch / TF 把"给 loss 自动算梯度" 封装好,手写 (8.38) 既贵又易错。形式等价:对 detach 的 \(w_T\) 算 \((y_T - \hat q)^2\),backward 出来的梯度恰好 是 (8.38) 的样本估。所以代码用 MSE + autograd,数学解释用 (8.38),两边都对。
