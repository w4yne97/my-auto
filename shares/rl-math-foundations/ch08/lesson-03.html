L3: Optimization Algorithm — TD with Function Approximation (优化算法:从 SGD 到 TD-FA)

§1 Why this lesson

L2 把三个 目标(MSE / MSBE / MSPBE)严格写下,指出 \(v_\pi\) 未知逼着 我们用 MSPBE 类替身。本节(教材 §8.2.2-8.2.3, pp. 161-164)给具体优化算法:对 \(J(w)\) 做 SGD,把 \(v_\pi(s)\) 换成样本估(MC 回报或 TD bootstrap),立刻得带函数近似的 TD(0) 算法 —— Ch.7 L3 的 update 公式 \(V \leftarrow V + \alpha \delta_t\) 在 FA 下变 \(w \leftarrow w + \alpha \delta_t \nabla_w \hat v(s; w)\)。本节也把"半梯度" 概念落地:扔掉 \(\nabla_w \hat v(s'; w)\) 项是结构 选择 —— bootstrap 的"把 target 当常数" 直觉的形式化。

核心 takeaway:(a) "Monte-Carlo + FA" 用 \(G_t\) 估 \(v_\pi(s_t)\),SGD 直接给 \(w \leftarrow w + \alpha (G_t - \hat v(s_t; w)) \nabla_w \hat v(s_t; w)\) —— 无偏(\(\mathbb E[G_t|s_t]=v_\pi(s_t)\))、真 梯度,优化的是 MSE。(b) "TD(0) + FA" 用 \(r + \gamma \hat v(s'; w)\) 估,SGD 给 \(w \leftarrow w + \alpha [r + \gamma \hat v(s'; w) - \hat v(s; w)] \nabla_w \hat v(s; w)\) —— 有偏(target 自身依赖 \(w\))、半 梯度(扔掉 \(\nabla_w \hat v(s'; w)\) 项),优化的不 是 MSE 而是 MSPBE。(c) 线性 FA 下 \(\nabla_w \hat v(s; w) = \phi(s)\),与 \(w\) 无关 ⇒ 算法形式化:\(w \leftarrow w + \alpha \delta_t \phi(s_t)\)。这是 Ch.7 L3 update 公式的梯度形态。(d) "真梯度 vs 半梯度" 的关键差异:真梯度的 \(\nabla J^{\mathrm{TD}}\) 含 \(-\gamma \nabla_w \hat v(s'; w)\),会让 \(\hat v(s')\) 跟着 \(\hat v(s)\) 一起被推动 —— 这反而不稳定(target 跟着估计器走);半梯度钉住 target 不动,稳定但有偏。(e) 桥到 Ch.7 + Ch.6:Ch.6 RM 模板 \(w_{k+1} = w_k - \alpha_k \tilde g_k\) 在此具体化为"对 MSPBE 做 SGD,梯度估 \(\tilde g = -\delta_t \phi(s_t)\)"。

§2 核心定义(直觉先行)

  • SGD on \(J(w)\)(教材 §8.2.2):梯度下降的随机近似版:\(w_{k+1} = w_k - \alpha_k \widehat{\nabla J}(w_k)\),\(\widehat{\nabla J}\) 是单样本梯度估。Ch.6 L4-L7 的 SGD 框架,在此推到 RL 数据流。
  • 真梯度 (true gradient):\(\nabla_w J(w)\) 严格按数学求导。对 MSE \(J = \mathbb E[(v_\pi - \hat v)^2]\),\(\nabla J = -2 \mathbb E[(v_\pi - \hat v)\nabla\hat v]\)。但 \(v_\pi\) 未知,无法直接采。MC 用 \(G_t\) 替 \(v_\pi(s_t)\) ⇒ MC + FA 是真梯度 SGD on MSE。
  • 半梯度 (semi-gradient,教材 §8.2.3):对 \(J^{\mathrm{TD}} = \mathbb E[(r + \gamma \hat v(s'; w) - \hat v(s; w))^2]\),真 梯度含 \(\nabla_w \hat v(s'; w)\) 项(因 target 也依赖 \(w\));半梯度 把这项扔掉,只保留 \(-\nabla_w \hat v(s; w)\):\(w \leftarrow w + \alpha \delta_t \nabla_w \hat v(s; w)\),其中 \(\delta_t = r + \gamma \hat v(s'; w) - \hat v(s; w)\)。不是 \(J^{\mathrm{TD}}\) 的真梯度,但恰好 对应 MSPBE 的真梯度(线性 FA)。
  • "扔掉 \(\nabla_w \hat v(s')\)" 的结构动机:bootstrap 的本意是"用 \(\hat v(s')\) 当 target";target 应当不 随当前估计器即时移动 —— 否则 \(\hat v(s)\) 推一下,\(\hat v(s')\) 也跟着推,正反馈致发散。半梯度形式上"暂时把 target 当常数",截断这一反馈。L7 DQN 的 target 网络是这一思想的完整工程化。
  • 线性 FA 形态:\(\nabla_w \hat v(s;w) = \phi(s)\)(与 \(w\) 无关)。半梯度 update 简化:\(w \leftarrow w + \alpha \delta_t \phi(s_t)\)。这正是 Ch.7 L3 \(V(s) \leftarrow V(s) + \alpha \delta_t\) 在 FA 下的1:1 对应。
  • NN FA 形态:\(\nabla_w \hat v(s; w)\) 由 backprop 给出,依赖 \(w\)。半梯度 update:\(w \leftarrow w + \alpha \delta_t \nabla_w \hat v(s_t; w)\)。每步算 backprop 一次,这是 deep RL 计算开销的主要来源。
  • 动作-值 FA:把 \(\hat v(s; w)\) 换成 \(\hat q(s, a; w)\),\(\delta_t = r + \gamma \hat q(s_{t+1}, a_{t+1}; w) - \hat q(s_t, a_t; w)\)(Sarsa)或 \(r + \gamma \max_{a'}\hat q(s', a'; w) - \hat q(s,a;w)\)(Q-learning)。L5 详细处理。

Step 1. "半梯度" 的认知顺序:先理解 MC + FA 是真 SGD on MSE(无偏、收敛、慢);再把 \(v_\pi\) 换成 bootstrap target,形式上 仍写"SGD on \([\delta_t]^2\)";最后发现"真 梯度走得慢、易振",而"扔掉 \(\nabla \hat v(s')\)" 反而稳。这个"违反直觉" 的算法选择,在线性 FA 下被严格证明对应 MSPBE,在 NN 下成为 deep RL 的事实标准。
Step 2. "半梯度截断反馈" 的物理图像。\(\hat v(s)\) 与 \(\hat v(s')\) 共享参数 \(w\) ⇒ 改 \(w\) 同时 改两者。真 梯度让 \(\hat v(s)\) 推向 target,但同时 target 也跟着推 ⇒ 像追自己的影子。半 梯度把 target 当此刻常数,只移动 \(\hat v(s)\) ⇒ 收敛动力学清晰。L7 DQN 把这一思想完整工程化:用独立 frozen 网络 \(w^-\) 算 target,主网络 \(w\) 只追它,几千步后才同步 \(w^- \leftarrow w\)。
Step 3. "线性 FA = 1 字翻译 Ch.7" 的具体形态。Ch.7 L3 TD(0):\(V(s) \leftarrow V(s) + \alpha \delta_t\)。Ch.8 L3 TD-FA:\(w \leftarrow w + \alpha \delta_t \phi(s)\)。差别只在右边乘了 \(\phi(s)\):把"只更新这一格" 推广到"同时更新所有 \(\phi_i(s) \neq 0\) 的方向"。one-hot \(\phi = e_s\) 时退化成 Ch.7 表式。所以 Ch.8 L3 不是新算法,是 Ch.7 L3 的同结构推广。

§5 常见误解 / 易混概念

很多人以为:"半梯度" 是工程近似 —— 真 梯度才是数学正解,只是太贵所以半梯度顶替。
实际上:半梯度不是 真梯度的近似,是不同目标(MSPBE)的真梯度。L2 已经把这件事讲清:线性 FA 下半梯度 \(\delta_t \phi(s_t)\) 严格等于 \(\nabla_w J_{\mathrm{PBE}}\) 的单样本无偏估。真 梯度(残差梯度)对应另一 目标(不 是 MSE 也不 是 MSPBE),且实测收敛慢、振。所以"半梯度" 不是"偷懒",是"选了一个能优化的好目标"。
因为:RL 的目标函数 与算法 是耦合 的:选 MC + FA = 优化 MSE;选 TD + FA(半梯度)= 优化 MSPBE;选残差梯度 = 优化"BE 的某种平方"。每个组合各有 bias-variance-stability trade-off。"半梯度" 是"稳 + 在线 + 低方差" 那一组的合理选择。
很多人以为:"线性 FA TD(0) 一定收敛 —— 与表式 TD(0) 同结构。"
实际上:仅在 on-policy 下严格收敛(到 MSPBE 极小);off-policy 可发散(Baird 反例,L4 详谈)。表式 TD(0) 不论 on/off-policy 都收敛(只要 \((s,a)\) 充分访问),因为"每格独立" 不会触发"泛化引发的反馈"。FA 下"共享 \(w\)" 让 off-policy 样本与"错的加权" 叠加,可发散。这是 Ch.7 与 Ch.8 在收敛保证 上的本质 差异。
因为:FA 收敛分析比表式多一层:不仅要"样本足够",还要"梯度估的方向 + 加权 正确"。on-policy + \(\mu_\pi\) 加权刚好满足;off-policy 或 NN 任何一条破坏即可触发不稳。L4 把这条说严。
很多人以为:"既然 MC + FA 是真梯度且收敛到 MSE 极小,实操就该用 MC,不该用 TD。"
实际上:方差与 \(\gamma^t\) 衰减让 MC 在长 horizon 任务上几乎不可用。Atari 一局可10^4 步,\(G_t\) 的方差与 horizon 成立方(reward-by-step 噪声 + \(\gamma^t\) 累积);TD 用 \(r + \gamma \hat v(s')\) 截断在 1 步,方差常数级。再加上"等 episode 结束" 的延迟,MC 在 deep RL 几乎不用。TD 的"偏一点" 远胜 MC 的"方差爆炸 + 延迟"。
因为:"真梯度" 是形式 优势,方差 与延迟 是实操 劣势。Ch.6 RM 步长条件证收敛但不保速度,实操选 TD 是为了速度 + 稳定,接受 MSPBE-MSE 那一点点偏差。这是 RL 工程哲学的核心。

§6 自检 + 视频 + 下一节

Q1. 把"半梯度" 写成数学定义,并指出它对应哪个目标的真梯度。

A. 半梯度 update:\(w \leftarrow w + \alpha [r + \gamma \hat v(s'; w) - \hat v(s; w)] \nabla_w \hat v(s; w)\)。它扔掉了 真 \(\nabla_w[(\delta_t)^2]\) 中的 \(-2 \delta_t \cdot \gamma \nabla_w \hat v(s'; w)\) 项。线性 FA 下,半梯度的不动点 即 MSPBE \(=\|\hat v - \Pi T_\pi \hat v\|_\mu^2\) 的极小;严格说,半梯度 = MSPBE 真梯度的单样本无偏估(用 on-policy 数据)。所以"半梯度" 在线性 FA 下是另一 目标的真 梯度,不是 MSE 真梯度的近似。

Q2. 为什么残差梯度(对 \(J^{\mathrm{TD}}\) 求真梯度)在实操中不流行?

A. (i) 收敛慢:\(\nabla J^{\mathrm{TD}}\) 含 \(-\gamma \nabla_w \hat v(s')\) 项,与 \(+\nabla_w \hat v(s)\) 相反方向,实际推动力缩小;\(\gamma\) 接近 1 时几乎相消。(ii) 双采样问题:\(\nabla J^{\mathrm{TD}}\) 含 \(\mathbb E[\delta_t \cdot \nabla_w \hat v(s')]\),展开后含 \(\mathbb E[\hat v(s')] \cdot \mathbb E[\nabla \hat v(s')]\) 与 \(\mathbb E[\hat v(s') \nabla \hat v(s')]\),在同一 \(s\) 出发的 \(s'\) 上一般不等。(iii) 实测振动:\(\hat v(s)\) 与 target 同步推动,反馈环深。这三条让残差梯度在 RL 工程中失宠;半梯度成为 deep RL 默认。

Q3. 把表式 TD(0) 与 FA TD(0) update 公式并列,指出"1 字翻译" 的1 字 在哪里。

A. 表式:\(V(s_t) \leftarrow V(s_t) + \alpha \delta_t\)。FA(线性):\(w \leftarrow w + \alpha \delta_t \phi(s_t)\)。差别:表式 update 的"方向" 是"第 \(s_t\) 格的指示向量 \(e_{s_t}\)";FA update 的"方向" 是"特征向量 \(\phi(s_t)\)"。这就是1 字:把 \(e_{s_t}\) 推广到任意 \(\phi(s_t)\)。one-hot \(\phi = e_s\) 时退化成表式;\(\phi\) 多项式/RBF/NN 时是有泛化 的 FA。所以 Ch.8 L3 不是新算法,是 Ch.7 L3 的同骨架推广。
