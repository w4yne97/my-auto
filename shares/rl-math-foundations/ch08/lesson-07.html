L7: DQN — Target Network + Experience Replay (DQN 两大 trick:目标网络 + 经验回放)

§1 Why this lesson

L6 给出 DQN 目标函数 (8.37) + target network 的第一 个动机(让梯度可算 + 截断 bootstrap 反馈)。本节(教材 §8.4.1, pp. 183-184)把两大 trick 独立 摊开:(a) target network 解决"bootstrap 自反馈";(b) experience replay 解决"样本时间相关 + off-policy 加权失配"。两者缺一 都让 NN-Q-learning 实测发散(Mnih 2015 的 ablation 验证)。本节给出各自的数学动机 与实现细节,以及把它们拼成完整 DQN(Algorithm 8.3)。

核心 takeaway:(a) target net 把"追自己影子" 改成"追固定影子",每 \(C\) 步同步 \(w_T \leftarrow w\)。\(C\) 太小退化无 trick,\(C\) 太大 target 长期 stale ⇒ 实操 \(C \in [10^3, 10^4]\)(Atari)或 \(C \approx 100\)(教材 grid)。(b) Replay buffer \(\mathcal B = \{(s, a, r, s')\}\) 存最近若干 transition,每次更新主网络均匀 抽 mini-batch。两个效果:(b1) 打散时间相关(连续步同一轨迹 → 抽样后近似独立),让 SGD 收敛假设近似成立;(b2) 近似均匀分布(buffer 是多 episode 多策略 历史的混合,均匀抽 ≈ uniform 分布)⇒ (8.37) 中"(S, A) ~ Uniform" 假设近似满足 ⇒ off-policy 加权失配缓解。(c) 副效益:data efficiency —— 每条 transition 平均被用 \(B\) 次(buffer 满 \(N\)、batch \(B\)、迭代 \(K\) 次 ⇒ 每条用约 \(B K / N\) 次)。(d) 完整 DQN(Algorithm 8.3):behavior policy \(\pi_b\) 收集 \(\mathcal B\) → 均匀抽 batch → 算 \(y_T\)(用 \(w_T\)) + MSE loss + Adam → 每 \(C\) iter 同步 \(w_T \leftarrow w\)。

§2 核心定义(直觉先行)

  • Replay buffer \(\mathcal B\):一个 FIFO 容器,存最近 \(N\) 条 transition \((s_i, a_i, r_i, s'_i)\)。\(N\) 典型值 \(10^5\)-\(10^6\)(Atari);教材 grid 实验 \(N=1000\)(单个 episode 1000 步)。新 transition 进 ⇒ 旧的出。
  • 均匀抽样 (uniform sampling):每次 update 从 \(\mathcal B\) 中独立同分布 抽 \(B\) 条样本。"必须均匀"(教材 §8.4.1 反复强调)—— 因为 (8.37) 假设 \((S, A) \sim \text{Uniform}\)。Prioritized replay(Schaul et al., 2016)是非 均匀变体,需重要性采样 修正,本节不展开。
  • Target network \(w_T\):与主网络同结构异参。每 \(C\) 步同步 \(w_T \leftarrow w\)。在算 target \(y_T = r + \gamma \max_{a'}\hat q(s', a'; w_T)\) 时only 它出现,梯度不传播(autograd 的 detach)。
  • 同步频率 \(C\):超参,典型 \(10^3\)-\(10^4\)(Atari)。物理像低通滤波:\(C\) 大 ⇒ target 平滑慢、稳但延迟;\(C\) 小 ⇒ target 响应快、噪但实时。无理论最优值,凭实测调。
  • Mini-batch loss:\(L(w) = \frac{1}{B}\sum_{i \in \text{batch}}(y_{T,i} - \hat q(s_i, a_i; w))^2\),\(y_{T,i} = r_i + \gamma \max_{a'}\hat q(s'_i, a'; w_T)\)。这是教材 §8.4.1 的"TD error / loss function",对 \(B\) 条样本求平均。
  • "break correlation" 的数学含义:原始 RL 数据 \((s_t, a_t, r_t, s_{t+1})\) 来自同一 trajectory,\((s_{t+1}, a_{t+1})\) 与 \((s_t, a_t)\) 高度相关。SGD 的"独立样本" 假设要求样本独立 i.i.d.;直接训 NN 在连续步 上 ⇒ 梯度估强相关 ⇒ Adam 步长估错 ⇒ 不稳。Replay buffer 把多 episode 样本混存,均匀抽样 ⇒ 抽到的 \(B\) 条样本来自不同 时刻 / episode,近似独立。
  • "uniform 分布假设" 的数学含义:(8.37) 的 \(\mathbb E\) 含(S, A) 分布。教材选 uniform("最简"),但 behavior policy 写 buffer ⇒ \((S, A)\) 的实际分布是policy-induced(off-policy 偏离)。Replay buffer 是多策略历史 的混合,均匀抽样 \(\approx\) "把多个 policy-induced 分布混成均匀"。这是 deadly triad 中"off-policy 加权失配" 那一条的工程缓解。

§3 关键公式与 Trick 详解

本节核心是两大 trick 的独立 callout:每个 trick 一个 WHY,以及它解决什么问题。

Trick 1: Target Network(教材 §8.4.1, "The first technique is to use two networks")
WHAT: 维护一份与主网络同结构异参 的 \(\hat q(s, a; w_T)\)。算 target \(y_T = r + \gamma \max_{a'}\hat q(s', a'; w_T)\) 时用 \(w_T\),主网络只对自己的 \(w\) 求梯度。每 \(C\) 步同步 \(w_T \leftarrow w\)。
WHY (i): 截断 bootstrap 自反馈。无 target net 时,\(\hat q(S, A; w)\) 和 \(\max_{a'}\hat q(S', a'; w)\) 共用 \(w\) ⇒ update \(w\) 同时移动两边 ⇒ "追自己影子" ⇒ 正反馈循环 ⇒ 实测发散(尤其 NN 谱条件差时)。target net 把 max 中的 \(w\) 替成 frozen \(w_T\) ⇒ target 在 \(C\) 步内是常数 ⇒ 主网络追固定 目标 ⇒ 收敛动力学清晰(像监督学习的"固定标签")。
WHY (ii): 梯度可算。无 target net 时,\(\nabla_w[r + \gamma \max_{a'}\hat q(S', a'; w) - \hat q(S, A; w)]^2\) 含两个 \(\nabla_w \hat q\) 项,链式法则展开极复杂,且 \(\max\) 的不光滑 让第二个梯度 仅对最优 \(a'\) 非零(数值不稳)。target net 把 max 内的 \(w\) 替 \(w_T\) ⇒ 梯度只对前面 \(\hat q(S, A; w)\) 求 ⇒ (8.38) 简洁,autograd 自然支持。
同步频率 \(C\): 实操 \(C \in [10^3, 10^4]\)(Atari);太小退化无 trick,太大 target 长期 stale 收敛慢。教材 grid 实验 \(C \approx 100\) 即可,因网络小、任务简单。
Trick 2: Experience Replay(教材 §8.4.1, "The second technique is experience replay")
WHAT: 维护 FIFO buffer \(\mathcal B = \{(s_i, a_i, r_i, s'_i)\}\)(typical \(|\mathcal B| = 10^5\)-\(10^6\),教材 grid 实验 \(10^3\))。behavior policy \(\pi_b\) 不断写 \(\mathcal B\)。每次 update 主网络时均匀 抽 \(B\) 条样本(\(B=32\)-\(256\) 典型,教材 grid \(B=100\))。
WHY (i): 打散时间相关性。原始 trajectory \((s_0, a_0, r_1, s_1, a_1, \ldots)\) 中,\((s_t, a_t)\) 与 \((s_{t+1}, a_{t+1})\) 高度相关。SGD/Adam 假设样本独立,相关样本让梯度估 强偏 ⇒ Adam 自适应步长估错 ⇒ NN 不稳。Buffer 均匀抽样 ⇒ 抽到的 \(B\) 条来自不同时刻 / episode,近似独立 ⇒ SGD 假设近似满足。
WHY (ii): 近似均匀分布(缓解 off-policy 加权失配)。(8.37) 的 \(\mathbb E\) 用 \((S, A) \sim \text{Uniform}\),behavior policy 写的实际分布是 policy-induced 的(\(d_b \neq\) uniform)。但 buffer 是多 episode、多 policy 历史的混合(behavior policy 在训练中也变,如 ε-greedy ε 衰减),均匀抽样 \(\approx\) "多策略平均" \(\approx\) uniform。这把 deadly triad 的 off-policy 加权失配那一条显著 缓解(虽不严格消除)。
WHY (iii): Data efficiency(副效益)。每条 transition 平均被抽用 \(B \cdot K / N\) 次(\(K\) 次 update,buffer 大小 \(N\))。教材 grid 实验 \(B=100, N=1000, K=1000\) ⇒ 每条用 \(\approx 100\) 次。这让稀缺真实交互 充分挖掘,Atari 一般 1 frame 用 8 次。这是副效益,主效益是 (i)(ii)。
必须均匀抽样:(8.37) 的 \(\mathbb E\) 假设 uniform。非均匀(prioritized replay)需重要性采样修正(Schaul 2016),否则梯度估有偏,本节不展开。

把两件事合起来看 —— 完整 DQN 数据流(教材 Algorithm 8.3):

FIFO N transitions]
  Buf -- uniform sample B --> Loss[mini-batch MSE loss]
  Main[main net w] --> Loss

frozen] --> Loss

数据流:behavior policy 写 buffer;buffer 给 mini-batch;mini-batch + 主网络 + target 网络共算 loss;loss 反向 update 主网络;每 \(C\) 步主网络同步 给 target。两大 trick 各管一段:replay 处理"buffer → batch"(打散相关 + 近均匀);target net 处理"loss 算 target"(截断 bootstrap 自反馈)。两条独立但合力 让 NN-Q-learning 稳。

§4 Ablation:去掉某一 trick 会怎样

Mnih 2015 与教材 Fig. 8.12 的实验提示:两大 trick缺一 让 DQN 训练曲线显著 恶化。

读法:完整 DQN loss 单调下降至 0;无 target net loss 先降后指数发散(追自己影子的正反馈累积爆);无 replay loss 早期下降但停在中位(连续步样本相关 + off-policy 加权失配 ⇒ NN 优化方向不准,卡在 bad local)。两个 trick 各必要,无法互相替代。

§5 常见误解 / 易混概念

很多人以为:"target network 的同步频率 \(C\) 越大越稳,所以 \(C \to \infty\) 应该最好(target 永远固定)。"
实际上:\(C \to \infty\) 让 target 永远 stale。\(w_T\) 一直保持在初始化值 ⇒ \(y_T = r + \gamma \max\hat q(s', \cdot; w_T^{\text{init}})\) 是初始化随机 的 fake target ⇒ 主网络追错 target 学到一团乱码。主网络收敛到 target,但 target 是错的。所以 \(C\) 必须有限,让 target 缓慢跟进真值。实操 \(C\) 调到"主网络 \(w\) 在 \(C\) 步内变化幅度可控" 即可,典型 \(10^3\)-\(10^4\)。
因为:target net 的设计意图 是"短期固定让主网络追,长期同步让 target 不停留",像低通滤波器。\(C\) 是滤波时间常数:太小没滤,太大相位延迟过大。最佳 \(C\) 让 target 跟得上"真值在 \(\hat q\) 空间的运动" 但又比主网络慢一步。
很多人以为:"experience replay 的主效益是 data efficiency(每条样本用多次)。"
实际上:data efficiency 只是副 效益,主效益是打散时间相关 + 近似均匀分布。即使无限 样本,replay 仍必要:连续步必然 时间相关(SGD 假设破坏);behavior policy 数据必然 非均匀(off-policy 加权失配)。这两条与样本量 无关,纯粹是分布 性质。教材 §8.6 Q&A 明确:"It is necessary to break the correlation between the samples in the sequence to satisfy the assumption of uniform distribution." Data efficiency 是buffer + 均匀抽样 的顺带 好处。
因为:(8.37) 的 \(\mathbb E\) 假设 \((S, A) \sim \text{Uniform}\) 且独立。原始 RL 数据都不 满足。Replay buffer 的两条作用(打散相关、近似均匀)正分别 修这两条。Data efficiency 是"同样的样本被多次抽" 自然产生,不是设计目标。
很多人以为:"tabular Q-learning 也可以用 replay buffer,效果应该更好(data efficiency)。"
实际上:tabular Q-learning 可以 用 replay,但不必须(教材 §8.6 Q&A 明确)。tabular Q-learning 是 off-policy ⇒ 不要求样本来自当前策略 ⇒ buffer 重抽样不引入额外偏差。但 tabular 没有 deadly triad:每格独立 update,无 FA 共享 \(w\) 的放大效应 ⇒ 时间相关 + 加权失配不致命。所以 tabular Q-learning 收敛不依赖 replay,replay 只提升data efficiency。这反过来说明:DQN 中 replay 的核心价值 不是 data efficiency(若是,tabular 也必须 用 replay),而是缓解 deadly triad(只在 FA 下才致命)。
因为:tabular 没有 deadly triad ⇒ 无分布失配的致命放大;FA 有 deadly triad ⇒ 必须 工程缓解。Replay 是 FA 下"把 deadly triad 的两条(off-policy 加权 + 时间相关)同时弱化" 的设计;tabular 不需要这种缓解。

§6 自检 + 视频 + 下一节

Q1. 写出 target network 的两个动机(WHY)。

A. (i) 截断 bootstrap 自反馈:无 target net 时 \(\hat q(S, A; w)\) 与 \(\max\hat q(S', \cdot; w)\) 共用 \(w\),update 同时移动两边 ⇒ 追自己影子 ⇒ 正反馈致发散。target net 把 max 内的 \(w\) 冻成 \(w_T\) ⇒ target 短期固定 ⇒ 主网络追固定 target ⇒ 收敛动力学清晰。(ii) 梯度可算:无 target net 时 \(\nabla_w(r + \gamma \max\hat q(S', \cdot; w) - \hat q(S, A; w))^2\) 含两个 \(\nabla_w \hat q\) 项 + max 的不光滑 ⇒ 复杂且不稳;target net 让梯度只对 \(\hat q(S, A; w)\) 求 ⇒ (8.38) 简洁。两件事同源,都因"w 同时出现两次"。

Q2. 写出 experience replay 的三个动机,并指出哪个是主、哪个是副。

A. (i) 打散时间相关:连续步样本高相关,违反 SGD i.i.d. 假设;buffer 均匀抽样 ⇒ 样本近似独立。(ii) 近似均匀分布:(8.37) 假设 \((S, A) \sim \text{Uniform}\),behavior policy 数据非均匀 ⇒ off-policy 加权失配;buffer 是多策略历史混合,均匀抽样 \(\approx\) uniform ⇒ 失配缓解。(iii) data efficiency:每条样本被多次抽用。主:(i) 与 (ii)(都是 SGD 假设的近似满足,与样本量无关);副:(iii)(顺带好处)。教材 §8.6 Q&A 与 §8.4.1 都强调主效益是 (i)+(ii),不是 (iii)。

Q3. 教材为什么说"tabular Q-learning 可以用 replay 但不必须",而 DQN 必须 用 replay?

A. tabular Q-learning 没有 deadly triad:每格 \(Q(s, a)\) 独立 update,无 FA 共享 \(w\) 的放大效应 ⇒ 时间相关 + off-policy 加权失配不致命(只是慢,不发散)。tabular Q-learning 是 off-policy(target 用 \(\max\)),buffer 重抽样不引入额外偏差,可 用 replay 提升 data efficiency,但收敛不依赖 它。DQN 是 NN-FA + bootstrap + off-policy = 三联齐 ⇒ deadly triad 真 触发,无 replay 时实测发散。所以 DQN 必须 用 replay 把"off-policy 加权失配" + "时间相关" 两条同时 缓解。这是 tabular 与 FA 在稳定性需求 上的本质差。
