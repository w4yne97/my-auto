L8: DQN Implementation + Chapter 8 Summary (DQN 完整实现示例 + 全章总结 + Ch.9 桥)

§1 Why this lesson

L6-L7 给出 DQN 的核心组件(目标函数、target network、experience replay)。本节(教材 §8.4.2-8.6, pp. 184-190)做三件事:(a) 把所有组件组装 成完整 DQN(Algorithm 8.3),在 5×5 grid 上跑实验(教材 Fig. 8.11),展示"1 局 1000 步均匀 behavior + 1000 iter" 学出最优策略;(b) 做少样本 ablation(Fig. 8.12,buffer 100 步)展示 DQN 的"NN 拟合 ≠ 策略最优" 区别;(c) 总结 Ch.8 的3-4 个 key takeaway,并桥到 Ch.9。

核心 takeaway:(a) Algorithm 8.3 完整流程:behavior policy \(\pi_b\) 写 buffer \(\mathcal B\) → 均匀抽 mini-batch → 算 \(y_T\)(用 target net) → MSE loss + Adam → 每 \(C\) iter 同步 \(w_T \leftarrow w\)。教材 grid 实验:1 隐藏层 100 神经元、behavior 完全均匀、batch 100、buffer 1000、1000 iter ⇒ loss + value error 都 收敛到 0,greedy 策略全状态最优。(b) 高效率(Fig. 8.11):1000 步交互即学到最优,vs Ch.7 tabular Q-learning 需 \(10^5\) 步(Fig. 7.4)⇒ NN 泛化能力 + replay 重抽样让 deep RL 在样本量上~100× 优于 tabular。(c) Fig. 8.12 ablation:buffer 100 步 ⇒ loss 仍降到 0(NN 拟合这 100 条 样本),但 value error 不降(数据不足以恢复 \(q^*\))⇒ "NN 训得动 ≠ 策略最优";(d) 全章 takeaway:(i) FA 让大状态空间可学(Ch.7 表式不够);(ii) DQN = TD + FA + NN + 两大 trick;(iii) deadly triad 警告 + 工程缓解;(iv) 策略仍是表 形式(\(\pi(a|s)\) 显式枚举),max 限制discrete actions ⇒ Ch.9 把策略 也参数化,打开 continuous actions。

§2 核心定义(直觉先行)

  • Algorithm 8.3 完整 DQN(off-policy 版):(1) behavior policy \(\pi_b\) 在环境中采 transition 写 \(\mathcal B\);(2) 每 iteration 从 \(\mathcal B\) 均匀抽 \(B\) 条样本;(3) 对每条算 \(y_T = r + \gamma \max_{a'}\hat q(s', a'; w_T)\);(4) loss \(= \frac{1}{B}\sum (y_T - \hat q(s, a; w))^2\);(5) backward + Adam update \(w\);(6) 每 \(C\) iter 同步 \(w_T \leftarrow w\)。
  • 教材 grid 实验设置(§8.4.2,Fig. 8.11):5×5 grid,target=(4,3),forbidden 5 格;\(\gamma=0.9\),\(r_{\text{boundary}} = r_{\text{forbidden}} = -10\),\(r_{\text{target}} = 1\);behavior \(\pi_b\) 完全均匀(每动作 0.2);1 episode 1000 步;buffer \(N=1000\)(=1 episode);batch \(B=100\);1 隐藏层 100 神经元 + ReLU;3 输入(row, col, action_idx,归一化 [0,1])+ 1 输出。
  • "loss 收敛 vs value error 收敛":loss = mini-batch MSE(网络拟合 buffer 数据的好坏),value error = \(\sqrt{\mathbb E_s[(\hat q(s, a^*) - q^*(s, a^*))^2]}\)(网络真实 离 \(q^*\) 多远)。两者不一定同步:数据足时一起降,数据不足时 loss 降但 value error 不降(Fig. 8.12)。
  • "策略仍是表式" 的限制:Algorithm 8.2/8.3 的 \(\pi_t(a | s)\) 仍显式枚举所有动作(ε-greedy)。这要求(i) 动作集 \(\mathcal A\) 有限离散;(ii) 算 \(\max_a \hat q(s, a)\) 需对所有 \(a\) 各一次 前向。连续动作(机器人控制、连续力)无 max 可言 ⇒ DQN 失效。
  • Generalization 优势:tabular update 一格只影响一格,FA update 一格影响所有 \(\phi(s, a)\) 与之相关的 \((s, a)\)。教材 §8.6 Q&A:"updating one state value would not change the other state values" (tabular) vs "updating the function parameter affects the values of many states" (FA)。这是 deep RL 在样本 维度比 tabular 高效的根。
  • Stationary distribution \(d_\pi\) 的章际地位:Ch.8 引入 \(d_\pi\) 用于定义 (8.13)/(8.37) 的 \(\mathbb E\) 加权,L4 证 \(A \succ 0\) 用 \(d_\pi^\top P_\pi = d_\pi\)。Ch.9 PG 也用 \(d_\pi\)(策略目标 \(J = \mathbb E_{s \sim d_\pi}[v_\pi(s)]\))。这是 \(d_\pi\) 跨章工具。
  • Q&A(教材 §8.6)的核心问题:tabular vs FA 的 retrieve / update 差异;FA 的存储 + 泛化优势;为什么 DQN 需要 replay(uniform 假设);为什么 DQN 需要 two networks(梯度可算 + 截断反馈);NN 训练用框架(不直接写 (8.38))。这些问答串 起 Ch.8 的所有核心 trade-off。

Fig. 8.12 ablation(buffer 100 步):

读法:loss 仍 降到 0(NN 把这 100 条 样本拟合干净),但 value error 停 在 5 附近(NN 学到的 \(\hat q\) 对 buffer 准,对全状态空间偏)。这是"NN 拟合 ≠ 真值最优" 的实证:网络可能过拟 buffer,数据不足时策略不最优(Fig. 8.12c 显示部分格策略仍随机)。启示:DQN 需"足够 buffer + 足够探索",光靠"loss 降到 0" 不能保证策略最优。

Step 1. "loss 降 vs value error 降" 的语义差。Loss 是"NN 拟合给定 buffer 的好坏"(buffer 内样本的 SL 误差);value error 是"NN 学到的 \(\hat q\) 与真值 \(q^*\) 的距离"(全状态 RMSE)。两者耦合但不同步:loss 降说明网络容量足、训练成功;value error 降说明数据足、buffer 充分代表全状态。Fig. 8.12 把两者分离:数据不足时,网络仍能 拟合(loss 0),但 \(\hat q\) 偏 \(q^*\)(value error 不降)。这是 deep RL 调试时的核心工具:看 loss 判断网络问题,看 value error 判断数据问题。
Step 2. "1000 步 vs \(10^5\) 步" 的效率提升 来自三条。Ch.7 tabular Q-learning 在同 grid 上需 \(10^5\) 步(Fig. 7.4)收敛;Ch.8 DQN 只需 \(10^3\) 步(1 episode)。提升~100× 来自:(i) 泛化:tabular 学一格只影响一格,DQN 学一格影响所有 \(\phi\) 相关的 \((s, a)\) ⇒ 25 格状态互相 教学;(ii) replay 重抽样:每条 transition 平均用 100 次,样本利用率 100×;(iii) 平滑梯度:mini-batch + Adam 让每次 update 更准,无需 RM 步长慢慢衰减。三条乘起来给 ~100× 提升。这是 deep RL 在样本量 维度的真实优势(Atari 6×10^7 步学最优 vs tabular 不可能学动 Atari)。
Step 3. 策略仍是表式 ⇒ discrete actions only。Algorithm 8.2/8.3 的 \(\pi_t(a | s)\) 显式 列出每动作概率(ε-greedy 配 \(\arg\max_a\hat q\))。算 \(\max_a\hat q(s, a)\) 需对所有 \(a \in \mathcal A(s)\) 各一次 前向 ⇒ \(|\mathcal A|\) 必须有限。连续动作(机器人力矩、转向角)无 max 可言 ⇒ DQN 失效。这是 Ch.8 的关键限制,也是 Ch.9 的动机:把策略 自身参数化为 \(\pi(a | s; \theta)\),用策略梯度 直接优化 \(\theta\),无需 max ⇒ 连续动作直接可解。

§5 Chapter 8 全章4 个 takeaway

Takeaway 1: FA 让大状态空间可学。tabular(Ch.4-Ch.7)需每格存储 + 每格 update,\(|\mathcal S|\) 大到 \(10^4\) 就不可行(Atari 84×84×3 = 21168 维像素,组合状态 \(\sim 256^{21168}\))。FA 用共享参数 \(w\)(\(|w| \ll |\mathcal S|\))表达 \(\hat v\) 或 \(\hat q\),存储 + 泛化两件事一起做。代价:逼近误差 + 收敛保证削弱(L2 三目标 + L4 误差界 (8.32))。
Takeaway 2: DQN = TD + FA + NN + 两大 trick。L1-L4 给 TD-FA(线性版),L5 推到动作值,L6 把线性换 NN,L7 加 target net + replay。少任一 件 DQN 不是 DQN:无 NN 退线性 FA(Atari 跑不动);无 target net 追自己影子致发散;无 replay 时间相关 + off-policy 失配训不动。这四件 缺一不可,合起来才是"实操可跑的 deep RL"。
Takeaway 3: Deadly triad 警告 + 工程缓解。L4 给的结构警告:FA + bootstrap + off-policy 三联齐 ⇒ 可发散(Baird 反例)。缓解三策:(i) target net 削 bootstrap 反馈;(ii) replay 削 off-policy 加权失配 + 时间相关;(iii) 小 lr + 大 batch 削 FA 共享 \(w\) 的放大。三策合力 ⇒ 实操稳。但 deadly triad 不消除,只缓解;DQN 仍可能在极端任务(稀疏 reward + 长 horizon + 大动作空间)发散。
Takeaway 4: 策略仍是表 ⇒ discrete actions only(桥到 Ch.9)。Algorithm 8.2/8.3 的 \(\pi(a|s)\) 显式枚举,\(\max_a\hat q\) 要求 \(|\mathcal A|\) 有限。连续动作(机器人控制、自动驾驶转向)无 max ⇒ DQN 不可 直接用。Ch.9 的动机:不再先学 \(\hat q\) 再 max,而直接 把策略 参数化 \(\pi(a|s; \theta)\),用策略梯度 \(\nabla_\theta J(\theta)\) 优化 \(\theta\),无 max ⇒ 连续动作直接可解。Ch.10 把 Ch.8 的"critic \(\hat q\)" 与 Ch.9 的"actor \(\pi\)" 合 成 actor-critic 框架,把两者优势合并。

§6 自检 + 视频 + 下一节

Q1. 写出 Algorithm 8.3 的5 步流程,并指出每步用主网络还是 target 网络。

A. (1) behavior policy 采 transition 写 buffer(不 用网络);(2) 从 buffer 均匀抽 \(B\) 条样本(无网络);(3) 算 \(y_T = r + \gamma \max_{a'}\hat q(s', a'; w_T)\) — 用target 网络;(4) 算 \(\hat q(s, a; w)\) + loss + backward + Adam — 用主网络(梯度只对 \(w\) 传);(5) 每 \(C\) iter 同步 \(w_T \leftarrow w\) — 主网络覆盖 target 网络。关键:y_T 用 target 网络是截断 bootstrap 反馈 的核心,主网络只对自己 \(w\) 求导。

Q2. 教材 Fig. 8.12(buffer 100)中 loss 降到 0 但 value error 不降,这说明什么?

A. 说明"NN 拟合 ≠ 策略最优"。Loss = NN 拟合buffer 内 100 条样本 的 MSE,降到 0 说明网络容量足、训练收敛。Value error = NN 估 \(\hat q\) 离真值 \(q^*\) 的距离,不降说明buffer 100 条样本不足以代表全状态空间,NN 学到的 \(\hat q\) 对buffer 内 准、对buffer 外 偏 ⇒ 提取的 greedy 策略不全状态最优。启示:DQN 需足够 buffer + 足够探索;光"loss 降到 0" 不能保证策略最优。这是 deep RL 调试的核心 lens:loss 看网络,value error 看数据。

Q3. Ch.8 的 DQN 为什么不能 直接处理连续动作?Ch.9 怎么解决?

A. DQN 的核心 update 含 \(\max_{a'}\hat q(s', a'; w_T)\),要求枚举所有 \(a'\)。连续动作(\(a \in \mathbb R^d\))无 枚举,max 无法直接算(数值优化每步贵)。Ch.9 PG 不 算 max,而直接 参数化策略 \(\pi(a|s; \theta)\)(可输出连续动作的均值 + 方差),用策略梯度 \(\nabla_\theta J = \mathbb E[\nabla_\theta \log \pi_\theta(a|s) \cdot q_\pi(s, a)]\) 优化 \(\theta\)。这跳过 max,连续动作直接可解。代价:策略梯度方差大,Ch.10 actor-critic 用 \(\hat q\) 当 critic 减方差,合 Ch.8 + Ch.9 优势。

下一章预告:Ch.9 Policy Gradient

Ch.8 用"先学 \(\hat q\) 再 max" 路径学最优策略,FA 给了大状态空间 能力,但 \(\max_a\) 把我们钉 在discrete actions。Ch.9 走另一 主线:不学 \(\hat q\),直接 把策略 参数化 \(\pi(a|s; \theta)\),对策略目标 \(J(\theta) = \mathbb E_{s \sim d_\pi}[v_\pi(s)]\) 做梯度上升。核心定理(policy gradient theorem):\(\nabla_\theta J(\theta) = \mathbb E_\pi[\nabla_\theta \log \pi_\theta(a|s) \cdot q_\pi(s, a)]\) — log-derivative trick 让我们能用样本 估梯度,不需对环境模型求导。Ch.9 L1-L5 把basic idea / metric / theorem / REINFORCE / variance reduction 摊开,Ch.10 再把 \(q_\pi\)(critic)与 \(\pi_\theta\)(actor)合成 actor-critic,把 Ch.8 与 Ch.9 的优势整合。
