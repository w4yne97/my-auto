L5: Sarsa and Q-learning with Function Approximation (Sarsa / Q-learning 的 FA 版本)

§1 Why this lesson

L1-L4 把 \(\hat v(s; w)\) 的故事讲完(目标 + 算法 + 例子 + 收敛 + deadly triad 警告)。但状态值 只能"评估" 给定策略,不能直接导出 最优策略 —— 后者要 \(\hat q(s, a; w)\)。本节(教材 §8.3, pp. 179-181)把 \(\hat v\) 推到 \(\hat q\):Sarsa-FA(教材 (8.35))与 Q-learning-FA(教材 (8.36)),并把它们嵌入 generalized policy iteration(GPI)框架(算法 8.2、8.3)。

核心 takeaway:(a) 形式1:1 翻译 Ch.7 表式 —— 把 \(\hat v(s; w)\) 换成 \(\hat q(s, a; w)\),把 \(\delta_t\) 中的 \(\hat v(s_{t+1}; w)\) 换成动作-值 形式即可。Sarsa-FA: \(\delta_t = r + \gamma \hat q(s', a'; w) - \hat q(s, a; w)\),需要"第二个动作 \(a'\)"(on-policy);Q-learning-FA: \(\delta_t = r + \gamma \max_{a'}\hat q(s', a'; w) - \hat q(s, a; w)\),用 max 不依赖 \(a'\) 来源(off-policy)。(b) 半梯度 update 仍然 是 \(w \leftarrow w + \alpha \delta_t \nabla_w \hat q(s, a; w)\)。线性 FA 时 \(\nabla_w \hat q = \phi(s, a)\),退化为 Ch.7 update 乘以 \(\phi\)。(c) GPI 嵌入:每步 \(\hat q\)-update 后立即 ε-greedy 改 \(\pi\) ⇒ 收敛到给定起点 下的"good path",非全局最优(教材实验 Fig. 8.9, 8.10)。(d) Q-learning-FA 形式上 凑齐 deadly triad(FA + bootstrap + off-policy 的 max),线性 FA 下实测 多数稳(\(d_b \approx d_\pi\) + 小 \(\alpha\)),但 NN-FA 下需要 DQN trick(L6-L7)。

§2 核心定义(直觉先行)

  • 动作-值 FA \(\hat q(s, a; w)\):把"状态特征" \(\phi(s)\) 推广到"状态-动作特征" \(\phi(s, a)\)。线性 FA 形态:\(\hat q(s, a; w) = \phi(s, a)^\top w\)。一种常见构造:tile 编码—— \(\phi(s, a) = e_a \otimes \phi(s)\)(每个动作有独立 一份 \(\phi(s)\) 权重),维度 \(|\mathcal A| \cdot |\phi|\)。
  • Sarsa-FA(教材 (8.35)):\(w \leftarrow w + \alpha\, [r + \gamma \hat q(s_{t+1}, a_{t+1}; w) - \hat q(s_t, a_t; w)]\, \nabla_w \hat q(s_t, a_t; w)\)。on-policy:\(a_{t+1} \sim \pi_t(\cdot | s_{t+1})\),来自当前 策略。
  • Q-learning-FA(教材 (8.36)):\(w \leftarrow w + \alpha\, [r + \gamma \max_{a'} \hat q(s_{t+1}, a'; w) - \hat q(s_t, a_t; w)]\, \nabla_w \hat q(s_t, a_t; w)\)。off-policy:\(\max_{a'}\) 不 依赖实际采 的 \(a_{t+1}\),数据可来自任意 behavior policy。
  • 半梯度的动作-值 形态:与 \(\hat v\) 版完全同构 —— 扔掉 \(\nabla_w \hat q(s', \cdot; w)\) 项,只保留 \(\nabla_w \hat q(s, a; w)\)。线性 FA 下退化为 \(w \leftarrow w + \alpha \delta_t \phi(s, a)\),与 Ch.7 L5 \(Q(s, a) \leftarrow Q(s, a) + \alpha \delta_t\) 对应("把 1 换成 \(\phi(s, a)\)")。
  • GPI 嵌入(教材 Algorithm 8.2 / 8.3):与 Ch.7 L7 同套路。q-step:用一条样本 update \(w\)。π-step:对当前 state \(s_t\),取 \(a^* = \arg\max_a \hat q(s_t, a; w_{t+1})\),设 \(\pi_{t+1}(a^* | s_t) = 1 - \epsilon + \epsilon/|\mathcal A|\),其余动作 \(\epsilon/|\mathcal A|\)。两步交替直至 episode 终止。
  • "good path" vs "全局最优":算法 8.2 / 8.3 从固定起点 \(s_0\) 出发,只 update 路径上 的状态;非路径状态的 \(\hat q\) 不更新 ⇒ 学到的策略对起点 \(s_0\) 最优,不一定全状态最优。要全局最优需充分探索(behavior policy ε-greedy 大 ε 或多个起点)。
  • off-policy 的 deadly triad 触发:Q-learning-FA 的 \(\max_{a'}\) 让"target policy = greedy",但样本由 ε-greedy(behavior policy)采 ⇒ off-policy。再加 FA + bootstrap ⇒ 三联齐。线性 FA 实测稳(教材 Fig. 8.10,500 episodes 收敛),NN 必须用 DQN trick(L6-L7)。

直观:6 步流程每条 transition 都跑一遍。Sarsa 与 Q-learning 的差别只在 Step 2(target 用 \(\hat q(s', a')\) 还是 \(\max_{a'}\hat q(s', a')\)),其他 5 步完全相同。这就是"1 字翻译" 的结构清晰:从 Ch.7 表式到 Ch.8 FA,只多了 Step 4 的 \(\nabla_w \hat q\)。NN 时 Step 4 是 backprop,deep RL 算力开销主要在此。

Step 1. "1:1 翻译 Ch.7 表式" 的结构对应。Ch.7 L5 表式 Sarsa:\(Q(s, a) \leftarrow Q(s, a) + \alpha \delta_t\)。Ch.8 L5 Sarsa-FA:\(w \leftarrow w + \alpha \delta_t \phi(s, a)\)。差别:表式只更新"第 \((s, a)\) 格",FA 同时更新"所有 \(\phi_i(s, a) \neq 0\) 的方向"。one-hot \(\phi(s, a) = e_{(s, a)}\) 时退化成表式;\(\phi\) 共享时"泛化" 到所有 \((s', a')\) with \(\phi(s', a')\) 与 \(\phi(s, a)\) 相关。这是 FA 的核心好处:学一格惠及多格,让大状态空间可学。
Step 2. Sarsa-FA 与 Q-learning-FA 的收敛对比。Sarsa-FA 在线性 FA + on-policy 下严格收敛(Tsitsiklis-Van Roy);Q-learning-FA 无线性 FA 收敛保证(off-policy + bootstrap + FA = 三联),但实测 多数 grid-world 任务收敛(教材 Fig. 8.10)。这是"实测稳 vs 理论保证" 的典型例子。L6-L7 的 DQN 在 NN-FA 下用 target net + replay 把"实测稳" 工程化,代价是无理论收敛。
Step 3. "good path vs 全局最优" 的定位。教材 Algorithm 8.2 / 8.3 是"固定起点 \(s_0\)" 模式:每 episode 从 \(s_0\) 出发,沿当前 \(\pi\) 走到 target;只 update 路径上的 \((s, a)\) 的 \(\hat q\)。结果:路径状态 学得好,非路径状态 不更新 ⇒ 学到的策略对 \(s_0\) 最优,对其他起点可能非最优(Fig. 8.10c 看得到:左下角格的箭头是初始化随机)。要全局最优,需多起点 或大 ε 探索(Q-learning 的 off-policy 对此更友好 —— 数据可来自完全均匀的 behavior)。L8 DQN 实验用了"1 局 1000 步均匀 behavior" 几乎覆盖全状态。

§5 常见误解 / 易混概念

很多人以为:"Q-learning-FA 是 Sarsa-FA 的更强版本,既然 Q-learning 表式更优,FA 版应该也更优。"
实际上:Sarsa-FA 与 Q-learning-FA 是不同 算法,各有适用场景。Sarsa-FA 有线性 FA 收敛保证(on-policy),适合需要稳定的 任务。Q-learning-FA 无 收敛保证(off-policy + FA + bootstrap = 三联),但样本利用更广(可重用旧 buffer 数据)。NN-FA 时 Q-learning-FA 必须配 target net + replay(DQN);否则发散。Sarsa-NN 也不稳但稍好(无 max 的 off-policy)。
因为:表式时 Q-learning 的"off-policy + max" 不引发问题(每格独立);FA 时共享 \(w\) 让"错的加权" 放大到多状态。所以从 Ch.7 到 Ch.8,Q-learning 的"样本广" 优势受 deadly triad 折扣,Sarsa 的"稳" 反而升值。
很多人以为:"既然教材 Algorithm 8.2 / 8.3 学到的是 good path 不是全局最优,那这两个算法不实用,DQN 才实用。"
实际上:"good path" 对许多实际任务足够。机器人导航、游戏 AI 多数场景"从 \(s_0\) 走到 goal" 即任务定义,不需要"全状态最优策略"。Algorithm 8.2 的固定起点 + 路径 update 恰好 是这类任务的最自然设计。要全局最优(如 Atari)需要充分探索(behavior policy 大 ε 或随机 reset),但这只是采样设计,与"算法 8.2/8.3" 公式无关。
因为:"从 \(s_0\) 起好路径" vs "全局最优" 是问题设定 差别,不是算法能力 差别。Sarsa-FA / Q-learning-FA 的 update 公式不限制 起点;限制来自数据采样。改变采样(多起点、大 ε、replay)即可学全局最优。教材实验设置 选了固定起点,所以实验 是 good path,不代表算法本身 只能学 good path。
很多人以为:"线性 FA + Q-learning 实测稳 已经说明 deadly triad 是过度警告。"
实际上:线性 FA 实测稳 ≠ 理论收敛保证。教材 Fig. 8.10 的"500 episodes 收敛" 依赖(i) behavior policy ε-greedy 让 \(d_b \approx d_\pi\)(都偏 target);(ii) \(\alpha=0.001\) 极小;(iii) Fourier 特征光滑(\(\|A\|\) 谱条件好);(iv) grid 规模小,500 步 episode 多次访问每格。换大状态空间(NN)、大 \(\alpha\)、稀疏访问 ⇒ 很容易触发发散。Baird 反例就纯线性 也炸 —— 不是 NN 才会出事。
因为:deadly triad 的"触发" 需要(off-policy 加权失配)+ (FA 放大)+ (bootstrap 反馈) 三件事都足够强。线性 FA + 小 \(\alpha\) + grid 规模小 ⇒ 三件事都弱,合力不爆。NN + 大 \(\alpha\) + Atari 规模 ⇒ 三件事都强,合力炸。L7 DQN 的 trick 让三件事分别 减弱,工程上保稳。

§6 自检 + 视频 + 下一节

Q1. 写出 Sarsa-FA 与 Q-learning-FA 的 update 公式,并指出它们的唯一区别。

A. Sarsa-FA: \(w \leftarrow w + \alpha [r + \gamma \hat q(s', a'; w) - \hat q(s, a; w)]\nabla_w \hat q(s, a; w)\),其中 \(a' \sim \pi_t(\cdot | s')\)。Q-learning-FA: \(w \leftarrow w + \alpha [r + \gamma \max_{a'}\hat q(s', a'; w) - \hat q(s, a; w)]\nabla_w \hat q(s, a; w)\)。唯一区别:Sarsa 用"实际下一动作 \(a'\) 来自当前 \(\pi\)",Q-learning 用"greedy 下一动作 \(\arg\max_{a'}\)"。Sarsa = on-policy(数据加权 = \(D_\pi\));Q-learning = off-policy(target = greedy ≠ behavior)。

Q2. 为什么 Q-learning-FA 在线性 FA 下实测 稳,但 NN-FA 下必须 用 target net + replay?

A. 线性 FA 实测稳因:(i) 行为策略 ε-greedy 与 target greedy 接近 ⇒ \(d_b \approx d_\pi\) ⇒ 加权失配弱;(ii) 小 \(\alpha\)(0.001)+ 谱条件好的 Fourier 特征 ⇒ \(\|I - \alpha A\|\) 即便 A 非正定也近 1(慢移动);(iii) 小 grid + 长 episode ⇒ 每格充分访问。NN-FA 三件事都恶化:NN 谱条件差(\(\|A\|\) 大),状态空间大(每格采样稀)、大 \(\alpha\)(\(\|I-\alpha A\| > 1\) 普遍)。所以 NN 必须三道防线:target net 截断 bootstrap 自反馈;replay 让加权近均匀;mini-batch 平滑梯度。三者合力让"实测稳" 在 NN 下也成立。

Q3. 教材 Algorithm 8.2 学的是"good path" 而非"全局最优",为什么?算法本身改不改 update?

A. 因为采样设计:每 episode 从固定 \(s_0\) 出发,沿当前 \(\pi\)(ε-greedy)走到 target ⇒ 路径外的状态少访问,\(\hat q\) 不更新 ⇒ 路径外策略保持初始化随机。要全局最优不 改 update 公式,只改采样:(i) 每 episode 随机 reset;(ii) 用独立 behavior(如均匀随机)写 buffer + Q-learning off-policy 训练。L8 DQN 实验采用 (ii):1 局 1000 步均匀 behavior 几乎覆盖全 \((s, a)\),所以学到全状态最优策略。

下一节预告

L5 把 Sarsa / Q-learning 的FA 公式 写完,但 \(\hat q(s, a; w)\) 仍可能是线性 的(\(\phi^\top w\))。L6(教材 §8.4.1) 把 \(\hat q\) 换成神经网络:\(\hat q(s, a; \theta) = \text{NN}_\theta(s, a)\)。这看似 只是把"\(\phi(s, a)^\top w\)" 换成"NN",但带来三个新问题:(i) \(\nabla_\theta \hat q\) 由 backprop 给(算 Step 4 比线性贵 100×);(ii) NN 谱条件差,deadly triad 真触发(线性 FA 实测稳的 trick 失效);(iii) 数据需 mini-batch 才能稳定 backprop ⇒ 单步在线 update 不可。L6 把这三条说严,引入"主网络 + target 网络" 解决 (ii),L7 引入 experience replay 解决 (iii) 与off-policy 加权失配。
