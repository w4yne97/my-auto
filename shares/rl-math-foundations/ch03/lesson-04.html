L4: Bellman Optimality Equation (P4 - Interesting properties) (贝尔曼最优方程(P4):有趣的性质)

§1 Why this lesson

L1-L3 我们用一整章把 BOE 立了起来:定义 \(v^*\) / \(\pi^*\)、写出 BOE、用 Banach contraction 证存在唯一 + 迭代收敛。但 BOE 立起来之后,马上有一组工程问题:我把 \(\gamma\) 调小,\(\pi^*\) 会怎么变?reward 加一个常数,\(\pi^*\) 会变吗?reward 全部乘以 2,\(v^*\) 会怎么变?如果 reward 全是 0,会发生什么?这些不是花絮,是调参时不踩坑的基础 —— 本节(教材 §3.5-3.7,pp. 49-56)系统回答它们。

核心结论有两条:(1) 调 \(\gamma\) 是真换问题,\(\pi^*\) 会变 —— 减小 \(\gamma\) 让最优策略变近视(short-sighted),敢冒险变成怕冒险;(2) reward 仿射变换 \(r \to \alpha r + \beta\)(\(\alpha > 0\))不影响 \(\pi^*\),只让 \(v^*\) 按 \(\alpha v^* + \dfrac{\beta}{1-\gamma} \mathbf{1}\) 平移缩放(教材 Theorem 3.6)。这两条结论的实操含义:(a) reward 加常数(shifting)是"无害" 的、reward 乘正数(scaling)也"无害",所以工程上常用 reward 归一化(避免梯度爆炸)而不必担心改变最优策略;(b) 一旦动 \(\gamma\),你换的是问题本身,所以模型选 \(\gamma\) 时要慎重。本节的另一个有趣观察:"agent 不会无意义绕路" 不是因为 reward 设负数惩罚多走步,而是 \(\gamma\) 折扣自动惩罚了延迟 —— 这把 RL 工程里"我应不应该给每步设 -1 reward 来鼓励快速到 target" 的问题彻底澄清(答案:不必,\(\gamma\) 已经在做这件事)。本节不引入 VI/PI 算法 —— 那些是 Ch.4 的主菜,本节只关心"BOE 解出来后的性质",并把 L3 的迭代法用一次,看不同 \(\gamma\) / reward 配置下 BOE 收敛到的 \(\pi^*\) 长什么样。

§2 核心定义(直觉先行)

  • 影响 \(v^*\) 与 \(\pi^*\) 的三个参数(教材 §3.5,p. 49):观察 BOE \(v(s) = \max_\pi \sum_a \pi(a\mid s)[\sum_r p(r\mid s,a) r + \gamma \sum_{s'} p(s'\mid s,a) v(s')]\),只有 (a) reward \(r\)、(b) discount rate \(\gamma\)、(c) system model \(p(r\mid s,a), p(s'\mid s,a)\) 三类参数能影响解。本节 §3.5 主要讨论调 \(r\) 与 \(\gamma\) 时 \(v^* / \pi^*\) 的变化;模型 \(p\) 是 MDP 的物理设定,通常不动。
  • \(\gamma\) 调小 \(\Rightarrow\) \(\pi^*\) 变近视(教材 §3.5 "Impact of the discount rate" + Figure 3.4(b)(c)):把 baseline (\(\gamma = 0.9\)) 改成 \(\gamma = 0.5\),\(\pi^*\) 不再敢"穿 forbidden 走捷径",改走绕外圈安全长路;\(\gamma = 0\) 极限时,\(\pi^*\) 只看当前一步 reward,丧失"为远期 \(+1\) 而忍受当前 \(-1\)" 的远视能力,大多数格子的 \(v^* = 0\)、policy 完全乱 —— 这是极端短视 的反例。
  • reward 仿射不变性 (Theorem 3.6,p. 51):若每个 \(r \in \mathcal{R}\) 通过仿射变换变为 \(\alpha r + \beta\)(\(\alpha, \beta \in \mathbb{R}\),\(\alpha > 0\)),则新最优状态值 \(v'^* = \alpha v^* + \dfrac{\beta}{1-\gamma} \mathbf{1}\),且最优策略不变。具体含义:\(\beta = 0, \alpha > 0\) 是 scaling(rescale reward 不动 \(\pi^*\));\(\alpha = 1, \beta \ne 0\) 是 shifting(全局加常数也不动 \(\pi^*\),只是让 \(v^*\) 整体抬高 \(\beta/(1-\gamma)\))。这就是为什么工程上常 normalize reward 而无后顾之忧。
  • "agent 不会无意义绕路"(教材 §3.5 "Avoiding meaningless detours"):\(r_\text{other} = 0\) 而非 \(-1\) 时,有人会担心"反正多走几步也不扣分,agent 会不会闲逛?"答案是:不会,因为discount 已经隐式惩罚了延迟。绕一步路 \(\Rightarrow\) target 到达时刻晚一步 \(\Rightarrow\) 折扣 \(\gamma\) 多乘一次 \(\Rightarrow\) return 减少 \(\gamma^k\) 系数 —— 这不是"扣分",但等效"少给分"。教材 Figure 3.5 给了 2×2 例:\(s_2\) 选 \(\downarrow\) 直接到 target,return = 10;\(s_2\) 选 \(\leftarrow\) 绕路 \(s_2 \to s_1 \to s_3 \to s_4\),return = 8.1(\(\gamma^2/(1-\gamma)\))—— 后者明显被折扣惩罚。
  • 关于"加 -1 step penalty"(教材 §3.5 末段):有人想"如果我在每步上加 -1 reward,会不会让 agent 更快冲向 target?" 答:不会改变 \(\pi^*\)(因为 -1 是 affine shifting,Theorem 3.6 直接给出 \(\pi^*\) 不变);它只让 \(v^*\) 整体抬低 \(\dfrac{-1}{1-\gamma}\) —— 等于在所有状态上扣同一个常数,排序不变。所以"per-step penalty 用来催 agent" 是无效的工程偏方 —— \(\gamma\) 才是真正催 agent 的东西。
  • 3 种参数变换的对比表:γ rescaling \(\gamma \to \gamma'\):换问题,\(\pi^*\) 一般会变,\(v^*\) 量级 \(R_\text{max}/(1-\gamma)\) 也变;reward shifting \(r \to r + \beta\):\(\pi^*\) 不变,\(v^* \to v^* + \beta/(1-\gamma)\);reward scaling \(r \to \alpha r\)(\(\alpha > 0\)):\(\pi^*\) 不变,\(v^* \to \alpha v^*\)。记忆口诀:动 reward 是"等价变换",动 \(\gamma\) 是"换问题"。

§3 关键公式与拆解

本节核心公式是 Theorem 3.6 的仿射不变性。证明思路简明:把 \(r \to \alpha r + \beta\) 代入 BOE,用 \(P_\pi \mathbf{1} = \mathbf{1}\) 把 \(\beta\) 项凑成"常数 \(c\) 加到所有 \(v\) 上不变" 的不动点条件 \(\beta + c\gamma - c = 0\),解出 \(c = \beta/(1-\gamma)\)。完整 4-step 推导在第二个 callout 里给。

公式 1:仿射变换下的 \(v^*\) 变换(教材 §3.5 Theorem 3.6 公式 (3.8),p. 51)

Step 1. 新 BOE 长什么样。Reward 变 \(r \to \alpha r + \beta\) 后,新 \(r_\pi(s) = \sum_a \pi(a\mid s) \sum_r p(r\mid s,a)(\alpha r + \beta) = \alpha \sum_a \pi(a\mid s) \sum_r p(r\mid s,a) r + \beta = \alpha r_\pi^\text{old}(s) + \beta\)。即新 \(r_\pi = \alpha r_\pi^\text{old} + \beta \mathbf{1}\)。代入 BOE 得新 \(v' = \max_\pi (\alpha r_\pi + \beta \mathbf{1} + \gamma P_\pi v')\)(教材 (3.9))。
Step 2. 猜解 \(v' = \alpha v^* + c \mathbf{1}\) 形式。Theorem 3.6 的 ansatz 是新 \(v'\) 仍是原 \(v^*\) 的仿射变换。代入新 BOE:\(\alpha v^* + c \mathbf{1} = \max_\pi [\alpha r_\pi + \beta \mathbf{1} + \gamma P_\pi (\alpha v^* + c\mathbf{1})] = \max_\pi [\alpha r_\pi + \alpha \gamma P_\pi v^*] + \beta \mathbf{1} + c \gamma \mathbf{1}\)(用 \(P_\pi \mathbf{1} = \mathbf{1}\))。注意 \(\max_\pi [\alpha r_\pi + \alpha \gamma P_\pi v^*] = \alpha \max_\pi [r_\pi + \gamma P_\pi v^*] = \alpha v^*\)(因 \(\alpha > 0\),\(\max\) 与正系数可交换;旧 \(v^*\) 是旧 BOE 的解)。
Step 3. 解出常数 \(c\)。代回 Step 2 的等式:\(\alpha v^* + c \mathbf{1} = \alpha v^* + \beta \mathbf{1} + c \gamma \mathbf{1}\),化简得 \(c = \beta + c \gamma \Leftrightarrow c(1 - \gamma) = \beta \Leftrightarrow c = \dfrac{\beta}{1-\gamma}\)。这就是 Theorem 3.6 公式 (3.8) 中那个看似"从天而降的 \(\beta/(1-\gamma)\)"的来源 —— 它来自"常数 \(c\) 在折扣下被 \(\gamma c + \beta = c\) 这条不动点方程解出"。直觉:把 \(\beta\) 在每步上累加,折扣无穷级数 \(\sum_{k=0}^\infty \gamma^k \beta = \beta/(1-\gamma)\),正好就是常数 reward 在折扣下的 state value。
Step 4. π* 不变的最后一步。新最优策略 \(\pi'^* = \arg\max_\pi (\alpha r_\pi + \beta \mathbf{1} + \gamma P_\pi v') = \arg\max_\pi (\alpha r_\pi + \alpha \gamma P_\pi v^*)\)(\(\beta \mathbf{1} + \gamma P_\pi (c \mathbf{1}) = \beta \mathbf{1} + c\gamma \mathbf{1} = c\mathbf{1}\) 与 \(\pi\) 无关,可移出 \(\arg\max\))。又 \(\alpha > 0\),\(\arg\max_\pi \alpha [\cdot] = \arg\max_\pi [\cdot]\)。所以 \(\pi'^* = \arg\max_\pi (r_\pi + \gamma P_\pi v^*) = \pi^*\)(原最优策略)。所以仿射变换不动 \(\pi^*\)。关键洞察:仿射变换在所有动作上"同等地"调整数值,\(\arg\max\) 的排序不变。

§4 Grid-world 实例化

用 5×5 网格直观对比三种 \(\gamma\) 配置下的 \(\pi^*\) 与 \(v^*\)。target = \((3, 2)\),forbidden = \(\{(1,1),(2,1),(2,2),(1,3),(3,3)\}\),其他参数与 L2 §4 一致。下图是 \(\gamma = 0.5\) 时的最优策略与最优 \(v^*\)(教材 Figure 3.4(b)),agent 不再敢穿 forbidden,改沿绕外圈走 —— 短视化的代表。

关键差异:与 L2 §4 baseline (\(\gamma = 0.9\)) 对比,\(\pi^*\) 在 \((1, 2)\) 上从 \(\downarrow\)(穿 forbidden)变成 \(\rightarrow\)(绕外圈);\(v^*((0, 0)) \approx 5.8\) 跌到 \(\approx 0\) —— 因为 \(\gamma = 0.5\) 下,从 \((0, 0)\) 走 6 步到 target,折扣 \(0.5^6 \approx 0.0156\),即便到 target 之后每步 \(+1\) 拿到的"未来" 也被压平了。这正是教材 Figure 3.4(b) 给的现象。

γ rescaling 时收敛速率的对比图。同时跑 BOE 迭代 \(v_{k+1} = f(v_k)\),把误差 \(\|v_k - v^*\|_\infty\) 对 k 画 log-y。Banach 定理告诉我们斜率 = \(\log_{10} \gamma\),\(\gamma\) 越小越陡(收敛越快),\(\gamma\) 越接近 1 越平。注意这跟 L3 §4 用的是同一类图 —— 这次我们用它来强调"调 \(\gamma\) 不只是改 \(\pi^*\),还显著改算法收敛速度":

验证 Theorem 3.6:reward shifting/scaling 的 \(v^*\) 数值实验。取 baseline 的 \(v^*\)(\((0, 0)\) 处 \(\approx 5.8\),L2 §4)。若把所有 reward加 1 (\(\beta = 1\)),理论新 \(v'^*((0, 0)) = v^*((0, 0)) + 1/(1 - 0.9) = 5.8 + 10 = 15.8\)。若把所有 reward乘 2 (\(\alpha = 2\)),新 \(v'^*((0, 0)) = 2 \cdot 5.8 = 11.6\)。重新跑 BOE 迭代验证(本节不画图,留作 Q1 数值习题),结果完全对得上。关键:这两种变换下的 \(\pi^*\) 跟 baseline 完全相同 —— "agent 在 \((1, 2)\) 上是不是穿 forbidden" 这件事跟 reward 是不是加常数无关,只跟"当前 reward 与未来折扣 reward 的相对大小" 有关,而仿射变换不动相对大小。

"加 step penalty 不改变最优策略" 的具体证据。Baseline 中 \(r_\text{other} = 0\)(每步不扣分);若改成 \(r_\text{other} = -1\)(每步扣 1),由 Theorem 3.6(\(\alpha = 1, \beta = -1\)),理论新 \(v'^* = v^* - 10 \mathbf{1}\)、\(\pi^*\) 不变。这反驳了"设负 step penalty 能逼 agent 加速到 target" 这个常见误解:\(\pi^*\) 在 baseline 下已经是"最快可达 target" 的策略(\(\gamma\) 折扣自动惩罚延迟),再加 -1 只是把所有 \(v\) 整体往下挪 10 个单位、排序不变、\(\arg\max\) 不变、\(\pi^*\) 不变。

§5 常见误解 / 易混概念

很多人以为:"\(\gamma\) 是 hyperparameter,跟 reward 一样可调;调 \(\gamma\) 跟调 reward 应该等价。"
实际上:调 \(\gamma\) 是真换问题,调 reward(仿射变换)是等价变换。具体:仿射变换 \(r \to \alpha r + \beta\)(\(\alpha > 0\))不动 \(\pi^*\)(Theorem 3.6);\(\gamma\) 改变会让 \(\pi^*\) 变(本节 Figure 3.4(b)、L4 §4 例)。两者数学结构不同 —— reward 是 BOE 中"\(r_\pi\)" 这一加性项的内容,而 \(\gamma\) 是"折扣因子",出现在系数位置(\(\gamma P_\pi v\)),且影响压缩率(L3 收敛速度也由 \(\gamma\) 决定)。
因为:\(\gamma\) 编码"现在 vs 未来" 的权衡比例,reward 编码每个具体 \((s, a)\) 上的奖惩绝对值。比例改变会让"远期 +10 vs 当前 -1" 的 trade-off 翻转,绝对值改变只是"全场加常数"不会翻转 trade-off。这是 RL 调参里最容易混的两件事 —— 工程上 reward normalize 通常安全,\(\gamma\) 调整必须重做实验。
很多人以为:"既然 \(\gamma = 0.5\) 下 \(\pi^*\) 不敢穿 forbidden,说明 \(\gamma = 0.5\) 比 \(\gamma = 0.9\) "差"——它没找到全局最优策略。"
实际上:"全局最优" 是相对当前 \(\gamma\) 而言 的。\(\gamma = 0.5\) 下,Theorem 3.3 仍然保证 BOE 解 \(v^*_{0.5}\) 唯一存在,且 \(\pi^*_{0.5}\)(绕外圈)是 \(\gamma = 0.5\) 设定下的真正最优 —— 没有别的策略能在 \(\gamma = 0.5\) 下给出更高的 \(v\)。说"它没找到全局最优" 是把 \(\gamma = 0.9\) 的最优 \(\pi^*_{0.9}\)(穿 forbidden)当成普世标准了 —— 但这不是同一个问题的解。\(\gamma = 0.5\) 下,\(\pi^*_{0.9}\) 给的 \(v\) 反而低于 \(\pi^*_{0.5}\) 给的 \(v\)(因为 \(\gamma = 0.5\) 下"远期 +1" 被压得太狠,值不上踩 forbidden 的 -1 罚分)。
因为:\(\gamma\) 是 MDP 定义 的一部分,改它就改了优化目标;每个 \(\gamma\) 都有自己的 \((\pi^*, v^*)\) 对。Q-learning / VI 实现里"\(\gamma\) 是固定的环境配置",训练前敲定,训练中不变 —— 这反映了 \(\gamma\) 是"问题的一部分" 而不是"算法的一部分"。
很多人以为:"reward shifting 不动 \(\pi^*\) 这条性质,只在 \(r_\text{other} = 0\) 这种 baseline 下成立;若 reward 已经是负的(罚分),再加常数应该会动 \(\pi^*\)。"
实际上:Theorem 3.6 对任何 baseline reward 都成立,加多少常数都不动 \(\pi^*\)。可以这样理解:仿射变换是整体平移,所有 \(q^*(s, a)\) 都加同一个常数 \(\beta/(1-\gamma)\),\(\arg\max_a q^*(s, a)\) 不变 —— 跟 baseline 是 \(0\) 还是 \(-100\) 都一样。具体例:把 baseline 的 reward 全加 \(-100\)(每步罚 100),\(\pi^*\) 仍是穿 forbidden 那条;\(v^*\) 整体挪 \(-1000\),变成大负数,但谁比谁大这件事不动。
因为:\(\arg\max\) 只关心相对大小,加常数(or 乘正数)是 ranking-preserving 变换。这条性质在 RL 工程上极有用:你可以放心 normalize reward(zero-mean unit-var)而不担心 \(\pi^*\) 跑偏 —— 这条性质是所有 PPO / DDPG / SAC 实现里 reward normalization tricks 的理论合法性。

§6 自检 + 视频 + 下一节

Q1. Baseline (\(\gamma = 0.9\),\(r_\text{target} = +1\),\(r_\text{forbidden} = r_\text{boundary} = -1\),\(r_\text{other} = 0\))给出 \(v^*((4, 4)) = 8.1\)。若把所有 reward 加 \(\beta = 5\),新 \(v'^*((4, 4))\) 是多少?\(\pi^*\) 会变吗?

A. 由 Theorem 3.6,\(v'^* = v^* + \beta/(1-\gamma) \cdot \mathbf{1} = v^* + 5/0.1 \cdot \mathbf{1} = v^* + 50 \mathbf{1}\)。所以 \(v'^*((4, 4)) = 8.1 + 50 = 58.1\)。\(\pi^*\) 不变 —— 仿射变换是 ranking-preserving 的,每个 \(q^*(s, a)\) 都加同一个 50,\(\arg\max\) 不动。直觉验证:加 5 等于"没踩 target、没踩 forbidden、没撞墙" 的步也每步多拿 5 分;这把全局 baseline 抬高,但"穿 forbidden 还是绕外圈" 的 trade-off 跟 baseline 一样(因为穿 forbidden 的额外 -1 罚分跟 +5 全局奖励是独立 的)。

Q2. 教材 §3.5 末段说"per-step penalty 不能让 agent 更快到 target"。但你常见的开源 RL 项目里(如 Atari games)reward 仍然有 -1 step penalty。这是错的吗?

A. 不一定错,要看场景。两种情况:(a) 纯 tabular MDP + γ < 1:Theorem 3.6 严格成立,加 -1 step penalty 不影响 \(\pi^*\),只影响 \(v^*\) 数值大小 —— 在这种情形里,加 step penalty 确实是无效 的偏方,教材是对的;(b) 函数近似 (function approximation,Ch.8 起) + γ → 1 / 大状态空间:Theorem 3.6 严格依赖 \(\gamma < 1\) 与"BOE 闭式解" 的存在性;一旦用神经网络近似 \(v^*\),仿射变换可能改变梯度幅值(影响 SGD 收敛性、影响 exploration noise 的相对大小)、shifting 可能改变正负 reward 比例(影响 entropy regularization)。在这些"非理想"场景里,reward shaping(包括 step penalty)确实可能影响实际算法性能 —— 但这是算法层面的影响,不是问题层面的最优 \(\pi^*\) 变化。教材立场是"BOE 层面 \(\pi^*\) 不变",工程立场是"学习过程可能因 reward shape 变化而变快/慢",两者不矛盾,只是讨论不同的层。

Q3. 把 baseline 的 \(r_\text{forbidden}\) 从 -1 改成 -10,\(\pi^*\) 会变吗?这是 Theorem 3.6 的反例吗?

A. \(\pi^*\) 一般会变(教材 Figure 3.4(d) 直接画了:\(r_\text{forbidden} = -10\) 时 agent 不再敢穿 forbidden),但这不是 Theorem 3.6 的反例。原因:Theorem 3.6 要求所有 reward 同步仿射变换 \(r \to \alpha r + \beta\);本题只动 \(r_\text{forbidden}\) 一种,其他 reward (\(r_\text{target}, r_\text{boundary}, r_\text{other}\)) 不变 —— 这不是 仿射变换,是"非均匀" 修改 reward,改变了不同 reward 之间的相对大小,自然改了 trade-off。区别仿射变换 vs 非均匀修改 是工程调参的关键:仿射(scale/shift)是无害的,非均匀(单独改某种 reward 的权重)是真改问题。这就是 reward shaping 在实践中需要小心的根本原因。

下一章预告

到这一节为止,Ch.3 的三件大事全部完成:(1) 用 \(v^* \ge v_\pi\) 定义最优,得 BOE \(v = \max_\pi (r_\pi + \gamma P_\pi v)\);(2) 用 Banach 压缩映射定理证 BOE 解唯一存在 + 迭代收敛;(3) 用 Theorem 3.6 把"reward 仿射 vs \(\gamma\) rescale" 两类参数变换的影响讲清。但 Ch.3 一直在谈论"迭代算法 \(v_{k+1} = \max_\pi (r_\pi + \gamma P_\pi v_k)\) 会收敛",从没正面落地实现细节。下一章(Ch.4 Value Iteration & Policy Iteration)就把 BOE 落地成两个具体算法:Value Iteration (VI) 直接迭代 \(v_{k+1} = \max_\pi (r_\pi + \gamma P_\pi v_k)\)(本节预告中的"迭代法" 就叫这个);Policy Iteration (PI) 把 L1 §4 的"算 \(q\) - \(\arg\max\) - 重算 \(q\)" 思路严格化,在策略评估(用 Ch.2 闭式解或迭代 \(v_\pi\))与策略改进(\(\arg\max_a q\))之间交替。再加上truncated PI(把策略评估也截断成几轮迭代)作为 VI / PI 的折中。这是"模型已知" 情形下 RL 的最后一站 —— 之后 Ch.5 起开始拆掉"已知模型" 这个假设,进入 Monte Carlo / TD 的世界。
