L1: Motivating Example — Mean Estimation (动机示例:从样本均值开始的故事)

§1 Why this lesson

Ch.5 末段我们立了一面旗:"样本均值 \(\frac{1}{n}\sum x_i\) 真的会收敛吗?多快?为什么是 \((1/n)\) 这个权而不是别的?"。L6 §6 直接说这一刀交给 Ch.6 来切。本节(教材 §6.1,pp. 101-103)用最简单的均值估计问题 把非递推 → 递推 这一步走给我们看,顺势引出"一般步长 \(\alpha_k\)" 的悬念。

关键洞察:(a) "等所有样本到齐再算 \(\bar x\)" 不是必须。教材 (6.2) 给出递推等价式 \(w_{k+1} = w_k - \frac{1}{k}(w_k - x_k)\),把"一次性求和" 变成"每来一个样本,改一次估计"。(b) 从 \(w_1=x_1\)、\(w_2=x_1\)、\(w_3=(x_1+x_2)/2\)、\(w_4=(x_1+x_2+x_3)/3\) 一路验下去,\(w_{k+1} = \frac{1}{k}\sum_{i=1}^k x_i\) 的等式严丝合缝(教材 (6.3))。(c) 把 \(1/k\) 替换成一般 \(\alpha_k > 0\) 得到 \(w_{k+1} = w_k - \alpha_k(w_k - x_k)\)(教材 (6.4)) —— 此时 \(w_k\) 不再有闭式解,但若 \(\{\alpha_k\}\) 满足"某些温和条件",仍可证 \(w_k \to \mathbb E[X]\)。(d) 这就是 Ch.6 整章的第一颗种子:增量算法 + 一般步长 + 收敛条件 = Robbins-Monro 框架(L2-L3 详谈),SGD 是它的特例(L4-L7)。(e) Ch.7 TD 学习的"\(V(s) \leftarrow V(s) + \alpha[r + \gamma V(s') - V(s)]\)" 就是 (6.4) 的孪生兄弟:都用样本、都带步长、都收敛靠 \(\alpha_k\) 条件。

§2 核心定义(直觉先行)

  • 非递推均值估计 (non-incremental,教材 (6.1)):\(\bar x \doteq \frac{1}{n}\sum_{i=1}^n x_i\)。缺点:必须等所有 \(n\) 个样本到齐,内存 \(O(n)\),且"没看到所有数据前没有任何估计"。
  • 递推均值估计 (incremental,教材 (6.2)):\(w_{k+1} = w_k - \frac{1}{k}(w_k - x_k)\),\(w_1\) 任意。优点:每来一个 \(x_k\) 就更新一次,内存 \(O(1)\),早期就有"不准但可用" 的估计 —— 这是online learning 的雏形。
  • (6.2) 与 (6.1) 的等价性(教材 (6.3)):递推 \(k\) 步后 \(w_{k+1} = \frac{1}{k}\sum_{i=1}^k x_i\),与非递推完全相等。所以递推不是近似,是一字不差的重写 —— 只是计算次序不同。
  • 一般步长形式(教材 (6.4)):\(w_{k+1} = w_k - \alpha_k(w_k - x_k)\),\(\alpha_k > 0\)。关键:\(\alpha_k = 1/k\) 时退化成 (6.2);其他 \(\alpha_k\) 没有闭式解,但仍可能收敛。"什么条件下收敛" 的答案在 L3 RM/Dvoretzky 定理。
  • "残差形式" 的几何直觉:\(w_{k+1} - w_k = -\alpha_k(w_k - x_k)\)。当 \(w_k > x_k\),\(w\) 往下走;\(w_k < x_k\),\(w\) 往上走。\(\alpha_k\) 是步长,\((w_k - x_k)\) 是方向 ×大小。这正是 L2 RM 算法 \(w_{k+1} = w_k - \alpha_k \tilde g(w_k)\) 的雏形 —— 把 \((w_k - x_k)\) 看作 \(\tilde g(w_k, x_k)\),即"函数 \(g(w) = w - \mathbb E[X]\) 的一次带噪观测"。
  • "\(1/k\) 这个权" 的好处(直觉):\(1/k\) 让新样本权重不断变小(第 \(k\) 个样本权重 = \(1/k\)),保证"累积估计稳定";但也不能太快变小(否则信息不够),L3 会用 \(\sum 1/k = \infty\) 与 \(\sum 1/k^2 < \infty\) 把这话讲严。
  • "为什么 RL 要 incremental"(教材 §6.1 末段 + Ch.7 §7.1):RL 中数据是边交互边产生 的,你不可能"等 100 万 episode 收完再算 q-value"。incremental 是 RL 的结构性 需求,Ch.7 起所有算法(TD / SARSA / Q-learning / DQN)都是 incremental 的。

§3 关键公式与拆解

本节的核心是(6.1) → (6.2) → (6.4) 三步推导,每一步都把"下一节 RM 框架" 往前推一步。

公式 1:非递推 → 递推等价(教材 (6.1)→(6.2))

公式 2:从 \(1/k\) 推广到 \(\alpha_k\)(教材 (6.4))

Step 1. (6.1) → (6.2) 的代数恒等。已知 \(w_k = \frac{1}{k-1}\sum_{i=1}^{k-1}x_i\)(\(k \ge 2\))。计算 \(w_{k+1} = \frac{1}{k}\sum_{i=1}^k x_i = \frac{1}{k}\big[(k-1)w_k + x_k\big] = \frac{(k-1)w_k + x_k}{k} = w_k - \frac{1}{k}(w_k - x_k)\)。两端等号严格成立,无近似。所以"递推" 与"非递推" 在数值上完全相同,只是计算 schedule 不同。
Step 2. 初始条件验证(教材 p. 103 顶端)。设 \(w_1 = x_1\)。则 \(w_2 = w_1 - \frac{1}{1}(w_1 - x_1) = x_1\) ✓;\(w_3 = w_2 - \frac{1}{2}(w_2 - x_2) = x_1 - \frac{1}{2}(x_1 - x_2) = \frac{x_1+x_2}{2}\) ✓;\(w_4 = w_3 - \frac{1}{3}(w_3 - x_3) = \frac{x_1+x_2}{2} - \frac{1}{3}\big(\frac{x_1+x_2}{2} - x_3\big) = \frac{x_1+x_2+x_3}{3}\) ✓。规律一目了然 —— (6.2) 的迭代恰好 计算样本均值。
Step 3. "\(\alpha_k\) 替换 \(1/k\)" 的实操意义。在 RL 中常令 \(\alpha_k = c/k\)、\(c/k^\beta\)(\(\beta \in (0.5, 1]\))、甚至常数 \(\alpha_k \equiv 0.1\)。常数情形不收敛到 \(\mathbb E[X]\)(\(\sum \alpha_k^2 = \infty\)),但能跟踪非平稳分布 —— 这是 Q-learning 的常用配置。L3 把这条 trade-off 形式化:稳定收敛要 \(\sum \alpha^2 < \infty\),覆盖足够要 \(\sum \alpha = \infty\)。

§4 数值演示:递推 vs 非递推 vs 一般步长

本章不用 grid-world(纯抽象数学,空间分布不重要)。我们直接用 \(x_k \sim \mathcal N(\mu=2, \sigma^2=1)\) 的 i.i.d. 序列,跑三种估计方案 \(n\) 步后对比误差 \(|w_n - \mu|\):(A) 非递推 \(w_n = \frac{1}{n}\sum x_i\);(B) 递推 \(w_{k+1} = w_k - \frac{1}{k}(w_k - x_k)\);(C) 常步长 \(w_{k+1} = w_k - 0.1(w_k - x_k)\)。

直观:(A) 与 (B) 完全重合(数值恒等,这就是 (6.3))—— 递推不损失精度。(C) 常步长不收敛到 0,而是稳定在 \(\sigma\sqrt{\alpha/(2-\alpha)} \approx 0.32\)(噪声平稳值);它能跟踪 但不消解 噪声。这就是 L3 要解释的"\(\sum \alpha_k^2 < \infty\)" 必要性的具体体现 —— 常步长违反此条件。

§5 常见误解 / 易混概念

很多人以为:"递推算法 (6.2) 是 (6.1) 的近似 —— 一定有精度损失。"
实际上:(6.2) 与 (6.1) 在数值上一字不差(教材 (6.3))。Step 1-2 的代数恒等给出 \(w_{k+1}\) 在两种方法下完全相同。区别只在计算 schedule:非递推先收完所有样本再算一次;递推每收一个样本就更新。所以 incremental 不是"用精度换内存",而是"免费午餐" —— 同样精度下,内存 \(O(n) \to O(1)\),且早期就有可用估计。
因为:"递推" 是计算结构,不是数学近似。代数恒等保证两种方案产生相同的 \(w_k\) 序列。这是 online algorithm 在统计与 ML 中无所不在的根:不是因为 online "更准",而是因为 online "同样准 + 更便宜"。
很多人以为:"既然 \(\alpha_k = 1/k\) 是正解(数值上等于均值),其他 \(\alpha_k\) 都是劣解,实操应该总用 \(1/k\)。"
实际上:\(1/k\) 只在i.i.d. 平稳分布 下最优,RL 几乎从不平稳。常步长 \(\alpha_k = 0.1\) 虽然不收敛到 \(\mathbb E[X]\),但能跟踪 缓慢漂移的均值(\(\mu\) 随时间变化时)。Q-learning、SARSA 通常用常步长,因为 target 在不停更新,1/k 衰减后期太小、追不上变化。这是 deep RL 默认 \(\alpha = 1\text{e-}3\) 等常数 lr 的根。
因为:"收敛到固定点" 与"跟踪移动目标" 是对立 的诉求。\(\alpha_k \to 0\) 让算法"定下来";\(\alpha_k\) 不衰减让算法"持续追新"。RL 经常需要后者,所以 Ch.7+ 的step-size schedule 是工程艺术,L3 RM 给的纯收敛框架 是下界,不是实操终点。
很多人以为:"均值估计太简单,值不得用一整章数学工具讲。"
实际上:均值估计是最简单的 RL 子问题,所有"用样本估期望" 的算法都是它的扩展。Ch.5 MC 估 \(q(s,a) = \mathbb E[G_t \mid s, a]\):本质是均值估计。Ch.7 TD 估 \(V(s) = \mathbb E[r + \gamma V(s') \mid s]\):带 bootstrap 的均值估计。Ch.8 DQN 估 \(\nabla L(w) = \mathbb E[(\text{target} - \hat q)\nabla \hat q]\):梯度的均值估计。所有 都是 \(w_{k+1} = w_k - \alpha_k \tilde g_k\) 这一行公式的变奏。Ch.6 把这一行的数学骨架 搭好,Ch.7-Ch.10 直接用。
因为:RL 的算法多样性不在"不同的 update rule",而在"不同的 \(\tilde g\) 怎么从交互数据里采"。骨架 \(w_{k+1} = w_k - \alpha_k \tilde g\) 永远是 RM。这就是 Ch.6 在 Figure 6.1 chapter map 里被画在正中间 的原因 —— 它向上承接 Ch.4-5 的 model-based / MC,向下分发 Ch.7-10 的 TD / FA / PG / AC。

§6 自检 + 视频 + 下一节

Q1. 给出 (6.1) 与 (6.2) 数值等同的代数证明。

A. 设 \(w_k = \frac{1}{k-1}\sum_{i=1}^{k-1}x_i\)。则 \(w_{k+1} = w_k - \frac{1}{k}(w_k - x_k) = \frac{k-1}{k}w_k + \frac{1}{k}x_k = \frac{k-1}{k}\cdot\frac{1}{k-1}\sum_{i=1}^{k-1}x_i + \frac{1}{k}x_k = \frac{1}{k}\sum_{i=1}^{k-1}x_i + \frac{1}{k}x_k = \frac{1}{k}\sum_{i=1}^k x_i\)。归纳即得任意 \(k\)。这是 (6.3) 的精确证明。

Q2. 为什么 \(\alpha_k = 0.1\)(常数)在 i.i.d. 平稳分布下不 收敛到 \(\mathbb E[X]\)?用 \(w_{k+1} - \mathbb E[X]\) 的方差直观估算。

A. 写 \(\Delta_k \doteq w_k - \mathbb E[X]\)。则 \(\Delta_{k+1} = (1-\alpha)\Delta_k + \alpha(\mathbb E[X] - x_k) = (1-\alpha)\Delta_k - \alpha\eta_k\),\(\eta_k\) i.i.d. 零均值方差 \(\sigma^2\)。稳态方差 \(\text{Var}[\Delta_\infty] = \alpha\sigma^2/(2-\alpha)\)(几何级数和)。\(\alpha=0.1, \sigma=1\) ⇒ 稳态标准差 \(\sqrt{0.1/1.9} \approx 0.23\),不为零。原因:常步长不让"新噪声" 衰减,稳态下每步引入的新随机量 与"已有 \(\Delta\) 衰减掉的部分" 平衡。对应 L3 \(\sum \alpha^2 < \infty\):常 \(\alpha\) 让 \(\sum \alpha^2 = \infty\),违反 RM 第二条件,故不收敛到 0。

Q3. 把 (6.4) 写成"g(w) 的根 finding" 形式 —— 找到 \(g\)、\(\tilde g\)、\(\eta_k\),并指出根是什么。

A. 令 \(g(w) \doteq w - \mathbb E[X]\)。则 \(g\) 的根 \(w^* = \mathbb E[X]\),正是均值估计的目标。\(\tilde g(w_k, x_k) \doteq w_k - x_k = (w_k - \mathbb E[X]) + (\mathbb E[X] - x_k) = g(w_k) + \eta_k\),其中 \(\eta_k \doteq \mathbb E[X] - x_k\) 满足 \(\mathbb E[\eta_k] = 0\)、\(\mathbb E[\eta_k^2] = \sigma^2 < \infty\)。代入 (6.4) 得 \(w_{k+1} = w_k - \alpha_k \tilde g(w_k, x_k)\) —— 这正是 L2 要讲的 RM 算法 (6.5) 的精确形式。所以"均值估计" 与"解 \(g(w) = 0\) 用带噪样本" 是同一件事的不同语言。Ch.6 整章就是把这件事讲透。

本节把"均值估计" 改写成 \(w_{k+1} = w_k - \alpha_k(w_k - x_k)\),并暗示这是"解 \(g(w) = w - \mathbb E[X] = 0\)" 用带噪样本的递推。L2(教材 §6.2.1) 把这个观察推广到任意 \(g\):给定一个未知函数 \(g(w)\),只能拿到 \(\tilde g(w, \eta) = g(w) + \eta\),如何用 \(w_{k+1} = w_k - \alpha_k \tilde g(w_k, \eta_k)\) 找根 \(g(w^*) = 0\)?这就是 Robbins-Monro 算法。L2 用 \(g(w) = \tanh(w-1)\) 作图直观说明"当 \(w_k > w^*\) 时 \(g(w_k) > 0\),更新让 \(w\) 下降;当 \(w_k < w^*\) 时反之" 这个朴素的"负反馈" 机制为什么能扛住噪声 —— 关键是 \(\alpha_k \cdot \eta_k\) 的随机扰动 在适当步长下被吸收,而不是放大。
