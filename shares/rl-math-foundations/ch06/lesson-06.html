L6: BGD / SGD / MBGD — Three Forms in One Formula (BGD / SGD / MBGD:三个名字一个公式)

§1 Why this lesson

L4-L5 把 SGD 的定义 与动力学 pattern 讲清。本节(教材 §6.4.3-6.4.4,pp. 118-121)做三件事:(1) Deterministic SGD(教材 §6.4.3) —— SGD 不一定要"真随机变量",有限数据集 \(\{x_i\}_{i=1}^n\) 通过均匀抽样 也能跑 SGD;(2) BGD / SGD / MBGD 三者统一(教材 §6.4.4 + Figure 6.5) —— 三个算法是同一族 的不同 \(m\) 配置(\(m=1\) ⇒ SGD,\(m=n\) ⇒ BGD,中间 ⇒ MBGD);(3) 收敛速度 vs 单步成本 trade-off(教材 Figure 6.5 右图)—— MBGD 通常是实操甜点。

关键洞察(教材 §6.4.3-6.4.4 + Figure 6.5):(a) Deterministic SGD(教材 §6.4.3,p. 118):有限集 \(\{x_i\}_{i=1}^n\),目标 \(\min_w (1/n)\sum_i f(w, x_i)\)。把它看成随机化版本:在 \(\{x_i\}\) 上定义均匀分布 \(p(X = x_i) = 1/n\),原问题变成 \(\min_w \mathbb E[f(w, X)]\),SGD (6.13) 直接用。关键:SGD 中 \(x_k\) 是从 \(\{x_i\}\) 中随机抽样(可重复),不是按序号。(b) 三算法统一公式(教材 §6.4.4):BGD \(w_{k+1} = w_k - (\alpha_k/n)\sum_{i=1}^n \nabla_w f(w_k, x_i)\),MBGD \(w_{k+1} = w_k - (\alpha_k/m)\sum_{j \in \mathcal I_k} \nabla_w f(w_k, x_j)\),SGD \(w_{k+1} = w_k - \alpha_k \nabla_w f(w_k, x_k)\)。(c) 三者关系:\(m=1\) ⇒ SGD;\(m=n\) ⇒ MBGD ≠ BGD(MBGD 是放回抽样,可能重复;BGD 是所有 \(n\) 个);\(1 < m < n\) ⇒ MBGD。(d) 单步方差衰减:MBGD 用 \(m\) 样本平均梯度,方差降到 \(\sigma^2/m\)。\(m=50\) 比 SGD \(m=1\) 单步噪声降 50 倍,但单步成本升 50 倍。(e) Figure 6.5 右图(教材 p. 118):同 100 样本均值估计,m=50 收敛最快,m=5 中等,m=1(SGD)最慢。但 x 轴是迭代次数,m=50 一次需 50 样本,m=1 一次只需 1 ⇒ 把 x 轴换成"总样本 = m × k",三条曲线大致重合。(f) 实操推论:总样本预算给定,\(m\) 多大都差不多;但 GPU 上 mini-batch 让并行 优化生效,\(m\) 大利用硬件吞吐;\(m\) 太大让"每步 wall-clock" 长,update 频率降。实操甜点:\(m \in [32, 512]\) 是 deep learning 默认,RL 里 \(m \in [32, 256]\)。

§2 核心定义(直觉先行)

  • Batch Gradient Descent (BGD,教材 §6.4.4):\(w_{k+1} = w_k - \frac{\alpha_k}{n}\sum_{i=1}^n \nabla_w f(w_k, x_i)\)。每步用全部 \(n\) 样本,梯度估计精确(对应有限集均匀分布的真梯度)。单步成本:\(O(n)\)。
  • Mini-Batch GD (MBGD,教材 §6.4.4):\(w_{k+1} = w_k - \frac{\alpha_k}{m}\sum_{j \in \mathcal I_k} \nabla_w f(w_k, x_j)\),其中 \(\mathcal I_k \subset \{1,\dots,n\}\) 是大小 \(m\) 的随机子集(放回 i.i.d.)。单步成本:\(O(m)\),方差 \(\sigma^2/m\)。
  • Stochastic GD (SGD,教材 (6.13)):MBGD 的 \(m=1\) 退化。\(w_{k+1} = w_k - \alpha_k \nabla_w f(w_k, x_k)\)。每步成本:\(O(1)\),方差 \(\sigma^2\)(满方差)。
  • "\(m=n\) MBGD ≠ BGD"(教材 §6.4.4 中段):MBGD 用放回随机抽样,即 \(\mathcal I_k\) 中 \(n\) 个样本可重复;BGD 用所有 \(n\) 个。所以 MBGD m=n 与 BGD 在梯度估计期望 一致(都是真梯度),但方差不同 —— MBGD m=n 因抽样不全有残余方差。这是教材一个微妙点,实操影响小。
  • Deterministic SGD(教材 §6.4.3):有限集 \(\{x_i\}_{i=1}^n\),无真随机变量。把"min (1/n)Σf" 改写成"min E[f(w, X)] with p(X=x_i)=1/n",SGD 直接套用。这是机器学习教科书 SGD 的标准定义(non-RL 设定)。
  • 方差与速率的关系(教材 §6.4.4 末段):MBGD 比 SGD 收敛快,因为方差降 m 倍。但每步代价升 m 倍。在固定总样本预算 下,m 大或 m 小总样本量相同,收敛速率大致一致(教材 Figure 6.5 右图横轴换成总样本后三曲线重合)。
  • "为什么 deep learning 用 m≈64-512"(教材未明示,业界共识):GPU 并行使 \(m\) 个样本梯度同时 算成本与 \(m=1\) 几乎相同(只要 batch fit GPU memory)。\(m=1\) 不利用 GPU,\(m=10000\) 频率太低 + 梯度太"平均" 失去 SGD 跳出局部最优的好处。\(m \in [32, 512]\) 是硬件并行 + 算法噪声 + 优化频率 三者妥协。

Step 1. "\(m=n\) MBGD ≠ BGD" 的代数演示(教材 §6.4.4 中段)。BGD 的梯度估计 \(\hat g_\text{BGD} = \frac{1}{n}\sum_{i=1}^n \nabla f(w, x_i)\) 恒等于 真梯度 \(\nabla J(w)\)(对有限集均匀分布)—— 方差为 0。MBGD m=n 的梯度估计:\(\hat g_\text{MBGD} = \frac{1}{n}\sum_{j \in \mathcal I_k} \nabla f(w, x_j)\),\(\mathcal I_k\) 是从 \(\{1,\dots,n\}\) 放回抽 \(n\) 次。放回抽样 让 \(\mathcal I_k\) 可能重复(如 \(\mathcal I_k = \{1, 1, 3, 5, 5\}\) 漏掉 \(\{2, 4\}\))。方差:\(\text{Var}[\hat g_\text{MBGD}] = \sigma^2/n > 0\)。所以 MBGD m=n 是 BGD 的有偏放回近似。实操影响小(\(\sigma^2/n\) 很小),教科书中常混用。
Step 2. "固定总样本量 下三者收敛速率近似"(教材 Figure 6.5 右图)。设总样本预算 \(N\)。BGD:每步 \(n\),共 \(N/n\) 步,误差 \(\rho^{N/n}\)。MBGD m:每步 \(m\) 样本,共 \(N/m\) 步,误差 \(\sigma/\sqrt{(N/m) \cdot m} = \sigma/\sqrt N\)。SGD m=1:\(N\) 步,误差 \(\sigma/\sqrt N\)。BGD 优势:指数 vs 多项式 ⇒ 长跑下 BGD 远超 SGD/MBGD。但 BGD 单步\(O(n)\),deep learning \(n=10^6\) 时单步太贵 ⇒ 实操不可行。SGD 与 MBGD 在固定总样本下大致一致(\(\sigma/\sqrt N\)),但 MBGD 利用 GPU 并行 ⇒ wall-clock 更快。
Step 3. Deterministic SGD 的"抽样视角"(教材 §6.4.3)。给定有限集 \(\{x_i\}_{i=1}^n\),\(\bar x = (1/n)\sum x_i\)。求 \(w^* = \bar x\)(无随机性)。方法:把\(\{x_i\}\) 上定义离散均匀分布 \(p(X = x_i) = 1/n\) ⇒ \(\mathbb E[X] = \bar x = w^*\),原问题变成"估 \(\mathbb E[X]\)",直接用 SGD (6.13)。具体:每步从 \(\{x_i\}\) 中放回 抽一个 \(x_k\),\(w_{k+1} = w_k - \alpha_k(w_k - x_k)\)。关键:\(x_k\) 可重复(放回抽样 ⇒ 同一 \(x_i\) 可能被抽多次)。含义:这把"有限集优化" 与"真期望优化" 在 SGD 框架下统一,ML 教科书的 SGD 标配定义。

§4 三算法收敛速度对比

本章不用 grid-world。复刻教材 Figure 6.5 右图设定:\(\{x_i\}_{i=1}^{100}\) 在 \([-10, 10]^2\) 均匀,\(\bar x \approx 0\),起点 \((-20, 20)\),\(\alpha_k = 1/k\)。比较 \(m \in \{1, 5, 50, 100\}\) 的 \(|w_k - \bar x|\) 演化。

解读 (教材 Figure 6.5 右图):(A) BGD m=100:每步精确梯度,指数收敛 \(O(\rho^k)\),\(k=20\) 已达 \(5\times 10^{-7}\)。但每步用 100 样本,k=20 总样本 = 2000。(B) MBGD m=50:k=20 距离 0.085,总样本 1000。(C) MBGD m=5:k=20 距离 0.32,总样本 100。(D) SGD m=1:k=20 距离 1.0,总样本 20。横轴换成"总样本":同 1000 总样本下 BGD 已达 \(10^{-9}\)、MBGD m=50 达 0.085、MBGD m=5 达 0.18、SGD 达 0.20 —— BGD 仍最快,但SGD/MBGD 几乎一致。实操:数据极大 + 不可重复访问时(在线 RL),只能 SGD;静态数据集(监督学习)用 MBGD;小数据集(\(n < 1000\))才用 BGD。

§5 常见误解 / 易混概念

很多人以为:"BGD 在所有指标下都最优,只因计算贵 不实操。"
实际上:BGD 在泛化 上未必最优。深度学习实证(Keskar et al. 2017 等)发现大 batch BGD 收敛到尖锐 local min,泛化差;小 batch SGD 收敛到平坦 local min,泛化好。直觉:SGD 单步噪声让 \(w_k\) 在w-空间随机游走,自然逃离尖锐谷(尖谷被噪声"反弹" 出去)、停在平谷(平谷允许小幅游走)。BGD 没有噪声,见到啥谷停啥谷。这是 SGD 在非凸 优化(深度学习)中胜过 BGD 的关键。
因为:"找全局最优" 与"找泛化最优" 不等价。Theorem 6.4 保证 SGD 收敛到 \(w^* = \arg\min J\),但 \(J\) 是training loss,测试 loss 在 \(w^*\) 附近的形态(尖 vs 平)决定泛化。SGD 噪声意外帮助 选择"平谷 \(w\)",这是凸优化理论解释不了、实证发现的特性。Deep RL 用常 lr SGD/Adam 也是同理 —— 不是为了严格收敛,而是为了探索 + 平谷 + 跟踪非平稳 target。
很多人以为:"MBGD 比 SGD 好,只因方差降 m 倍。所以 \(m\) 越大越好。"
实际上:\(m\) 太大让更新频率降 与泛化变差。\(m = n\) 时一次 epoch 只更新一次,同样 epoch 数 收敛慢。\(m\) 太大丢失 SGD 的"平谷选择" 副产品(噪声不够)。实操:\(m \in [32, 512]\) 是 deep learning 黄金区间,deep RL 也类似。Adam、RMSProp 在这区间表现最佳。更深层:\(m\) 选择是"方差 vs 频率 vs 泛化" 三者权衡,不是单一目标。
因为:"方差降低" 与"更新频率" 是反比(同样总样本下),且泛化 这个第三维度让事情更复杂。Goyal et al. 2017 发现"线性缩放规则":batch size 翻倍时 lr 也翻倍,可保持泛化(有上限,batch 极大时崩溃)。这是工程艺术,不是数学定理。Ch.6 RM 框架给底层保证,实操调参是frontier。
很多人以为:"Deterministic SGD(教材 §6.4.3)只是修辞,与真正 SGD 没区别。"
实际上:Deterministic SGD 涉及"有限集 vs 真随机" 的微妙。真 SGD 假设 \(X\) 是真随机变量,\(\{x_k\}\) i.i.d. 抽自无限 分布;Deterministic SGD 只有有限集 \(\{x_i\}_{i=1}^n\)。区别:(i) 有限集放回抽样可重复 ⇒ 同一 \(x_i\) 可能被用多次,这等效 于无限均匀分布抽样;(ii) 不放回抽样(epoch 内每个 \(x_i\) 用一次)不 严格 SGD,但实操常用(per-epoch shuffle)。后者收敛性需要更精细的分析(without-replacement SGD 理论)。
因为:RL 实践中数据是动态生成 的(每个 episode 与环境新交互),没有"固定数据集" 概念,Deterministic SGD 不适用。监督学习 / 离线 RL 才需要"有限集 + 抽样"。Ch.8 DQN 的experience replay 是这个思想的重要应用 —— 把交互数据存 buffer,从 buffer 均匀采样作 mini-batch SGD,既稳定训练 又消除轨迹相关性。

§6 自检 + 视频 + 下一节

Q1. MBGD m=50 单步方差是 SGD m=1 的 1/50。是否意味 MBGD m=50 收敛速率是 SGD 的 50 倍?

A. 不全是。"每步" 速率是 50 倍快(误差 \(O(\sigma/\sqrt{km})\) vs \(O(\sigma/\sqrt k)\),所以 \(\sqrt{50} \approx 7\) 倍精度)。但每步成本 也升 50 倍。每总样本 速率大致一致(\(O(\sigma/\sqrt N)\),\(N = mk\))。所以 MBGD 优势不在 "样本效率",在 "wall-clock 效率"(GPU 并行让 50 样本与 1 样本单步耗时相近,频率优势保留)。结论:MBGD 是硬件并行 的胜利,不是统计 的胜利。无 GPU 时 MBGD 与 SGD 相差不大。

Q2. Deterministic SGD 与"shuffle 后逐个扫" 的 epoch SGD 的差异?哪个收敛更快?

A. Deterministic SGD = 放回抽样,可能重复。Epoch SGD = shuffle 后扫一遍,每个 \(x_i\) 恰好用一次。方差比较:Epoch SGD 的梯度估计 在每 epoch 末等于 BGD 真梯度(全部样本扫过) ⇒ 长期看比放回 SGD 方差更小、收敛更快。但理论保证:Deterministic SGD 有 Theorem 6.4 严证,Epoch SGD 严证更复杂(without-replacement SGD,Mishchenko et al. 2020)。实操:深度学习默认用 Epoch SGD(每 epoch shuffle),收敛快;严证用放回抽样作教学。RL 中无 epoch 概念(数据动态生成),只能放回抽样(experience replay)。

Q3. 把 BGD/SGD/MBGD 在 RL 中的对应说出来。Q-learning 是哪个?A2C 是哪个?

A. Tabular Q-learning(Ch.7 L7):每步用当前一个 \((s, a, r, s')\) 转移 更新 \(Q(s, a)\),即SGD m=1。DQN(Ch.8):用 experience replay 抽 m=32 mini-batch 算 loss,即MBGD m=32。A2C / A3C(Ch.10):n-step actor-critic 用n 步轨迹 算 advantage,接近MBGD m=n。PPO(Ch.10 末段):collect rollout 后多 epoch 跑 mini-batch SGD,即MBGD + replay-like。Off-policy with full buffer replay(SAC):每步从 buffer 抽 m=256,与 BGD 在数据上等价(因 buffer 大且 i.i.d. 化采样)。BGD纯净版 在 RL 中罕见(数据集不固定),通常通过 replay buffer 模拟。结论:RL 算法的"m 选择" 与监督学习同源,但通过 replay 等机制适应非平稳数据流。
