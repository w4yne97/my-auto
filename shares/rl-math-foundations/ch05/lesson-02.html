L2: MC Basic — The First Model-Free RL (MC Basic 介绍)

§1 Why this lesson

L1 给我们"用样本均值估期望" 的钥匙。本节(教材 §5.2.1-5.2.2,pp. 80-83)把这把钥匙插进 PI 的锁孔:把 PI 的 PE 步("用 \(P_\pi\) 解 Bellman 方程")替换成"跑 episode,用样本均值估 \(q_{\pi_k}(s, a)\)"。policy improvement 不变。整本书第一个 model-free RL 算法 —— MC Basic —— 由此诞生。

关键洞察(教材 §5.2.1 + Algorithm 5.1):(a) 为什么直接估 \(q_{\pi_k}\) 而不是 \(v_{\pi_k}\):PI 的 improvement 步要 \(\arg\max_a q\)。如果只估 \(v_{\pi_k}\),还得用 \(q = \sum_r p(r\mid s,a)r + \gamma \sum_{s'} p(s'\mid s,a) v(s')\)(公式 5.1)从 \(v\) 算 \(q\) —— 这又把 \(p\) 拉了回来。所以 model-free RL 直接估 \(q\),把 \(p\) 完全踢出去。(b) 公式 (5.2) 的本质:\(q_{\pi_k}(s, a) = \mathbb E[G_t \mid S_t = s, A_t = a] \approx \frac{1}{n}\sum_{i=1}^n g_{\pi_k}^{(i)}(s, a)\)。这就是 L1 大数定律 + RL 估值的合体公式。(c) 算法骨架不变:外层"评估 → 改进" 仍是 GPI(Ch.4 §4.4 介绍的统一抽象);只是 PE 内层从"解线性系统" 变成"采样 + 平均"。(d) 与 PI 的收敛关系:PI 收敛 ⇒ 给足样本时 MC Basic 也收敛 —— "充分多 episode" 把样本均值推近真 \(q_{\pi_k}\),与 PI 等价。但实操中 MC Basic 样本效率太低(每 (s, a) 都要 collect "充分多" 条),所以是教学型 算法,工程版要等 L4 / L6 的 efficient 变体。

§2 核心定义(直觉先行)

  • MC Basic 算法 (Algorithm 5.1,p. 82):输入 \(\pi_0\)。每外循环 \(k\):对每个 \((s, a)\) collect "充分多" 条 episode → \(q_k(s, a) = \) 样本均值;policy improvement 把 \(\pi_{k+1}(s) = \arg\max_a q_k(s, a)\)。本质:把 PI 的 PE 替换成 MC 估计,其余不变。
  • g_{π_k}^{(i)}(s, a):第 \(i\) 条 episode(从 \((s, a)\) 起,跟 \(\pi_k\))得到的discounted return。是公式 (5.2) 求平均的对象。
  • "为什么直接估 q 而非 v":improvement 步要 \(\arg\max_a q(s, a)\)。从 \(v\) 推 \(q\) 需要公式 (5.1) 的 \(p(r\mid s,a),\ p(s'\mid s,a)\) —— 这正是 model-free 要避开的。MC Basic 直接估 \(q\),把 \(p\) 完全踢掉。这是 model-based vs model-free 切换的精确点。
  • 每个 (s, a) 都要采样的限制:Algorithm 5.1 要求"对每一个 (s, a) 都收集充分多 episode"。这意味着每个 episode 必须从指定的 \((s_0, a_0)\) 开始 —— 即"exploring starts" 假设(L4 重点)。MC Basic 默认这个条件成立,L4 / L5 才开始拆这个假设。
  • GPI (generalized policy iteration) 实例:Ch.4 §4.4 把"值评估 ↔ 策略改进" 的交替骨架命名为 GPI。MC Basic = GPI 在"PE 用样本均值" 这个特例下的具体落地。Ch.5-Ch.10 大部分算法都是 GPI 的不同 PE / improvement 实现。
  • 收敛(教材 §5.2.2 末段):PI 收敛 → 给足样本时 \(\hat q \to q_{\pi_k}\) → MC Basic 等价于 PI → 收敛到 \(\pi^*\)。"充分多" 这个量化条件来自 L1 的大数定律。注:实际算法里"充分多" 不可能严格满足,所以 MC Basic 的近似 收敛性更接近 truncated PI(Ch.4 L3) —— PE 没跑到精确,但仍能逼近 \(\pi^*\)。
  • 样本效率短板(教材 §5.2.2 末段):MC Basic 太朴素,每 (s, a) 都要单独 collect 充分多 episode。\(|\mathcal S| \times |\mathcal A|\) 大时根本跑不动。所以 MC Basic 是"教学版",L4 / L6 的 efficient 变体才是工程版。

Step 1. Algorithm 5.1 骨架(p. 82)。外循环 \(k = 0, 1, 2, \dots\)。内层 1 (PE):对每个 \(s \in \mathcal S\) 与 \(a \in \mathcal A(s)\),从 \((s, a)\) 起按 \(\pi_k\) 跑充分多 episode,取平均得 \(q_k(s, a) \approx q_{\pi_k}(s, a)\)。内层 2 (improvement):\(a_k^*(s) = \arg\max_a q_k(s, a)\),\(\pi_{k+1}(a \mid s) = 1\) if \(a = a_k^*(s)\) else 0。对照 PI:外循环骨架完全相同,差别只在 PE 实现 —— PI 解线性系统,MC Basic 跑样本。
Step 2. "为什么直接估 q,不估 v 后再算 q"。如果先估 \(v_{\pi_k}\)(L1 公式 \(\bar G \approx v_\pi\)),再用公式 (5.1) 推 \(q\),会再次需要 \(p(r\mid s,a)\) 和 \(p(s'\mid s,a)\) —— 这就把 model 拉了回来,违背 model-free 初衷。所以 MC Basic 直接对 \((s, a)\) 估 \(q\):每条 episode 从 \((s, a)\) 起跑,得到的 return 就是 \(g^{(i)}(s, a)\)。这是"q-估计是 MC 的 first-class 设计选择" 的根源。
Step 3. 收敛性(教材 §5.2.2 末段)。三步:(i) PI 严格收敛(Ch.4 L2 §3 Theorem 4.1);(ii) 给足样本时 \(\hat q_{\pi_k} \to q_{\pi_k}\)(L1 大数定律);(iii) 内嵌等价 ⇒ MC Basic 在 \(n \to \infty\) 极限下与 PI 行为相同 ⇒ 收敛到 \(v^*, \pi^*\)。实操含义:有限 \(n\) 时 \(\hat q\) 是带噪 的近似,MC Basic 实际更像 truncated PI(Ch.4 L3) —— PE 没解到精确,但靠 GPI "多跑几个外步弥补"。

§5 常见误解 / 易混概念

很多人以为:"MC Basic 直接把 PI 的 PE 替换掉,所以 \(\hat v\) 就是中间产物 —— 估了 v 再用公式 (5.1) 推 q。"
实际上:MC Basic 直接估 \(q\),不经过 \(v\)。原因:improvement 步要 \(\arg\max_a q\),从 \(v\) 推 \(q\) 需要 \(p\)(公式 5.1)—— 那 \(p\) 又被拉回来了,违背 model-free 设计。所以教材 Algorithm 5.1 直接 collect (s, a)-episode 估 \(q\),完全跳过 \(v\)。"q is the first-class citizen in MC" 是 Ch.5-Ch.7 所有 MC/TD 算法(包括 Q-learning)的共同设计原则。
因为:v 与 q 是双胞胎(\(v(s) = \sum_a \pi(a\mid s) q(s, a)\)),但 v → q 的反向变换需要 model。所以 model-free 选 q 当主语,model-based 选 v 当主语。这件事在 Ch.7 Q-learning 里再次出现 —— Q-learning 不维护 v,只维护 q。
很多人以为:"MC Basic 比 PI 慢是因为采样 比解方程 慢。"
实际上:MC Basic 慢是因为样本量需求大。对每个 (s, a) 要 collect "充分多" episode —— "充分多" 取决于 var(G) 与目标精度,典型几十到几千条。45 个 (s, a) 对 × 1000 episode/对 = 45000 episode。每条 episode 又有十几步。所以 MC Basic 总样本量比 PI 高 4-6 个数量级。这就是为什么 MC Basic 是教学版,不是工程版。
因为:样本均值方差衰减是 \(O(1/n)\),所以"从 0.1 到 0.01" 要 100 倍样本(L1 Q2)。L4 Exploring Starts 用 every-visit 把"一条 episode" 摊开成"多个子 episode",L5 ε-greedy 用"一条超长 episode" 替"多条独立 episode",都是为了榨干样本。Ch.7 TD 用 bootstrap 进一步降方差。
很多人以为:"MC Basic 算法的'GPI 实例' 标签是个标签游戏,没实际意义。"
实际上:GPI 视角让 Ch.5-Ch.10 全章无缝衔接。MC Basic = GPI(PE = 样本均值);TD(Ch.7) = GPI(PE = bootstrap);Q-learning = GPI(PE = bootstrap + max);Actor-Critic(Ch.10) = GPI(PE = critic 网络,improvement = actor 梯度)。读者只要抓住"外循环 PE ↔ improvement 交替" 这一张图,就能把 50 节课的 RL 算法世界压缩成"每个算法填两个空"。
因为:RL 算法本质上是"估值 ↔ 改策略" 的反馈回路。GPI 给出这个回路的抽象骨架,具体填法是设计选择。Ch.5 L1-L6 学的是 PE 怎么填(从样本均值 → first/every-visit → ε-greedy 取代 exploring-starts),improvement 仍是 \(\arg\max\)。Ch.7 起才开始动 improvement 一侧。

§6 自检 + 视频 + 下一节

Q1. MC Basic 直接估 \(q\) 而不估 \(v\)。如果硬要估 \(v\) 再算 \(q\),会发生什么?

A. 把 \(v\) 估完后,要用 \(q(s, a) = \sum_r p(r\mid s,a) r + \gamma \sum_{s'} p(s'\mid s,a) v(s')\)(公式 5.1)算 \(q\)。这条公式需要 \(p(r\mid s,a)\) 和 \(p(s'\mid s,a)\) —— 正是 model-free 要踢掉的。所以"估 v 再推 q" 路线会把 model 拉回来,违背初衷。MC Basic 选择"直接估 q",彻底切断对 \(p\) 的依赖。这件事在 Q-learning(Ch.7)里再次成为设计原则:Q-learning 永远不维护 \(v\),只维护 \(q\)。

Q2. Algorithm 5.1 第二行"对每个 (s, a) collect 充分多 episode" —— 这条要求暗藏什么假设?

A. Exploring starts 假设(L4 主题):每条 episode 必须能从指定的 \((s_0 = s, a_0 = a)\) 起。实操中这意味着 agent 可以任意 选择初始 state-action 对 —— 这在仿真器里成立,但在真实环境(机器人、推荐)里几乎不成立。MC Basic 默认这个条件成立,L4 用 every-visit 在一定程度上缓解(一条 episode 可估多个 (s, a)),L5 用 ε-greedy 彻底替换("不需要任意初始(s, a),只要策略 soft 让所有 (s, a) 都有正概率被访问")。这是 Ch.5 后半的主线张力。

Q3. 教材 §5.2.2 末段说"MC Basic 是 PI 的近似版",PI 收敛 ⇒ MC Basic 收敛。但严格地,样本数 \(n\) 必须多大才"够"?

A. 没有有限的"够"。L1 大数定律给的是 \(n \to \infty\) 时 \(\hat q \to q\),不给有限 \(n\) 时的精度保证。Hoeffding 等集中不等式可给"误差 \(\le \varepsilon\) 的概率 \(\ge 1 - \delta\)" 类型的样本复杂度 上界,但要求 reward 有界 + i.i.d.,且 \(n = O(\frac{1}{\varepsilon^2} \log \frac{1}{\delta})\) —— 通常很大。所以 MC Basic 实际靠"外循环 GPI 多跑几步弥补 PE 不精确",其行为更接近 truncated PI(Ch.4 L3)。Ch.6 RM 框架会给一般 stochastic approximation 的收敛证明,Ch.7 TD 在此框架下证明 \(\hat q\) 几乎必然收敛到 \(q^*\)。
