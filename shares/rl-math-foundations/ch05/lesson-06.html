L6: MC ε-Greedy + Chapter Summary (MC ε-贪心算法 + 全章总结)

§1 Why this lesson

L5 给 ε-greedy 策略的形态 与improvement 公式(教材 (5.5))。本节(教材 §5.4.3-5.7,pp. 91-100)做三件事:(1) Algorithm 5.3 把 ε-greedy 嵌入完整 MC 算法,跑一条超长 episode 学到接近 \(\pi^*\) 的策略;(2) §5.5 直面"探索 vs 利用" 的数值代价(Figure 5.6 / 5.7) —— ε 越大 v 越低,grid 覆盖越好;(3) §5.6 全章总结 + Ch.6 预告 —— "(1/n) 这个权" 的严格数学根基交给 Ch.6 RM/SGD。

关键洞察(教材 §5.4.3-5.6):(a) Algorithm 5.3 (p. 91) 与 Algorithm 5.2(MC ES)只差一行:improvement step 从"\(\pi(a^* \mid s) = 1\) else 0" 变成"\(\pi(a^* \mid s) = 1 - \frac{|\mathcal A| - 1}{|\mathcal A|}\varepsilon\) else \(\varepsilon/|\mathcal A|\)"。(b) 单 episode 学习极端例子(p. 91 末):5×5 grid,1 million step 一条 episode,ε=0.5 跑 2 次外迭代就收敛到 ε-greedy 最优。(c) 探索 vs 利用代价(§5.5,Figure 5.6/5.7):同一 grid 下 \(v_{\pi^*_\varepsilon}\) 随 ε 单调下降:ε=0 时 \(v\) 中心 = 10,ε=0.1 时 = 3.7,ε=0.2 时 = -2.0,ε=0.5 时 = -17(target 反成"负值高位"!)—— ε 太大 agent 反而不愿 留在 target,因为 ε 概率乱走会踩 forbidden 受罚。(d) 探索能力(§5.5 末):ε=1 时一条百万步 episode 几乎 uniform 覆盖每个 (s, a)(变异系数 0.02);ε=0.5 时覆盖严重不均(\(10^4 \sim 10^5\) 倍方差)。(e) Ch.5 大总结(§5.6):三个算法 = "策略迭代用样本均值替换 model-based PE" 的连续光谱;MC Basic(教学版)→ MC ES(every-visit + per-episode update)→ MC ε-Greedy(soft 策略去 exploring starts)。(f) 共同短板:全章 MC 都有高方差(完整 episode return 的统计噪声大),Ch.7 TD bootstrap 大幅降方差。(g) 数学根基(Ch.6 预告):"样本均值 (1/n)\(\sum g\) 收敛" 这件事的严格数学是 Robbins-Monro,Ch.6 把它讲清楚。

§2 核心定义(直觉先行)

  • MC ε-Greedy 算法 (Algorithm 5.3,p. 91):与 MC ES(Alg. 5.2)结构同,但 improvement 步用 (5.5) 在 \(\Pi_\varepsilon\) 内解。关键:不再要求 exploring starts —— "any starting (s_0, a_0)" 即可,因为 ε-greedy 让所有 (s, a) 自然被访问。
  • "ε-optimal" vs "true optimal"(教材 p. 91 末):Algorithm 5.3 收敛到 \(\pi^*_\varepsilon \in \Pi_\varepsilon\),即ε-greedy 子集内的最优。\(\pi^*_\varepsilon\) 一般不等于 \(\pi^* \in \Pi\)(全策略空间最优),除非 ε=0。
  • Consistent ε-greedy 策略(§5.5):两个 ε-greedy 策略"一致" 当且仅当它们指定的 greedy action 完全相同。Figure 5.6 各子图都是同一 greedy action,只换 ε ⇒ 它们 consistent;但 v 值差异巨大。
  • Figure 5.6 数值表(教材 p. 93):同 \(\pi^*\) 的 greedy action 分布,ε=0 时 v(target) = 10;ε=0.1 时 v(target) = 3.7;ε=0.2 时 v(target) = -2.0;ε=0.5 时 v(target) = -17.0。注意 ε=0.5 时所有 v 都是负的!
  • Figure 5.7 优化的 ε-greedy(p. 94):在 \(\Pi_\varepsilon\) 内重新解最优。ε=0.1 时与 \(\pi^*\) 一致;ε=0.2 时已不一致(target 处不再 stay,因为 ε 概率乱走会撞 forbidden);ε=0.5 时 target 处逃离(target 已成"陷阱",不如躲)。
  • Exploration ability(§5.5 末):ε=1 时百万步 episode visit 每个 (s, a) 大致 \(10^6 / 125 \approx 8000\) 次(方差小);ε=0.5 时分布极不均匀(target 附近 \(2.5 \times 10^5\) 次,远端不到 \(10^2\) 次)。
  • ε-decay 调度(§5.5 末段):实操常用"初期 ε 大、后期 ε 小"(如 ε: 1.0 → 0.05 over training)。前期强探索覆盖,后期收敛到准 greedy。Ch.7 Q-learning + Ch.8 DQN 默认搭配 ε-decay。

Step 1. Algorithm 5.3 与 5.2 的一行差异(p. 91)。两个算法的初始化、episode generation、倒推累加 g、q 更新都完全相同。唯一 差别在最后 improvement step:Algorithm 5.2 写"\(\pi(a \mid s_t) = 1\) if \(a = \arg\max_a q\) else 0";Algorithm 5.3 写"\(\pi(a \mid s_t) = 1 - \frac{|\mathcal A| - 1}{|\mathcal A|}\varepsilon\) if \(a = a^*\) else \(\varepsilon/|\mathcal A|\)"。改一行,exploring starts 消失,工程可用。
Step 2. "为什么 ε=0.5 让 v 全部变负"(p. 92 末段)。grid 设置 \(r_\text{forbidden} = -10\),ε=0.5 意味着每步 50% 概率"乱走" —— 在 forbidden 包围的 target 周围,乱走有显著概率撞 forbidden。target 处贪心 action 是"stay"(留在 target 持续拿 +1),但 ε 概率会触发"离开 target" 撞 forbidden,期望损失 \(\sim -5\) 每步。\(\gamma = 0.9\) 折扣下无穷 horizon 累积 \(\sim -50\),最终 v(target) = -17 是 +1 与 -50 的折衷。所以 ε 太大让"留在 target" 反而是坏选择。
Step 3. "为什么 Figure 5.7(c) target 处不再 stay"(p. 94)。Figure 5.7 是每个 ε 下重新解最优 ε-greedy。ε=0.1 时 \(\pi^*_\varepsilon\) 仍以 stay 为 target greedy action(损失小);ε=0.2 时已经接近临界,target 处可能改成"离开"(因为 stay + 20% 乱走 forbidden 的期望损失,大于"主动逃离" 的折扣损失);ε=0.5 时 target 处明确 逃离 —— "target 已成陷阱"。这是探索过强 的代价显化,提醒我们 ε 不能任意大。

§5 常见误解 / 易混概念

很多人以为:"MC ε-Greedy 收敛到 \(\pi^*\),所以等价 PI 但不依赖 model。"
实际上:MC ε-Greedy 收敛到 \(\pi^*_\varepsilon\) 而非 \(\pi^*\)(教材 p. 91 末段)。\(\pi^*_\varepsilon\) 是"\(\Pi_\varepsilon\) 内最优",通常严格次优于 \(\pi^*\)。要逼近 \(\pi^*\) 必须 ε → 0 —— 但 ε 太小让 q 估计不准 ⇒ 算法在"探索充分" 与"策略最优" 之间永远妥协。实操解:ε-decay 调度(\(\varepsilon_k = c / k\) 或指数衰减),前期大 ε 探索 + 后期小 ε 利用,这样最终策略近似 \(\pi^*\)。
因为:strict optimality 要求 \(\pi(a^* \mid s) = 1\)(deterministic),但 ε-greedy 强制 \(\pi(a^* \mid s) < 1\),所以 \(\pi^*_\varepsilon \ne \pi^*\) for \(\varepsilon > 0\)。Ch.10 Actor-Critic 用 stochastic policy + 直接梯度,不通过 ε-greedy 这条路,可逼近 \(\pi^*\) 而保留 stochasticity。
很多人以为:"ε 越大,探索越好,最终策略覆盖率 越高,所以 ε 选大些好。"
实际上:ε 太大让策略本身退化(Figure 5.6/5.7)。ε=0.5 时 \(v_{\pi^*_\varepsilon}(s) < 0\) 对所有 s,grid 上"留在 target" 的 v 反而最低 —— target 已成"陷阱"(ε 概率乱走撞 forbidden 受罚远超 +1 reward)。所以"覆盖率好" 与"策略好" 不是同向 —— 大 ε 帮估计但伤策略,小 ε 帮策略但探索不足。解决:ε-decay 让两阶段独立。
因为:"探索 vs 利用" 不是单维 trade-off,而是双目标:估准 q + 用准 q。ε 单一旋钮无法同时调好两个目标。Ch.7 Q-learning 还是用 ε-greedy + decay,Ch.8 DQN 增加 \(\epsilon: 1.0 \to 0.1\) 退火,Ch.10 SAC 用 entropy bonus 自适应调节,都是对这个问题的更精细回应。
很多人以为:"MC 是 model-free RL 的'终点',Ch.7 TD 是另起一支。"
实际上:MC 是"整条 episode 取样本" 的极端,TD 是"单步 bootstrap" 的另一极端(教材 §5.6 末段 + Ch.7 §7.1 预告)。两者夹中间是 n-step TD(Ch.7 L5)、TD(λ)(经典文献)。MC 优势:无 bootstrap bias;劣势:方差大、需要 episode 跑到底。TD 优势:方差小、可在线学习;劣势:bootstrap bias、需要 \(\hat v\) 初值合理。两者通过"n-step return" 桥接 —— Ch.7 把这个统一视图讲透。
因为:RL 的"多步信号 vs 单步信号" 是结构性 tradeoff。MC = 100% 样本(高方差低 bias),TD = 单步 + bootstrap(低方差高 bias),n-step 是连续光谱。Ch.7 把这个光谱完整展开,Ch.8 DQN / Ch.10 Actor-Critic 都用 n-step variant。

§6 自检 + 视频 + 下一章

Q1. Algorithm 5.3 与 Algorithm 5.2 的差别只有一行(improvement step)。具体差在哪里?

A. Algorithm 5.2(MC Exploring Starts)的 improvement:\(\pi(a \mid s_t) = 1\) if \(a = \arg\max_a q(s_t, a)\) else 0(纯 greedy)。Algorithm 5.3(MC ε-Greedy)的 improvement:\(\pi(a \mid s_t) = 1 - \frac{|\mathcal A(s_t)| - 1}{|\mathcal A(s_t)|}\varepsilon\) if \(a = \arg\max_a q(s_t, a)\),else \(\frac{\varepsilon}{|\mathcal A(s_t)|}\)(ε-greedy)。其他所有(初始化、episode generation、倒推累加 g、q update)都完全相同。这是为什么教材 Algorithm 5.3 在 5.2 上只增加"ε ∈ (0, 1]" 这一个新参数。

Q2. 教材 Figure 5.6 显示 ε=0.5 时 v(target) = -17。直观推算:为什么会这么负?

A. Target (3,2) 周围有四格 forbidden(2,1)(2,2)(1,3)(3,3) 中的若干。ε=0.5 时 target 处每步 有 50% 概率"乱走"(均匀抽 5 个动作中的 4 个非 stay)—— 其中 \(\sim 50\%\) 是邻接方向(↑/↓/←/→ 进 forbidden 或安全格)。粗算:每步乱走的期望 reward \(\sim 0.5 \cdot (-10) \cdot p_\text{forbidden 邻接} \approx -2\)。无穷 horizon 累积 \(-2 / (1 - 0.9) = -20\)。加上 stay 时的 +1 reward 贡献,综合 v(target) ≈ \(-17\)。结论:ε 太大让 target 处的 stay 反而是"持续踩 forbidden" 的同义词。这就是 §5.5 强调的"探索过强会破坏策略本身" 的具体数学。

Q3. 总结 Ch.5 三个算法(MC Basic / MC ES / MC ε-Greedy)的定位 与关系。

A. 三个算法构成连续光谱:MC Basic(Alg 5.1):"对每个 (s, a) 收集充分多 episode 估 q" + initial-visit + exploring starts。教学版,样本浪费严重。MC Exploring Starts(Alg 5.2):"一条 episode 内 every-visit + 倒推 + per-episode update" + 仍依赖 exploring starts。样本效率上一台阶,但 exploring starts 假设仍是死结。MC ε-Greedy(Alg 5.3):"5.2 + improvement step 在 \(\Pi_\varepsilon\) 内解" + 不再依赖 exploring starts。工程实用版,代价是收敛到 \(\pi^*_\varepsilon\) 而非 \(\pi^*\)。共同框架:三者都是 GPI(Ch.4 §4.4)的实例,PE 用样本均值替换 model-based 闭式,improvement 步在不同策略集合内解。共同短板:MC 必须等 episode 跑到底(完整 return),方差大、收敛慢。下一步:Ch.7 TD 用 bootstrap 把"等到底" 换成"走一步",大幅降方差,这是 RL 算法演化的下一台阶。

下一章预告

到这一节为止,Ch.5 完成:MC 三联(Basic / ES / ε-Greedy)把"用样本均值估期望" 这把钥匙系统嵌入 PI 框架,model-free RL 由此诞生。但 Ch.5 反复回避了一个核心问题:"样本均值 (1/n)\(\sum g\) 真的会收敛吗?多快?为什么是 (1/n) 这个权,不是其他?" 大数定律(L1 Box 5.1)给的是"n → ∞ 必然收敛",但有限步 的收敛速度、用一般步长 \(\alpha_k\)(而非 1/n)的收敛条件,都没解决。Ch.6 Stochastic Approximation 把这件事讲透:Robbins-Monro 算法 \(w_{k+1} = w_k - \alpha_k \tilde g(w_k, x_k)\) 给出一般 stochastic 估计的统一框架;Dvoretzky 收敛定理告诉我们 \(\sum \alpha_k = \infty\) 与 \(\sum \alpha_k^2 < \infty\) 这两条神奇条件如何保证收敛(\(\alpha_k = 1/k\) 恰好满足);SGD 是 RM 的特例。这把 Ch.5 的"样本均值收敛" 与 Ch.7 的 "TD 收敛" 都装进同一个数学框架。Ch.6 → Ch.7 的链条:RM 给数学,TD(0)、SARSA、Q-learning 都是 RM 在不同 Bellman 算子上的具体应用,收敛证明几乎免费。
