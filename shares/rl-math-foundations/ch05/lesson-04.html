L4: MC Exploring Starts (探索起点 MC + first-visit / every-visit)

§1 Why this lesson

L3 显示 MC Basic 能跑,但样本效率太低 —— 每个 \((s, a)\) 都要 collect "充分多" episode,共 \(|\mathcal S| \times |\mathcal A| \times n\) 条。本节(教材 §5.3,pp. 86-89)处理这个痛点的两面:(i) 把"n 条独立 episode" 重新组织为"1 条长 episode 上的多个子序列"(initial-visit / first-visit / every-visit 策略,§5.3.1);(ii) 把"每外步 collect 完所有 episode 才 update" 改成"每条 episode 都立刻 update"(§5.3.2)。两者合体得到 MC Exploring Starts(Algorithm 5.2,p. 88) —— MC 算法族里第一个工程实用版。但代价是依然依赖 exploring starts 假设,这正是 L5 / L6 ε-greedy 要解决的问题。

关键洞察(教材 §5.3 + Algorithm 5.2):(a) 一条 episode 包含多个 (s, a) 子序列(p. 86)。例如 episode \(s_1 \xrightarrow{a_2} s_2 \xrightarrow{a_4} s_1 \xrightarrow{a_2} s_2 \xrightarrow{a_3} s_5 \xrightarrow{a_1} \dots\) 内部包含 \((s_1, a_2), (s_2, a_4), (s_1, a_2)再次, (s_2, a_3), (s_5, a_1), \dots\) 多个 visit;每次 visit 后续都是合法的子 episode,可拿来估对应 \(q\)。(b) initial-visit (MC Basic) → first-visit → every-visit 三档:initial-visit 只用 \((s_0, a_0)\) 估一个 q,浪费 episode 内部信息;first-visit 用每个 \((s, a)\) 的第一次 visit;every-visit 用每一次 visit。"每升一档 sample efficiency 上升一档"。(c) Update 时机(§5.3.2):MC Basic "等所有 episode" → MC ES "每条 episode 立即 update" —— 后者属于 GPI(Ch.4 §4.4)框架下"PE 不必跑到精确" 的特例,接近 Ch.4 L3 的 truncated PI 思路。(d) Exploring starts 假设的代价(p. 89)。算法仍然 要求"从每个 (s, a) 起的 episode 充分多",在物理交互(机器人)、产品交互(推荐)等场景不可能满足。这是 L5 ε-greedy 的入口 —— "用 soft 策略 + 单条长 episode" 来等价"每个 (s, a) 都被探索"。

§2 核心定义(直觉先行)

  • Visit (访问):一条 episode 中,每个 \((s_t, a_t)\) 出现的位置叫一次 visit。一条长 episode 可能含同一 \((s, a)\) 的多次 visit。
  • Initial-visit 策略 (MC Basic 默认):只用 \((s_0, a_0)\) 那一个 visit 估 q,episode 中其他 visit 全部丢弃。样本浪费严重。
  • First-visit 策略:每个 \((s, a)\) 用第一次 出现位置之后的 sub-episode 估 q。每条 episode 可估多个 (s, a),但同一 (s, a) 只用一次。
  • Every-visit 策略:每个 \((s, a)\) 用每一次 出现位置之后的 sub-episode 估 q。样本效率最高,但同一 episode 内多次 visit 之间高度相关(后面 visit 是前面 visit 的 sub-trajectory) —— 不是严格 i.i.d.,但教材 p. 87 末段说"correlation would not be strong if the two visits are far away from each other"。
  • Update 时机:Per-episode update(§5.3.2):不等"充分多 episode",每条 episode 处理完立即 update q 与 \(\pi\)。这是 GPI 的"PE 不必精确" 思路在 MC 上的实例,与 Ch.4 L3 truncated PI 同源。
  • MC Exploring Starts (Algorithm 5.2,p. 88):MC Basic + every-visit + per-episode update + 后向 return 计算 \(g \leftarrow \gamma g + r_{t+1}\)(从 episode 末尾倒推到开头,效率 \(O(T)\) 而非朴素 \(O(T^2)\))。
  • Exploring Starts 条件 (p. 89):每个 \((s, a)\) 都要充分多次 被选作 episode 起点。这是强假设,真实环境常不可满足 —— 机器人不能"传送" 到任意状态再执行任意动作。L5 ε-greedy 用 soft 策略替换 这个条件,而不是假装 满足。

Step 1. Algorithm 5.2 骨架(p. 88)。For each episode:(i) select \((s_0, a_0)\) ensuring all (s, a) reachable(exploring starts 条件)。(ii) generate episode \(s_0, a_0, r_1, \dots, s_{T-1}, a_{T-1}, r_T\),长度 \(T\)。(iii) set \(g \gets 0\),倒推 \(t = T-1, \dots, 0\):\(g \gets \gamma g + r_{t+1}\),累加 Returns\((s_t, a_t) \mathrel{+}= g\),Num\((s_t, a_t) \mathrel{+}= 1\),\(q(s_t, a_t) = \text{Returns} / \text{Num}\),改 \(\pi\) 为 \(\arg\max_a q(s_t, a)\)。关键:每条 episode 处理完后立刻 update q 与 \(\pi\),不等"充分多"。
Step 2. "为什么 every-visit 是 sample efficient 上限"(p. 87 末段)。一条足够长 episode 在 soft 策略下能 visit 每个 \((s, a)\) 多次。every-visit 把每次 visit 都用上,所以"一条长 episode" 就够估所有 q,不必"多条独立 episode"。这把"多条独立 episode" 的需求近似 替换为"一条 soft 策略下的长 episode" —— 这正是 L5 ε-greedy 的核心想法。注意:every-visit 的样本不是严格独立(同 episode 内 visit 间相关),但教材直接说"correlation would not be strong if visits are far away",实操中可接受。
Step 3. "为什么仍依赖 exploring starts"(p. 89)。即使 every-visit + per-episode update,Algorithm 5.2 仍写"select a starting (s_0, a_0) and ensure all pairs can be possibly selected"。原因:若 \(\pi\) 是确定性 贪心策略,从 \((s, a)\) 起一旦 \(\pi\) 不选 \(a\),后续 visit 永远不到这个 \((s, a)\) —— q 估计永远空白。Exploring starts 强行 把 \((s, a)\) 当起点解决这点,但真实环境 agent 没有"任意传送 + 任意首动作" 能力。L5 用 soft 策略让 \(\pi(a \mid s) > 0\) 对所有 a,从而每个 (s, a) 自然 被访问,不再需要 exploring starts。

§5 常见误解 / 易混概念

很多人以为:"every-visit 的样本不独立,所以应该用 first-visit。"
实际上:two strategies 都收敛,但 sample efficiency 不同(教材 p. 87 末段)。every-visit 把同一 episode 多次 visit 都用上,样本量更大,但相邻 visit 高度相关。Sutton & Barto 经典证明 first-visit 是无偏 估计,every-visit 是有偏但渐进无偏 估计;实操中 every-visit 在有限 样本下方差更小、收敛更快。选哪个看场景:严格统计性质要 first-visit;追求 wall-clock 效率选 every-visit。本书 Algorithm 5.2 默认 every-visit,这是教材的工程偏好。
因为:"不严格独立" 不等于"不能用"。Ch.7 TD 用单步样本(更不独立)却收敛极快;Ch.8 DQN 用 experience replay buffer(把相关样本打散重抽)缓解相关性。所以"样本效率 vs 统计纯净" 是 RL 算法设计反复出现的 trade-off。
很多人以为:"exploring starts 是个'技术细节',真实问题里随便选个起点就好。"
实际上:Exploring starts 是强假设,在大多数真实环境不成立(教材 p. 89 直接强调)。机器人不能传送到任意 \((x, y, \theta)\) 再执行任意 \(a\);推荐系统不能"给同一用户假装是任意状态";自动驾驶不能"任意状态启动"。仿真器(Atari、MuJoCo)可以,所以 MC ES 在 RL benchmark 上能跑,但不能直接迁到真实物理 / 产品环境。这正是 L5 ε-greedy 的设计动机:用 soft 策略让 agent 自然 探索每个 \((s, a)\),而不依赖"任意起点"。
因为:RL 的"探索 vs 利用" 张力是结构性 的:agent 不知道哪些 \((s, a)\) 重要,要么"外部强制覆盖"(exploring starts)要么"自然采样覆盖"(soft policy 如 ε-greedy)。前者依赖环境配合,后者依赖策略设计。Ch.5 后两节走第二条路。
很多人以为:"per-episode update 让 q 估计很噪,应该等 N 条 episode 再 update。"
实际上:per-episode update 在 GPI 框架下仍收敛(教材 §5.3.2 末段)。原因:GPI 不要求 PE 跑到精确,只要"大致方向对" 即可改 \(\pi\)(Ch.4 §4.4)。每条 episode 后改 \(\pi\) 就像 truncated PI 在 \(j_\text{truncate} = 1\) 极限的 sample 版 —— "q 估计粗糙 + π 频繁改" 比"q 估计精确 + π 偶尔改" 反而更快。
因为:RL 的"速度 vs 精度" 总在权衡。MC Basic 站精度端,MC ES 站速度端。Ch.7 Q-learning 更激进 —— "每个 step 都 update q 与 π"(\(j = 1/T\),即 single-step truncated)。这是 GPI 的连续光谱在 MC/TD 上的具体实例。

§6 自检 + 视频 + 下一节

Q1. 一条 episode \(s_1 \xrightarrow{a_2} s_2 \xrightarrow{a_4} s_1 \xrightarrow{a_2} s_2 \xrightarrow{a_3} s_5 \xrightarrow{a_1} \dots\)(教材 p. 86)。first-visit 与 every-visit 在 \((s_1, a_2)\) 估计上分别用了哪些子序列?

A. First-visit:\((s_1, a_2)\) 第一次出现在位置 0(整条 episode 的起点),所以 first-visit 用整条 episode 的 \(g_0\) 估 \(q(s_1, a_2)\) 一次。第二次出现(位置 2)被丢弃。Every-visit:位置 0 用整条 \(g_0\) 估一次,位置 2 用从位置 2 起的 sub-episode \(g_2\) 估一次。两次都加入 \((s_1, a_2)\) 样本池,\(\hat q = (g_0 + g_2) / 2\)。直觉:every-visit 多榨出 1 个样本,代价是 \(g_0\) 与 \(g_2\) 不独立(\(g_0 = r_1 + \gamma r_2 + \gamma^2 g_2\),线性相关)。但教材 p. 87 末段说"相邻 visit 远时相关性弱",可接受。

Q2. Algorithm 5.2 的内层循环用了倒推 \(g \leftarrow \gamma g + r_{t+1}\)。如果改成正向"对每个 t,扫一遍 episode 算 \(g_t = r_{t+1} + \gamma r_{t+2} + \dots + \gamma^{T-t-1} r_T\)",复杂度变成多少?

A. 正向是 \(O(T^2)\),倒推是 \(O(T)\)。正向:对每个 \(t \in \{0, 1, \dots, T-1\}\) 算 \(g_t\) 都要扫 \(O(T)\) 个未来 reward,共 \(O(T^2)\)。倒推:维护标量 \(g\),从 \(t = T-1\) 起每步 \(O(1)\),共 \(O(T)\)。差距在长 episode(\(T = 10000\) 级)时是1 万倍 的工程差距。这就是 Algorithm 5.2 选倒推的原因。Ch.7 TD 把"整条 g 倒推" 进一步退化成"单步 g_t ≈ r_{t+1} + γ\(\hat v(s_{t+1})\)",连倒推都不需要。

Q3. Exploring starts 在哪些场景能 满足?哪些不能?

A. 能:仿真器(Atari、MuJoCo、AlphaGo 自对弈)、棋类游戏(可以从任意盘面开始)、有完整 reset 接口的游戏。不能:物理机器人(不能瞬移到任意 \((x, y, \theta)\) 再执行任意 a)、推荐系统(用户状态由真实历史决定,不能伪造)、自动驾驶(不能任意路况开始)、医疗 RL(病人状态不可伪造)。判定标准:能不能"任意 reset 到 (s, a) 起点"。这是为什么 model-based 仿真 + sim-to-real 是工业 RL 的主流路线 —— 仿真器满足 exploring starts,真实环境不满足。Ch.5 L5 的 ε-greedy 提供另一条路:不依赖 exploring starts,而靠 soft 策略让自然采样覆盖。

下一节预告

本节 MC Exploring Starts 仍然 假设 agent 能"任意起点" —— 真实环境的死结。下一节(L5)登场ε-greedy 策略:让 \(\pi(a \mid s) > 0\) 对所有 a(soft 策略),具体形式 \(\pi(a \mid s) = 1 - \frac{|\mathcal A| - 1}{|\mathcal A|}\varepsilon\) 给贪心动作,\(\frac{1}{|\mathcal A|}\varepsilon\) 给其他动作。这样一条充分长 episode 就能 visit 每个 \((s, a)\) 多次,无需 exploring starts。L5 重点是 ε-greedy 的分布形态 —— 用 distribution-bar 画出 greedy / ε=0.1 / ε=0.5 / uniform 的概率谱,直观看"从最贪到最探" 的连续过渡。L6 把 ε-greedy 嵌入 MC 算法(Algorithm 5.3)+ 全章总结。
