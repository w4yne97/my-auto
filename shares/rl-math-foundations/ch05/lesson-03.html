L3: MC Basic Walkthrough on Grid (MC Basic 示例与 episode-length 影响)

§1 Why this lesson

L2 给了 MC Basic 的算法骨架与公式 (5.2)。但"把 PE 替换成样本均值" 这件事在 grid 上跑起来到底什么样?本节(教材 §5.2.3,pp. 83-86)在两个具体例子上手算 —— (1) 教材 Figure 5.3 的 9-格 \(s_1\) 五动作 \(\hat q_{\pi_0}(s_1, \cdot)\) 估算与一步 improvement;(2) 教材 Figure 5.4 的 5×5 grid 上 episode-length \(L \in \{1, 2, 3, 4, 14, 15, 30, 100\}\) 系列实验,直观看"episode 太短 ⇒ 远端 state 学不到任何东西" 的"吸引场" 现象。

关键洞察(教材 §5.2.3):(a) 确定性策略 + 确定性环境 ⇒ 一条 episode 就够(p. 83 末段)。教材为节省篇幅,在 \(\pi_0\) 与 grid 都确定 的设置下,每个 \((s, a)\) 只跑一条 episode 估 \(\hat q\) —— 因为重跑会得到完全相同的 trajectory,样本均值 = 单样本。这不是 MC Basic 的一般行为,只是 deterministic 特例,但教学上能大幅简化手算。(b) 一步迭代到 \(\pi^*\)(p. 84)。Figure 5.3 的初始 \(\pi_0\) 仅在 \(s_1, s_3\) 上非最优。一外步后两 state 都被改对,新 \(\pi_1\) = \(\pi^*\)。这告诉我们 MC Basic 不弱于 PI 的"每步 improvement 单调有效"。(c) Episode-length 与 sparse reward 的耦合(教材 Figure 5.4,p. 85)。5×5 grid 上 \(L = 1\) 时只有 target 邻接格学到非零值;\(L = 2, 3, 4\) 时影响范围向外扩 1-2 格,呈"波纹" 形;\(L = 14\) 时左下角(到 target 至少 \(L_\min = 8\) 步,但因 forbidden 反射可能更长)开始非零,但仍未稳;\(L = 15\) 形成完整 \(v\) 估计;\(L = 30\) 已可学到 \(\pi^*\);\(L = 100\) value 也基本饱和。核心数学:从 state \(s\) 到 target 的 Manhattan 最短步数 \(L_\min(s)\),只要 \(L < L_\min\) 则 \(\hat G(s) = 0\)("这条 episode 还没到 target,所有 reward 都是 0" —— 注意 sparse reward 设置 \(r_\text{boundary} = -1, r_\text{forbidden} = -10, r_\text{target} = 1\))。(d) Sparse reward 的引子。这正是 RL 设计 dense reward / reward shaping 的根本动力 —— 否则 episode-length 对 \(L_\min\) 强依赖,大 grid 完全跑不动。

§2 核心定义(直觉先行)

  • Deterministic 简化(教材 §5.2.3 第一段末):若 \(\pi\) 与 \(p\) 都确定,则从 \((s, a)\) 起跑 \(n\) 条 episode 全相同。\(\hat q = g^{(1)}\),单样本即均值。教学例题用此,但不是 MC 一般情形。
  • Episode 长度 \(L\):每条 episode 跑多少步后被截断。教材 \(L\) 取 1, 2, 3, 4, 14, 15, 30, 100 系列演示效果。
  • 到 target 的最小步数 \(L_\min(s)\):从 \(s\) 出发最快 到 target 的步数(理想路径)。grid-world 4-动作下 \(L_\min(s) = |\Delta r| + |\Delta c|\)(Manhattan 距离),但 forbidden 阻挡时可能更大。
  • "L < L_min(s) ⇒ \(\hat G(s) = 0\)" 现象(p. 85):若 episode 在到达 target 前被截断,return 全部由 \(\{-1, -10, 0\}\) 系列拼成 —— sparse reward 下基本 = 0(若没踩 forbidden / 边界)。这是教材 Figure 5.4(a) "\(L = 1\) 时除 target 邻接外都是 0" 的原因。
  • "吸引场"(教材 p. 85 中段):随 \(L\) 增大,非零值从 target 向外扩散,呈现"越靠近 target 值越大" 的形态。这是 \(\gamma\)-discount 把"能否到 target" 转化成"多快到 target" 的几何效果。
  • 5×5 grid 的左下角 \(L_\min\) ≈ 8(教材 p. 86 顶):底-左 \((4, 0)\) 到 target \((3, 2)\) 的 Manhattan 距离 = \(|4-3| + |0-2| = 3\),但因 (2,1)(2,2)(3,3) 等 forbidden 阻挡,绕路至少 8 步(或踩 forbidden 受罚)。所以 \(L < 8\) 时左下角学不到任何东西。
  • Sparse reward 与 dense reward(教材 p. 86 末段):本例 \(r_\text{target} = 1\) 之外只有罚分(boundary = -1, forbidden = -10),其余 step 0,这是稀疏奖励。"无奖励信号" 让 MC 必须等 episode 跑到 target 才有 \(G > 0\)。解法:reward shaping(在 target 周围加小正奖励形成"吸引场"),或 dense reward 设计(每一步给 hint),或 hierarchical RL。

§3 关键公式与拆解

本节没有 新公式,而是把 L2 公式 (5.2) 在具体场景 落地。我们手算 Figure 5.3 例子 \(s_1\) 五个 action 的 \(\hat q_{\pi_0}\)。

公式回顾:MC Basic 的 q 估计(L2 公式 5.2)

手算:教材 §5.2.3 的 \(s_1\) 五个 action(γ = 0.9)。\(\pi_0(a_2 \mid s_2) = 1\)(\(s_2 \to\) ↓ 进 \(s_5\)),\(\pi_0(a_3 \mid s_5) = 1\)(\(s_5\) → 右往 \(s_6\) 或 target 方向),……(具体见教材 Figure 5.3)。

Step 1. \(a_1 = ↑\)(撞墙)。从 \(s_1\) 起走 ↑ 撞墙,\(r_1 = -1\),回到 \(s_1\)。\(\pi_0(s_1) = ↑\) ⇒ 继续撞墙,\(r_2 = -1\)。无穷循环 ⇒ \(g(s_1, a_1) = -1 - \gamma - \gamma^2 - \dots = -1/(1 - \gamma) = -10\)(\(\gamma = 0.9\))。
Step 2. \(a_2 = →\)(进 \(s_2\))。从 \(s_1\) 起走 → 进 \(s_2\),\(r_1 = 0\)。\(\pi_0(s_2) = ↓\) 进 \(s_5\),\(r_2 = 0\)。\(\pi_0(s_5)\) → \(s_8\) 等等,最终到 target \(s_9\)。教材给 \(g(s_1, a_2) = 0 + \gamma \cdot 0 + \gamma^2 \cdot 0 + \gamma^3 \cdot 1 + \gamma^4 \cdot 1 + \dots = \gamma^3 / (1-\gamma)\) ≈ 7.29(target 是 self-loop 吸收态)。
Step 3. \(a_3 = ↓\)(进 \(s_4\))。从 \(s_1\) 起走 ↓ 进 \(s_4\),\(r_1 = 0\)。\(\pi_0(s_4) = →\) 进 \(s_5\),后续与 Step 2 殊途同归 —— 也是 \(g(s_1, a_3) = \gamma^3 / (1-\gamma)\) ≈ 7.29。注意:\(a_2\) 与 \(a_3\) 在这例子里并列最优,improvement 步任选其一。
Step 4. \(a_4 = ←\)(撞左墙) 与 \(a_5 = stay\)。\(a_4\) 与 \(a_1\) 一样撞墙循环 ⇒ \(g \approx -10\)。\(a_5\) stay 在 \(s_1\),没罚也没赏,\(g = 0 + \gamma \cdot 0 + \dots\)? 不,\(\pi_0(s_1) = ↑\) 让下一步又撞墙,所以是 \(0 + \gamma(-1) + \gamma^2(-1) + \dots = -\gamma / (1-\gamma) = -9\)。所以 improvement 选 \(\arg\max_a \hat q = a_2\) 或 \(a_3\),把 \(\pi_1(s_1) = ↓\) 或 \(→\)(教材取 \(a_2 = →\)),与 \(\pi^*(s_1) = →\) 一致。

§4 Grid-world:Episode 长度的相变

用 5×5 grid(\(r_\text{boundary} = -1, r_\text{forbidden} = -10, r_\text{target} = 1, \gamma = 0.9\))演示 episode-length \(L\) 对 \(\hat v\) 与 \(\hat \pi\) 的影响。注意:这里展示的是最终(收敛后)的 \(v\) 与 \(\pi\) 估计,改的是 \(L\) 这个超参。

教材 Figure 5.4 的 5×5 grid。target = (3,2),forbidden 五格。下面 stepper 展示 \(L = 1, 4, 15, 30, 100\) 时的"非零值半径" 演化。

关键观察:Episode-length 与 sparse reward 是耦合 的痛点。(i) sparse reward ⇒ \(L < L_\min(s)\) 时 \(\hat v(s) = 0\);(ii) grid 越大、forbidden 越多 ⇒ \(L_\min\) 越大;(iii) MC 必须 \(L \ge \max_s L_\min(s)\) 才能让所有 state 学到。这正是 reward shaping 与 dense reward 设计的根本动力。

§5 常见误解 / 易混概念

很多人以为:"教材 §5.2.3 用 \(n = 1\)(单 episode) 估 \(\hat q\),那 MC Basic 一般也 \(n = 1\) 即可。"
实际上:\(n = 1\) 仅适用于确定性 π 与 \(p\)。教材明说"since this example is deterministic in terms of both the policy and model, running multiple times would generate the same trajectory"(p. 83)。一旦 \(\pi\) 是 stochastic 或 \(p\) 有随机性,从 \((s, a)\) 起的两次 trajectory 就不同 —— \(\hat q\) 是样本均值,n 必须充分多。MC Basic 的一般情形 n 要几十到几千。
因为:样本均值的方差是 \(\text{var}(G) / n\)。Deterministic 设置下 \(\text{var}(G) = 0\) ⇒ \(n = 1\) 就 0 误差。但这不 是 RL 一般情况。Q-learning(Ch.7)、Actor-Critic(Ch.10)默认 \(\pi\) stochastic,所以方差不为零,样本量要求严肃。
很多人以为:"教材 Figure 5.4 \(L = 1\) 时仅 target 邻接学到 1.0,这是 MC 算法失灵。"
实际上:这是 sparse reward + 短 episode 的必然结果,不是 MC bug。原因:\(L = 1\) ⇒ episode 只跑 1 步;若起点不邻接 target,这一步只能拿到 0(没踩边界 / forbidden 时)或 -1(撞墙)/ -10(踩 forbidden),都不能给 "到 target" 的正信号。解法:加大 \(L\) 让 episode 有机会到 target;或改 reward 设置让"朝 target 方向" 也有奖励(reward shaping)。
因为:RL 的"delayed reward" 困境:agent 在 \(t\) 步收到 reward,但功劳可能要追溯到 \(t - 100\) 步前的某个动作。MC 完整 return 解决"追溯" 但牺牲 episode-length。Ch.7 TD 用 bootstrap 把"多步追溯" 拆成"每步一小步",大幅减小 \(L\) 需求。
很多人以为:"L=15 与 L=30 \(\hat v\) 不一样,所以 MC 没收敛。"
实际上:π 已经收敛到 π*,但 \(\hat v\) 还没。教材 Figure 5.4(g) 注:"although the value estimate is not yet optimal"。原因:\(\pi^* \) 给定后,\(v_{\pi^*}(s) = \mathbb E_{\pi^*}[G_t \mid S_t = s]\) 本身依赖所有未来 step,truncated 到 \(L\) 步会少掉 \(\sum_{t > L} \gamma^t r_t\)。但 \(\arg\max_a q(s, a)\) 只关心相对大小,不需要 \(v\) 数值精确。所以 \(\pi^*\) 比 \(v^*\) 更早稳定 —— 这是 MC 与 PI 的共同特性,Ch.4 L1 §5 也讨论过。
因为:policy improvement 是个"argmax" 操作,只要 q 值的排序 对了,policy 就对。\(L\) 不够时 q 值绝对数偏小但相对排序仍可能正确,所以 \(\pi^*\) 优先收敛。这件事在 Q-learning 里再次出现 —— "q 估计噪" 不一定让"π 退化"。

§6 自检 + 视频 + 下一节

Q1. 在教材 §5.2.3 例子里(\(\gamma = 0.9\)),为什么 \(g(s_1, a_2) = \gamma^3 / (1 - \gamma)\) 而不是 \(\gamma^3\)?

A. 关键在 target 是吸收态(reaching target 后 self-loop 不断给 \(r = 1\))。从 \(s_1\) 起 \(a_2\) 进 \(s_2\)(\(r_1 = 0\)),\(s_2 \to s_5\)(\(r_2 = 0\)),\(s_5 \to s_8\)(\(r_3 = 0\)),\(s_8 \to s_9 = \text{target}\)(\(r_4 = 1\)),target self-loop \(r_5 = r_6 = \dots = 1\)。所以 \(g = 0 + \gamma \cdot 0 + \gamma^2 \cdot 0 + \gamma^3 \cdot 1 + \gamma^4 \cdot 1 + \dots = \gamma^3 (1 + \gamma + \gamma^2 + \dots) = \gamma^3 / (1 - \gamma)\)。如果 target 不是吸收态(到达后 episode 结束),则 \(g = \gamma^3\)。两者差一个 \(1/(1-\gamma)\) 倍数,但 improvement 的 argmax 不变(所有 q 都同比例缩放)。

Q2. 教材 Figure 5.4 显示 \(L = 14\) 时左下角 (4,0) 仍为 0,但 \(L = 15\) 时不再为 0。具体相变阈值是多少?

A. \(L_\min((4,0))\) = 15(从 (4,0) 绕开 forbidden 到 (3,2) 的最短"合法" 路径)。Manhattan 距离 = \(|4-3| + |0-2| = 3\),但 (2,1)(2,2)(3,3) 等 forbidden 阻挡,合法绕路至少 15 步。\(L = 14 < 15\) 时,从 (4,0) 起的 episode 没机会到 target,return 全靠 0/罚分构成 ⇒ \(\hat v((4,0)) = 0\)。\(L = 15\) 刚好够,所以 (4,0) 第一次有非零值。这就是教材 p. 86 顶"the episode length must be no less than 15" 的精确含义。

Q3. "Sparse reward 让 MC 跑不动" —— 实际工程里有哪些缓解方案?

A. 三类:(1) Reward shaping(教材 p. 86 末段):在 target 周围加小正奖励形成"吸引场",如 \(r(s) = 1 - \text{dist}(s, \text{target}) \cdot 0.1\)。(2) Curriculum learning:先在小 grid 训,再迁移到大 grid。(3) Hierarchical RL:把"到 target" 拆成"到 sub-goal 1, 2, ...",每个 sub-goal 给 dense reward。本质都是缩短"有效 sparse 距离"。Ch.7 TD 起则提供算法层缓解 —— bootstrap 让 reward 信号"每步往前传",不必等整条 episode。后两章(Ch.9 Policy Gradient、Ch.10 Actor-Critic)的 advantage / baseline 又进一步降方差。

下一节预告

本节展示 MC Basic 能跑,但样本效率低 —— 每个 \((s, a)\) 都要从指定起点 collect "充分多" episode。下一节(L4)处理这个痛点的两面:(i) Exploring Starts 假设 —— 强行从每个 \((s, a)\) 起跑,这是 MC Basic 与 MC ES 共同依赖的强假设,但在真实环境(机器人、推荐)常常不成立;(ii) first-visit / every-visit 策略 —— 一条 episode 内可同时 估多个 (s, a),把"n 条独立 episode" 摊开成"1 条 episode 的多个子序列",大幅榨干样本。MC Exploring Starts(Algorithm 5.2)= MC Basic + every-visit + 后向 return 计算 g ← γg + r_t,是 MC 算法族里第一个工程实用版本。
