L5: Expected Sarsa & n-step Sarsa (降方差与 bias-variance 全光谱)

§1 Why this lesson

Sarsa(L4)的 \(\delta_t\) 中"下一动作 \(a_{t+1}\) 是采样的" 引入动作随机性 的方差。本节(教材 §7.3)给 Sarsa 两个变种,各按一条轴降方差:(I) Expected Sarsa — 把"采样 \(a_{t+1}\)" 换成"当前 policy 期望" \(\sum_a \pi(a | s_{t+1}) Q(s_{t+1}, a)\),消除动作采样噪声;(II) n-step Sarsa — 往后多看 \(n\) 步真 reward 才 bootstrap,\(n=1\) 是 Sarsa、\(n=\infty\) 是 MC,中间是光谱。

核心 takeaway:(a) Expected Sarsa 仍是 on-policy(用当前 \(\pi\) 算期望),但比 Sarsa 方差小一档 ⇒ 同样 \(\alpha\) 下学得更稳。(b) \(n\)-step Sarsa 调 \(n\) ⇔ 调"bias vs variance":小 \(n\) 多 bootstrap 多 bias 少 variance;大 \(n\) 多真 reward 少 bias 多 variance。(c) 这两个变种是"RL 算法都是 \(\delta_t\) 中下一项怎么算的差别" 主旋律的强证据。

§2 核心定义(直觉先行)

  • Expected Sarsa target(教材 (7.18)):\(r_{t+1} + \gamma \sum_a \pi(a | s_{t+1}) Q(s_{t+1}, a)\)。把"下一动作 value" 用当前 policy 的期望 替换 sampled \(Q(s_{t+1}, a_{t+1})\)。
  • Expected Sarsa \(\delta_t\):\(\delta_t = r_{t+1} + \gamma \sum_a \pi(a | s_{t+1}) Q(s_{t+1}, a) - Q(s_t, a_t)\)。结构与 Sarsa 一字差,只是"sampled" 改"expected"。
  • 方差对比:Sarsa target 包含两个采样噪声 —— \(r_{t+1}\) 与 \(a_{t+1}\) 的随机性;Expected Sarsa 把 \(a_{t+1}\) 那一层解析地积掉,只剩 \(r_{t+1}\) 的噪声 ⇒ 方差小。代价:每步要算 \(\sum_a\)(代价 \(O(|\mathcal A|)\),小动作空间几乎免费)。
  • n-step Sarsa target(教材 (7.20)):\(G_t^{(n)} \doteq r_{t+1} + \gamma r_{t+2} + \ldots + \gamma^{n-1} r_{t+n} + \gamma^n Q(s_{t+n}, a_{t+n})\)。前 \(n\) 步用真 reward,第 \(n+1\) 步起用 bootstrap。
  • n-step Sarsa \(\delta_t^{(n)}\):\(\delta_t^{(n)} = G_t^{(n)} - Q(s_t, a_t)\)。\(n=1\) ⇒ Sarsa(δ_t);\(n \to \infty\) ⇒ MC(等 episode 完整 \(G_t\))。
  • "bias-variance 光谱":小 \(n\) ⇒ 多 bootstrap ⇒ \(Q\) 当前估计参与重 ⇒ bias 大;真 reward 少 ⇒ 方差小。大 \(n\) 反之。中间(\(n = 5\sim 10\))在很多任务上经验最优 —— deep RL 默认。

§3 关键公式与拆解

本节核心是"下一项怎么算" 的三种选择:Sarsa(sampled)、Expected Sarsa(expected)、\(n\)-step(往后多看几步)。

公式:Expected Sarsa 与 n-step Sarsa

Step 1. Expected Sarsa 仍是 on-policy 的关键证明。Expected Sarsa 用 \(\sum_a \pi(a | s_{t+1}) Q(s_{t+1}, a)\),期望按当前 \(\pi\) 取。若把 \(\pi\) 换成"greedy w.r.t. Q",Expected Sarsa 退化为 Q-learning(L6)—— 期望塌缩成单一 \(\max\)。所以"用什么 policy 算期望" 决定 on/off-policy 性质。Expected Sarsa = Sarsa 的方差降低版,Q-learning = Expected Sarsa 把 \(\pi\) 换成 \(\pi_*\)(greedy)的 off-policy 版。
Step 2. "n-step" 的代价:需要等 \(n\) 步样本到齐才能更新 \(Q(s_t, a_t)\) ⇒ 算法滞后 \(n\) 步。实操上用 ring buffer 暂存 \((s_{t-n+1:t}, a_{t-n+1:t}, r_{t-n+2:t+1})\),每收新样本时 pop 最老一档做更新。这是 deep RL "multi-step return + replay buffer" 的根。
Step 3. n-step 的 bias-variance 数值直觉。设 reward 噪声方差 \(\sigma_r^2\),bootstrap 估计偏差 \(b\)(\(|Q - q_\pi|\))。\(n\)-step target 方差 \(\approx n \sigma_r^2\),bias \(\approx \gamma^n b\)。总误差 = bias² + variance \(\approx \gamma^{2n} b^2 + n \sigma_r^2\)。对 \(n\) 求导取 0 给出最优 \(n^* \approx \log(2 b^2 |\ln \gamma|/\sigma_r^2)/(2|\ln \gamma|)\) —— 与"\(b\) 大 ⇒ \(n^*\) 大" "\(\sigma_r\) 大 ⇒ \(n^*\) 小" 直觉一致。

§5 常见误解 / 易混概念

很多人以为:"Expected Sarsa = 偷偷做 Q-learning,因为都不用真采样 \(a_{t+1}\)。"
实际上:Expected Sarsa 用当前 \(\pi\) 期望,Q-learning 用greedy max,二者差整个 RL 哲学。Expected Sarsa 期望按 \(\pi\) 取(包括探索动作),所以学到 \(q_\pi\),还是 on-policy。Q-learning 把期望塌缩成 \(\max_a\),即"假设之后总最优",学到 \(q_*\),off-policy。差别 = "评估当前 policy" vs "评估最优 policy"。
因为:"用 \(\pi\) 取期望" 与"用 max" 在数学上是不同算子:\(T_\pi^q\) vs \(T^*_q\)。前者收敛到 \(q_\pi\),后者收敛到 \(q^*\)。Expected Sarsa 与 Q-learning 共享 \(\sum_a\) 那一层结构,但期望权重不同 ⇒ 解的固定点不同。
很多人以为:"\(n\)-step 的 \(n\) 越大越好,因为更接近无 bias 的 MC。"
实际上:\(n\) 越大方差越大,\(n^*\) 是 trade-off 最优而非两端。Step 3 的方差/偏差估算给出"中间 \(n\) 常优"。任务 reward 噪声大、horizon 长,\(n^*\) 小;噪声小、horizon 短,\(n^*\) 大。没有"放之四海皆准" 的 \(n\),deep RL 实操常 \(n \in \{3, 5, 10\}\)。
因为:"无偏" 不代表"误差小"。\(\text{MSE} = \text{bias}^2 + \text{variance}\),variance 同样消耗误差预算。Ch.5 MC 的"纯无偏但方差大" 与 Ch.7 TD(0) 的"有偏但方差小" 之间,工程实操常胜的是中间。

§6 自检 + 视频 + 下一节

Q1. 写出 Expected Sarsa 与 \(n\)-step Sarsa 的 \(\delta_t\),并说明二者各从哪条轴降方差。

A. Expected Sarsa:\(\delta_t = r_{t+1} + \gamma \sum_a \pi(a | s_{t+1}) Q(s_{t+1}, a) - Q(s_t, a_t)\) —— 沿"动作随机" 这条轴消方差(把 sampled \(a_{t+1}\) 解析积掉)。\(n\)-step Sarsa:\(\delta_t^{(n)} = \sum_{i=0}^{n-1}\gamma^i r_{t+1+i} + \gamma^n Q(s_{t+n}, a_{t+n}) - Q(s_t, a_t)\) —— 沿"bootstrap vs 真 reward" 这条轴调比例,\(n\) 调 bias-variance trade-off。

Q2. Expected Sarsa 是 on-policy 还是 off-policy?为什么?

A. on-policy。期望 \(\sum_a \pi(a | s_{t+1}) Q(\cdot, a)\) 用当前 policy \(\pi\) 加权 ⇒ 学到的是当前 \(\pi\) 的 \(q_\pi\)。若把 \(\pi\) 换成"greedy w.r.t. Q"(\(\arg\max_a Q\) 上 mass=1),期望塌缩成 \(\max\) ⇒ 即是 Q-learning,off-policy。Expected Sarsa 与 Q-learning 共享"不要 sampled \(a_{t+1}\)" 的结构,差别在用什么 policy 取期望。

下一节预告

Sarsa 与 Expected Sarsa 都是 on-policy,学到的是当前 policy 的 \(q_\pi\)。L6(教材 §7.4.1-7.4.2) 引入 Q-learning:把"下一动作 value" 改用 \(\max_a Q(s_{t+1}, a)\) ——不论 behavior policy 是什么,target 总是"之后最优" 的假设。这一改让算法直接学 \(q^*\) 而非 \(q_\pi\),即 off-policy。Q-learning 是 deep RL 时代 DQN 的种子,L6/L7 详谈。
