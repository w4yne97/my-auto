L8: Unified Viewpoint & Chapter Summary (TD 家族一张表 + 通向 Ch.8)

§1 Why this lesson

L1-L7 把 Ch.7 的 4 个核心算法及其变种讲完。本节(教材 §7.5-7.7)做三件事:(1) 统一表达式:所有算法都是 \(Q \leftarrow Q + \alpha[\,\bar Q_t - Q\,]\),"target \(\bar Q_t\)" 的 4 种选择决定算法身份。(2) on/off-policy 系统对照:从 4 个轴(数据来源、target policy、收敛点、探索要求)整理。(3) Ch.8 桥接:把表式 \(Q\) 换成函数近似 \(\hat q(s,a; w)\),Q-learning 直接生 deep Q-network ⇒ DQN ⇒ deep RL 时代。

核心 takeaway:(a) Ch.7 整章"4 个算法,1 个公式" —— TD 家族在 \(\delta_t\) 中"下一动作 value 怎么算" 的 4 种选择。(b) on/off-policy 不是哲学,是结构:取决于 target policy 是否依赖 behavior。(c) Ch.8 把"表式 \(Q\)" 推广成"参数化 \(\hat q\)",update 公式不变(只是 \(\delta_t\) 推一个梯度),deadly triad 出现 —— 这是 deep RL 的新挑战 与新机会。

§2 核心定义(直觉先行)

  • 统一表达式(教材 §7.5):\(Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha_t [\bar Q_t - Q(s_t, a_t)]\),其中 \(\bar Q_t\) 是 target。4 种选择:(i) Sarsa \(\bar Q_t = r + \gamma Q(s', a')\)(sampled);(ii) Expected Sarsa \(\bar Q_t = r + \gamma \sum_a \pi(a|s') Q(s', a)\);(iii) Q-learning \(\bar Q_t = r + \gamma \max_a Q(s', a)\);(iv) n-step Sarsa \(\bar Q_t = G_t^{(n)}\)。
  • 4 个算法的固定点:Sarsa / Expected Sarsa / \(n\)-step Sarsa → \(q_\pi\)(on-policy);Q-learning → \(q^*\)(off-policy)。"target 用哪个算子" 决定固定点:\(T_\pi^q\) → \(q_\pi\),\(T^*\) → \(q^*\)。
  • on/off-policy 4 轴对照(本节 §3 表):(i) target policy(被估的 \(\pi\)):on=behavior,off=greedy(Q) 或固定;(ii) target 计算:on 用 \(\sum_a \pi\) 或 sampled,off 用 \(\max\);(iii) 数据复用:on 受 behavior change 限制,off 可重用;(iv) 探索要求:都需 behavior 充分覆盖。
  • Ch.6 RM 模板的统一收敛:全部 4 算法 = RM 在某 Bellman 算子上。固定点:\(T_\pi^q\) 或 \(T^*\);RM 三条件 + 每分量无穷访问 ⇒ 收敛。"归约 + 套定理" 母模板贯穿 Ch.7 全章。
  • Ch.8 桥接:把 \(Q(s,a)\) 表用 \(\hat q(s,a; w)\) 参数化(线性 FA 或 NN)。\(\delta_t\) 不变,update 改成 \(w \leftarrow w + \alpha \delta_t \nabla_w \hat q(s_t, a_t; w)\)(梯度一步)。结构形态相同,但带"函数近似 + bootstrap + off-policy" 的 deadly triad(Sutton-Barto)。
  • "第一族能 deploy" 的意义:Ch.4 DP 要 model;Ch.5 MC 要等 episode;Ch.7 TD 第一次给出"不要 model + 在线 + 每步更新" 的算法。这是RL 真正能跑起来 的转折点 —— 1990s 的 Q-learning,2010s 的 DQN,都是这一族的延续。

解读:同一样本预算下,Expected Sarsa 与 5-step Sarsa 比基线 Sarsa 快;Q-learning 与 Sarsa 同档(都是单步 bootstrap),但目标不同。"方差降低"(Expected Sarsa)与"bias-variance 中间档"(\(n\)-step)是 Ch.7 算法工程上的两条提速主线。Ch.8 把这条曲线推广到 NN 函数近似,deep RL 时代的核心曲线。

Step 1. 统一公式的"认知压缩" 价值。掌握 \(Q \leftarrow Q + \alpha[\bar Q - Q]\) 后,记 4 个 \(\bar Q\) 即记 4 个算法。后续 Sutton-Barto 的 SARSA(λ)、Watkins Q(λ)、tree-backup 都是"不同 \(\bar Q\) 的更复杂选择"。这是 RL 教科书"看似算法多,内核就 1 个" 的结构原因。
Step 2. on/off-policy 的结构判定法:看 \(\bar Q\) 是否引用当前 behavior policy。引用(Sarsa 的 sampled \(a'\)、Expected Sarsa 的 \(\pi\) 加权)⇒ on-policy。不引用(Q-learning 的 \(\max\),即 greedy(Q)) ⇒ off-policy。这条判定法在 Ch.8/9/10 的 actor-critic、policy gradient、TD3、SAC 中反复用。
Step 3. Ch.7 在全书中的位置。Ch.6 给"用样本估期望" 的抽象工具(RM/SGD)。Ch.7 把它具体化到 Bellman 算子,生出第一族真正能在线运行 的 RL 算法。Ch.8 把"表式 Q" 推广到"参数化 \(\hat q\)",处理大规模状态空间(图像、连续控制),deep RL 由此起。Ch.9-10 把"对 \(q\) 估" 改成"对 \(\pi\) 直接估"(policy gradient / actor-critic),处理连续动作空间。Ch.7 是经典 RL 与 deep RL 的枢纽。

§5 常见误解 / 易混概念

很多人以为:"Sarsa 学 \(q_\pi\) 是退化,Q-learning 学 \(q^*\) 才是正解。"
实际上:"学哪个" 取决于"用在哪里"。GPI 控制场景:Sarsa 在 \(\epsilon\)-greedy 探索下学 \(q_\pi\),通过 \(\epsilon \to 0\) GLIE 间接逼近 \(q^*\) ——渐近最优。Q-learning 直接学 \(q^*\),但实际 deploy 仍 \(\epsilon\)-greedy 时性能未必更优(L7 cliff-walking)。"policy evaluation vs policy improvement" 在 Ch.7 4 算法上有不同侧重。
因为:"评估当前 policy" 与"评估最优 policy" 是不同任务。GPI 让二者交替推进,4 个算法各有侧重。Sarsa = "evaluate then improve",Q-learning = "directly chase optimal"。哪个更优取决于探索-利用平衡与 deploy 场景。
很多人以为:"Ch.7 是经典 RL 终点,Ch.8 之后是另一回事。"
实际上:Ch.8 把 \(Q\) 表换成 \(\hat q(s,a; w)\) ——更新公式不变,只是 \(Q(s_t, a_t)\) 推一个梯度。Q-learning + NN \(\hat q\) = DQN(Mnih 2013/2015)。同一个 \(\delta_t\) 公式,从 \(Q\) 表表达到 NN 表达,从 5×5 grid 推广到 Atari 像素 ——结构延续。但 FA + bootstrap + off-policy 三联引入deadly triad,deep RL 训练不稳是新挑战(target network、experience replay 等是工程修复)。
因为:"新算法" 与"同一算法换表达" 不同。DQN 的骨架 仍是 Q-learning,创新在函数近似 + 工程稳定化。读懂 Ch.7 是读懂 deep RL 的结构前提;Ch.8 是同一思想在更大规模上的落地。

§6 自检 + 视频 + 下一章

Q1. 复述 TD 家族统一公式与 4 种 \(\bar Q\) 选择。

A. \(Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha[\bar Q_t - Q(s_t, a_t)]\)。(i) Sarsa:\(\bar Q = r + \gamma Q(s', a'_{\sim\pi})\) — sampled。(ii) Expected Sarsa:\(\bar Q = r + \gamma \sum_a \pi(a|s') Q(s', a)\) — \(\pi\) 期望。(iii) Q-learning:\(\bar Q = r + \gamma \max_a Q(s', a)\) — greedy max。(iv) \(n\)-step:\(\bar Q = G_t^{(n)} = \sum_{i=0}^{n-1}\gamma^i r_{t+1+i} + \gamma^n Q(s_{t+n}, a_{t+n})\) — 多步混合。4 选 1,即 4 个算法。

Q2. on/off-policy 的结构 判定法是什么?分别给一例。

A. 看 target \(\bar Q_t\) 是否引用当前 behavior policy。引用 ⇒ on-policy(Sarsa: sampled \(a'\) 来自当前 \(\pi\))。不引用 ⇒ off-policy(Q-learning: \(\max_a\) 与 \(\pi\) 无关,只看 \(Q\))。判定不依赖具体算法名,而依赖"下一动作 value 的来源"。

Q3. Ch.7 与 Ch.8 的结构 关系是什么?

A. Ch.7 用表式 \(Q(s,a)\)(每 \((s,a)\) 一个独立数);Ch.8 用函数近似 \(\hat q(s,a; w)\)(共享参数 \(w\))。同一 \(\delta_t\) 公式,update 从 \(Q(s_t, a_t) \leftarrow Q + \alpha \delta\) 改成 \(w \leftarrow w + \alpha \delta \nabla_w \hat q\)(梯度一步)。骨架不变,表达扩展。Ch.8 推到深度网络 ⇒ DQN ⇒ Atari ⇒ deep RL 时代。Ch.7 是"表式 RL 顶点" + "deep RL 起点"。
