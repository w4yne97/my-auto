L1: TD Motivating Example (TD 动机:从均值估计跨到 Bellman 估计)

§1 Why this lesson

Ch.6 L7 收尾时把 RM/SGD 的尾巴交给 Ch.7:"RM 给数学,Ch.7 把它套到 Bellman 算子的固定点上,生出第一族真正能在线运行的 RL 算法族。" 本节(教材 §7.1.1)做三件事:(1) 用三个递推 把"从均值估计 → 估 \(\mathbb E[R]\) → 估 \(\mathbb E[R + \gamma V(s')]\)" 这一阶梯铺给读者看;(2) 引入动机问题:为什么 MC 已经能估期望,我们还要 TD?(3) 直觉地说出 TD 的关键差别:不等 episode 结束,只用一步 数据 \((s_t, r_{t+1}, s_{t+1})\) 就能改 \(V(s_t)\)。

核心 takeaway:TD 是带 bootstrap 的均值估计。Ch.5 MC 用 \(G_t\)(整条 return)估 \(v_\pi(s_t)\),要等到 episode 结束;TD 用一步样本 \(r_{t+1}\) 加上对未来的当前估计 \(\gamma V(s_{t+1})\) 当 target,每一步就能更新。"把估计的尾巴接到自己的估计上" 即 bootstrap,正是 TD 与 MC 的结构差异,也是 §7.1.2 引入 TD error \(\delta_t\) 的根。

§2 核心定义(直觉先行)

  • 问题 1:估 \(\mathbb E[X]\)(教材 §7.1.1 (7.1))。Ch.6 L1 给的递推 \(w_{k+1} = w_k - \alpha_k(w_k - x_k)\):每来一个 \(x_k\) 就一步更新。RM 框架下收敛到 \(\mathbb E[X]\)。
  • 问题 2:估 \(\mathbb E[v(X)]\)(教材 (7.2))。把 \(x_k\) 替成 \(v(x_k)\):\(w_{k+1} = w_k - \alpha_k(w_k - v(x_k))\)。同样 RM 思路,只是被估对象更复杂(把"样本" 改成"样本的函数")。
  • 问题 3:估 \(\mathbb E[R + \gamma v(X')]\)(教材 (7.3))。target 含另一个状态的当前估计 \(v(X')\) ⇒ 即bootstrap。这是从"独立样本均值" 跳到"Bellman 风格的固定点估计" 的关键一跨。
  • 关键差别:bootstrap (自举)。MC target = 真采样的 \(G_t\)(无 bias);TD target = \(r_{t+1} + \gamma V(s_{t+1})\),其中 \(V(s_{t+1})\) 是当前估计(有 bias,但方差小)。
  • "online" 含义。MC 在 episode 末才一次性算所有 \(G_t\);TD 在每一步 立刻改 \(V(s_t)\)。memory 与延迟都更友好,且适配持续型任务(无明确 episode 终点)。
  • Ch.7 整章主线(预告)。L2 给出 TD(0) 与 \(\delta_t\);L3 用 RM 证收敛;L4 把 TD 推广到 \(q\) → Sarsa;L5 加 \(n\) 步与 expected 变种;L6-L7 把 sampled 改 max → Q-learning;L8 总结统一视角。

§3 关键公式与拆解

本节核心是三个递推阶梯:从估 \(\mathbb E[X]\)、到估 \(\mathbb E[v(X)]\)、再到估 \(\mathbb E[R + \gamma v(X')]\)。第三个就是 TD 的种子。

公式:三阶梯递推(教材 (7.1)-(7.3))

Step 1. 为什么 (7.3) 是跨越,不是顺势。(7.1)、(7.2) 中 \(x_k\) 是真采样,无偏。(7.3) 中 target 含 \(v(x_{k+1})\) —— 它是当前对 \(x_{k+1}\) 的估计,本身就有误差。把"带误差的估计" 当 target 来更新,在朴素直觉看似乎"错误传播",但 RM 理论会保证:只要 \(\alpha_k\) 满足两条件,\(V\) 仍能收敛到真 \(v_\pi\)。L3 给严证。
Step 2. "用 (7.3) 估 \(v_\pi(s)\)" 的对应。Bellman 期望方程 \(v_\pi(s) = \mathbb E_\pi[R_{t+1} + \gamma v_\pi(S_{t+1}) \mid S_t = s]\)。所以"估 \(\mathbb E[R + \gamma v(X')]\) 在 \(s\) 处的值"恰好 是估 \(v_\pi(s)\)。把 (7.3) 用在 \(s_t\) 处,target 即是 Bellman 右端的一次单样本估计 —— 这就是 TD(0)。L2 把它写出。
Step 3. MC vs TD 的第一对比。MC:用 \(G_t = R_{t+1} + \gamma R_{t+2} + \ldots + \gamma^{T-t-1} R_T\) 当 target,需要等 episode 末。TD:用 \(R_{t+1} + \gamma V(s_{t+1})\) 当 target,只看一步。MC target 无 bias、方差大;TD target 有 bias(因 \(V\) 不准)、方差小。这条 bias-variance trade-off 是 Ch.7 全章的主旋律,L5 \(n\)-step 把这条轴上每一点都给出来。

§4 Grid-world 例子

用作者的 5×5 网格直观理解"等到 episode 结束 vs 一步更新"。target=(3,2),forbidden={(1,1),(2,1),(2,2),(1,3),(3,3)}。设当前 episode 从 (0,0) 出发沿 policy 走到 target,共 5 步:

MC 视角:走完 5 步、拿到 \(R_1, \ldots, R_5\) 后,才一次性算 \(G_0 = R_1 + \gamma R_2 + \ldots + \gamma^4 R_5\),然后更新 \(V(s_0)\)。TD 视角:第 1 步走完(到 (1,0))就能用 \(R_1 + \gamma V((1,0))\) 当 target,立刻更新 \(V((0,0))\);第 2 步类似改 \(V((1,0))\);整条轨迹走完时,5 个状态各自被改了一次。"不等 episode" 意味着 5 次更新而非 1 次,信息利用率高。

§5 常见误解 / 易混概念

很多人以为:"既然 MC 无 bias、TD 有 bias,MC 应该总是更优。"
实际上:方差才是 RL 的真敌人。MC 的 \(G_t\) 是整条轨迹的累加,方差随 episode 长度爆炸。TD 用 \(r + \gamma V(s')\) 把未来的不确定性 压缩成一步 reward + 当前估计,方差小很多。在长 horizon 任务里 TD 几乎总比 MC 收敛快。
因为:"无偏 + 高方差" 与"低偏 + 低方差" 是经典 trade-off。RL 的目标不是"消 bias" 而是"消 总误差"。L5 \(n\)-step 把"一步 → 整条" 中间所有可能都给你 ——选哪一档 才是工程艺术。
很多人以为:"TD 的 bootstrap 是错误循环:用错的估计去校准错的估计。"
实际上:RM 理论保证 bootstrap 仍能收敛到真值。关键在 (7.3) 在期望意义下是对 \(v_\pi(s)\) 的无偏 RM 步(条件期望取出 \(\mathbb E[R + \gamma v(X')] = v_\pi(s)\)),只要 \(\alpha_k\) 满足两条件、\(V\) 初始值有界,长期能收敛。L3 详细给。
因为:"每步 \(V\) 不准" 不等于"更新平均后不准"。RM 的核心思想是用噪声样本沿"正确方向" 持续推动,长期平均后误差消失 —— bootstrap 只是把"正确方向" 定义到了"当前估计与下一步估计的差" 上,这正是 \(\delta_t\) 的几何含义。

§6 自检 + 视频 + 下一节

Q1. 把 (7.3) 的 target 改写成 \(R + \gamma v(X')\),为什么这就是Bellman 方程的右端?

A. Bellman 期望方程:\(v_\pi(s) = \mathbb E_\pi[R_{t+1} + \gamma v_\pi(S_{t+1}) \mid S_t = s]\)。(7.3) 的 target 期望就是右端:\(\mathbb E[R + \gamma v(X')]\)。所以"估 (7.3) 在 \(s\) 处的值" = "估 \(v_\pi(s)\)"。TD(0) 即把 (7.3) 部署到 \(s_t\) 处:每一步把 target 与当前估计的差乘 \(\alpha\) 加到 \(V(s_t)\)。这就是 L2 \(\delta_t\) 公式的直接推导。

Q2. MC 与 TD 都在估 \(v_\pi\),它们的更新时机 与target 形态 各是什么?

A. MC:更新时机 在 episode 末;target 是 \(G_t\)(从 \(t\) 起的真实累计 reward)。无 bias,方差大。TD(0):更新时机 在每个 \((s_t, r_{t+1}, s_{t+1})\) 之后立刻;target 是 \(r_{t+1} + \gamma V(s_{t+1})\)(一步 reward + 下一状态当前估计)。有 bias(因 \(V\) 不准),方差小。核心结构差异:bootstrap。
