L5: Bellman Equation (P5 - Action value) (贝尔曼方程(P5):动作值)

§1 Why this lesson

到上一节(L4)为止,我们已经能评估状态:给一个策略 \(\pi\),写出 25 条贝尔曼方程,矩阵化求解,得到每个状态的 \(v_\pi(s)\)。但 RL 真正想做的事情是评估动作 —— 在某个状态 \(s\) 下,"向右"是不是比"向下"更好?\(v_\pi(s)\) 只能告诉你"这个状态值多少分",不能告诉你"在这个状态下哪个动作最值"。

本节(教材 §2.8-2.10,pp. 30-34)引入动作值 (action value) \(q_\pi(s, a) \doteq \mathbb{E}_\pi[G_t \mid S_t = s, A_t = a]\):"在 \(s\) 下强行执行 \(a\)、之后再按 \(\pi\) 走"的折现 return 期望。这看似只是在 \(v_\pi\) 的条件里多塞一个 \(A_t = a\),但意义大不一样:有了 \(q_\pi\),我们可以在每个状态枚举所有动作、挑出最大的 —— 这就是 Ch.3 BOE 里 \(\pi^*(s) = \arg\max_a q^*(s, a)\) 的雏形,也是 Q-learning(Ch.7)、DQN(Ch.8)的核心对象。本节做三件事:(1) 给定义;(2) 推 \(v_\pi \leftrightarrow q_\pi\) 的双向关系;(3) 写出动作值版本的贝尔曼方程,为 Ch.3 起的"基于 \(q\) 的算法"铺路。

§2 核心定义(直觉先行)

  • 动作值 (action value, state-action value) \(q_\pi(s, a)\)(教材 §2.8 公式定义,p. 30):\(q_\pi(s, a) \doteq \mathbb{E}_\pi[G_t \mid S_t = s, A_t = a]\)。一句话:从 \(s\) 出发、第一步强制做 \(a\)、之后按 \(\pi\) 走,折现 return 的期望。注意:作者在 p. 30 末尾说"More rigorously it should be called state-action value",简称 action value 是历史习惯。
  • v_π 与 q_π 的关系 1(教材 §2.8 公式 (2.13),p. 30):\(v_\pi(s) = \sum_a \pi(a \mid s) \, q_\pi(s, a)\)。直觉:状态值 = 在该状态下,所有动作值按策略分布加权平均。换言之,\(v_\pi\) 是 \(q_\pi\) 沿动作维的"折叠"。
  • v_π 与 q_π 的关系 2(教材 §2.8 公式 (2.14),p. 31):\(q_\pi(s, a) = \sum_r p(r \mid s, a) \, r + \gamma \sum_{s'} p(s' \mid s, a) \, v_\pi(s')\)。直觉:动作值 = 在 \((s, a)\) 给定后的即时 reward 期望 + γ × 下一状态状态值的期望。这是 L3 公式 (2.7) 方括号里那一坨,直接被命名为 \(q_\pi(s, a)\)。所以 (2.7) 改写为 \(v_\pi(s) = \sum_a \pi(a \mid s) q_\pi(s, a)\) 就是 (2.13)。
  • "两面同硬币"(教材 §2.8 末段):(2.13) 是"从 q 算 v"((π 加权折叠 q)),(2.14) 是"从 v 算 q"(reward 期望 + γ × 下一状态 v 期望)。两条等式正好互逆,任意一边知道,另一边一步出。
  • 动作值版的贝尔曼方程(教材 §2.8.2 公式 (2.15) 上方):把 (2.13) 代入 (2.14),消去 \(v_\pi\),得 \(q_\pi(s, a) = \sum_r p(r\mid s, a) r + \gamma \sum_{s'} p(s'\mid s, a) \sum_{a'} \pi(a' \mid s') q_\pi(s', a')\)。这就是后面 Q-learning 直接迭代的方程,目标都是 \(q_\pi\) 而不是 \(v_\pi\)。
  • "未被策略选中的动作也有 q 值"(教材 §2.8.1 中段,p. 31-32):即使 \(\pi(a \mid s) = 0\)(策略不选 \(a\)),\(q_\pi(s, a)\) 仍然是良好定义的 —— 它问的是"假设强制做了 \(a\),后面再按 \(\pi\) 走"的期望。这一点对 Ch.3 BOE 至关重要:要找出比当前 \(\pi\) 更好的动作,必须先能评估当前 \(\pi\) 不选的那些动作。

Step 1. (2.13) 是怎么从定义出发的?从 \(v_\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]\) 出发,按 \(A_t\) 用全期望公式分层:\(\mathbb{E}_\pi[G_t \mid S_t = s] = \sum_a \pi(a \mid s) \, \mathbb{E}_\pi[G_t \mid S_t = s, A_t = a]\)。后者正是 \(q_\pi(s, a)\) 的定义。一行收工 —— 全期望公式而已。
Step 2. (2.14) 是怎么从定义出发的?把 \(q_\pi(s, a) = \mathbb{E}_\pi[G_t \mid S_t = s, A_t = a]\) 中的 \(G_t = R_{t+1} + \gamma G_{t+1}\) 代入,期望线性 \(\mathbb{E}[R_{t+1} \mid S_t=s, A_t=a] + \gamma \mathbb{E}[G_{t+1} \mid S_t=s, A_t=a]\)。第一项 = \(\sum_r p(r\mid s, a) r\)(reward 分布平均);第二项用 Markov 性 \(\mathbb{E}[G_{t+1} \mid S_t=s, A_t=a, S_{t+1}=s'] = \mathbb{E}[G_{t+1} \mid S_{t+1}=s'] = v_\pi(s')\),再按 \(S_{t+1}\) 分层得 \(\sum_{s'} p(s'\mid s, a) v_\pi(s')\)。合起来即 (2.14)。注意:这里不需要再对 \(a'\) 求和,因为第一步动作 \(a\) 已经给定,只有"之后"按 \(\pi\) 才有 \(\pi(a'\mid s')\) 的随机性 —— 那部分被打包进 \(v_\pi(s')\) 里了。

§4 Grid-world 实例化

用教材 §2.8.1 的 2×2 例(图 2.8,与 L3 §4 例 2 / L4 §4 例同一张),\(s_1\) 上策略 \(\pi(a_2\mid s_1) = \pi(a_3\mid s_1) = 0.5\),\(\gamma = 0.9\)。我们已知 \(v_\pi(s_1) = 8.5\)、\(v_\pi(s_2) = v_\pi(s_3) = v_\pi(s_4) = 10\)(L4 §4 算过)。把 \(s_1\) 的所有 5 个动作的 \(q_\pi(s_1, \cdot)\) 全算出来,看哪个动作最值:

  • \(q_\pi(s_1, a_1)\) (向上,撞墙弹回 \(s_1\),\(r_\text{boundary} = -1\)) = \(-1 + \gamma v_\pi(s_1) = -1 + 0.9 \cdot 8.5 = 6.65\)
  • \(q_\pi(s_1, a_2)\) (向右,撞 forbidden \(s_2\),\(r = -1\)) = \(-1 + \gamma v_\pi(s_2) = -1 + 0.9 \cdot 10 = 8\)
  • \(q_\pi(s_1, a_3)\) (向下,走 \(s_3\),\(r = 0\)) = \(0 + \gamma v_\pi(s_3) = 0 + 0.9 \cdot 10 = 9\)
  • \(q_\pi(s_1, a_4)\) (向左,撞墙弹回 \(s_1\),\(r = -1\)) = \(-1 + \gamma v_\pi(s_1) = 6.65\)
  • \(q_\pi(s_1, a_5)\) (停留,\(r = 0\),原地) = \(0 + \gamma v_\pi(s_1) = 0.9 \cdot 8.5 = 7.65\)

验证 (2.13):\(v_\pi(s_1) = 0.5 \cdot q_\pi(s_1, a_2) + 0.5 \cdot q_\pi(s_1, a_3) = 0.5 \cdot 8 + 0.5 \cdot 9 = 8.5\) ✓ 与 L4 算出的 \(v_\pi(s_1) = 8.5\) 完全对得上。

从 q 表读"最优动作"。在 \(s_1\) 上,\(\arg\max_a q_\pi(s_1, a) = a_3\)(向下,\(q = 9\))—— 比当前策略 \(\pi\) 的随机加权平均(8.5)还要好。这是 Ch.3 BOE 故事的开端:你只要扫一遍当前 \(\pi\) 下的 \(q_\pi\) 表,就能找到"局部能改进的动作",不断改进就走向 \(\pi^*\)。这正是 Ch.4 Policy Iteration 的内核。

把 q 想成 5×5×|𝒜| 的"加厚热图"。下图是 Ch.2 §4 一直在用的 5×5 状态值热图(\(v_\pi\));action value 把它"加厚"成 5×5×5 的张量 —— 每格上不再是一个数,而是 5 个数(对应 5 个动作)。本节没有显式画出 \(q_\pi\) 的张量,但你可以把它想象成"在每格上叠一个长度为 5 的小条形图(distribution-bar 形式)",条形高度 = 该动作的 \(q\) 值。Ch.7 起 DQN / Q-learning 直接学这个张量,跳过 \(v_\pi\) 这一中间层。

§5 常见误解 / 易混概念

很多人以为:"既然 \(\pi\) 不选 \(a_1, a_4, a_5\),它们的 \(q_\pi(s_1, \cdot)\) 干脆设为 0 或者 \(-\infty\) 就行,免得算多余的。"
实际上:这是教材 p. 31-32 专门点名的"common mistake"。"未被选中"≠"value 是 0":\(q_\pi(s, a)\) 问的是"如果我偏要做 \(a\),后面再按 \(\pi\) 走,期望回报多少?"——这跟 \(\pi\) 选不选 \(a\) 完全无关。例如本节 \(q_\pi(s_1, a_5) = 7.65\) 是个非零数,虽然 \(\pi(a_5 \mid s_1) = 0\)。
因为:找最优策略的整个动机就是"看那些当前策略不选的动作里有没有更好的"。如果你把它们全设 0,你就永远找不到改进方向 —— 这正是 Ch.3 BOE 与 Ch.4 PI 的核心运作机制(policy improvement:\(\pi'(s) = \arg\max_a q_\pi(s, a)\),要 \(\arg\max\) 的话,所有 \(a\) 的 \(q\) 都得有定义)。
很多人以为:"\(v_\pi(s) = \max_a q_\pi(s, a)\) —— 状态值就是最优动作的动作值。"
实际上:这是 \(v^*\) 与 \(q^*\) 之间的关系(下章 BOE),不是 \(v_\pi\) 与 \(q_\pi\) 的关系!对一般策略 \(\pi\),(2.13) 是加权平均 \(v_\pi(s) = \sum_a \pi(a\mid s) q_\pi(s, a)\),而不是 max。本节 §4 的 \(v_\pi(s_1) = 8.5\) 但 \(\max_a q_\pi(s_1, a) = 9\) 就是反例。
因为:"max" 只在最优策略上成立(因为最优策略一定贪心选最值的动作);任意策略下 max 给出的是"还可以更好"的暗示,不是当前 \(v_\pi\)。Ch.3 第一节就用这条 max 关系定义最优策略,本节先警告别提前混淆。
很多人以为:"既然 \(q\) 比 \(v\) 信息多,直接学 \(q\) 不就行了,何必先学 \(v\)?"
实际上:这正是 Ch.7 Q-learning / Ch.8 DQN 的取舍。代价是 \(q\) 的存储是 \(|\mathcal{S}| \cdot |\mathcal{A}|\)(本课 25 × 5 = 125),比 \(v\) 的 \(|\mathcal{S}| = 25\) 大 5 倍;一旦动作空间连续(机器人控制)就更糟。所以本章先用 \(v\) 把 Bellman / 矩阵 / 求解的骨架讲清,再到 Ch.7 把同一套搬到 \(q\) 上。
因为:"Q-learning vs Value Iteration"的优劣并不绝对,取决于动作空间大小、是否需要 model-free、是否能直接 \(\arg\max_a\)。本课的策略是:先学完 v 的世界、再学完 q 的世界、最后比较两者的算法栈(Ch.4 vs Ch.7)。

§6 自检 + 视频 + 下一节

Q1. 在 §4 的设定下,把 \(s_1\) 的策略改成确定性 \(\pi(a_3 \mid s_1) = 1\)(全压向下)。(a) 重算 \(v_\pi(s_1)\);(b) 重算 \(q_\pi(s_1, a_2)\)(原本不被选中的"向右"动作)。

A. (a) 现在 \(s_1\) 必走向下、reward = 0、转 \(s_3\),\(v_\pi(s_1) = 0 + \gamma v_\pi(s_3) = 0.9 \cdot 10 = 9\)。注意:这个 \(v_\pi\) 是新策略下的状态值,因为换了策略 \(\pi\) 就换了一个 \(v_\pi\) 函数 —— 这是 (R2) 的具体提醒。(b) \(q_\pi(s_1, a_2) = -1 + \gamma v_\pi(s_2) = -1 + 0.9 \cdot 10 = 8\)。关键观察:即使 \(\pi\) 永远不选 \(a_2\),\(q_\pi(s_1, a_2) = 8\) 仍然有意义 —— 它告诉你"假如你强行选了 \(a_2\),后果是 8 分(比当前 9 分差)";这就证实了当前确定性策略 \(a_3\) 比 \(a_2\) 更好,至少局部如此。

Q2. 用 (2.14) 验证 §4 中 \(q_\pi(s_3, a_2) = ?\)(\(s_3\) 上"向右"=到 target、reward = +1)。

A. \(q_\pi(s_3, a_2) = \sum_r p(r \mid s_3, a_2) r + \gamma \sum_{s'} p(s' \mid s_3, a_2) v_\pi(s') = 1 + 0.9 \cdot v_\pi(s_4) = 1 + 0.9 \cdot 10 = 10\)。验证 (2.13):\(v_\pi(s_3) = \pi(a_2\mid s_3) q_\pi(s_3, a_2) = 1 \cdot 10 = 10\) ✓(\(s_3\) 上策略是确定性向右)。这两条等式互相校验,完全自洽。

Q3. 矩阵-向量形式 \(q_\pi = \tilde r + \gamma P \Pi q_\pi\)(教材 (2.15))中,\(q_\pi\) 的维度是多少?\(P\)、\(\Pi\) 各是什么?为什么这套形式比 \(v\) 版"贵"?

A. \(q_\pi \in \mathbb{R}^{|\mathcal{S}| \cdot |\mathcal{A}|}\)(每个 (s, a) 对一个分量,本课 25 × 5 = 125 维)。\(P \in \mathbb{R}^{|\mathcal{S}\mathcal{A}| \times |\mathcal{S}|}\):\([P]_{(s,a), s'} = p(s' \mid s, a)\),把 (s, a) 映到 s'。\(\Pi \in \mathbb{R}^{|\mathcal{S}| \times |\mathcal{S}\mathcal{A}|}\):块对角矩阵,块 \(s'\) = 行向量 \([\pi(a' \mid s')]_{a'}\),把 s' 映到 (s', a') 的 π 加权和。"贵"是因为状态空间从 \(|\mathcal{S}|\) 膨胀到 \(|\mathcal{S}\mathcal{A}|\),矩阵规模是 \(v\) 版的 \(|\mathcal{A}|\) 倍 —— grid-world 5 个动作就要付 5 倍代价。这是后面"value-based vs policy-based"路线分歧的根源:value-based 直接学这个大 q,policy-based 绕开它。

下一章预告

到这一节为止,Ch.2 完成了它的三件事:(1) 用 state value 把"策略好坏"翻译成可比的标量;(2) 用 Bellman 方程把"算 \(v_\pi\)"变成线性方程组求解;(3) 用 action value 把"评估状态"细化为"评估动作"。但我们一直假设 \(\pi\) 是给定的 —— 没有回答 RL 的核心问题:哪个 \(\pi\) 最好?下章(Ch.3 Bellman Optimality Equation)直接登场:定义最优状态值 \(v^*(s) = \max_\pi v_\pi(s)\) 与最优策略 \(\pi^*\),把 (2.7) 中的 \(\sum_a \pi(a\mid s)[\dots]\) 换成 \(\max_a [\dots]\),得到非线性的 BOE \(v^*(s) = \max_a [\sum_r p(r\mid s,a)r + \gamma \sum_{s'} p(s'\mid s,a) v^*(s')]\),并用 Banach 不动点定理(γ-Lipschitz contraction)证明 BOE 有唯一解、且迭代法 \(v_{k+1} = T v_k\) 一定收敛到 \(v^*\)。这就是把"评估"升级为"优化"的转折点。
