§1 Why this lesson

本节的任务有三:(1) 给出 AC 家族的完整谱系图,清晰展示每个算法修改了哪一块;(2) 指出本书覆盖范围之外的重要下一步——PPO、SAC、多智能体、模型基础 RL;(3) 向坚持读到这里的读者说一声真心话。如果你跟完了全书 54 节,你已经具备了阅读大多数深度 RL 论文所需的数学基础——这是值得庆祝的事。

§2 核心定义

  • Actor-critic 的本质:任何同时维护策略网络(actor)和值函数网络(critic),并用 TD 误差训练 critic、用 critic 指导 actor 的算法,都是 actor-critic 方法。教材的广义定义:actor-critic = policy gradient + TD value estimation。
  • 优势函数 \(A(s,a)\) 的统一地位:\(A(s,a) = q_\pi(s,a) - v_\pi(s)\) 是整个 AC 家族的核心量。QAC 用 \(q\),A2C 用 \(A \approx \delta_t\),off-policy AC 用 \(\rho \cdot \delta_t\),DPG 用 \(\nabla_a q|_{a=\mu}\)——都是围绕"如何估计 A(s,a)"的不同方案。
  • policy-based + value-based 的婚姻:Ch.8 给了 critic 的理论(值函数近似 + DQN),Ch.9 给了 actor 的理论(策略梯度 + REINFORCE)。Ch.10 把两者合并。从这个视角看,AC 是全书逻辑的必然归宿,而非凭空出现的新算法族。
  • 深度 RL 的三大方向(书外):PPO/TRPO(on-policy AC + IS clip,OpenAI/DeepMind 的工程主流);SAC(Soft Actor-Critic)(off-policy AC + 最大熵框架,连续控制最强基线);IMPALA/APPO(大规模分布式 AC,数百 worker 并行)。这三类算法都直接建立在本书 Ch.10 的理论基础上。

§3 关键公式 + 推导

AC 家族谱系:从 REINFORCE 到 DDPG,每个算法在同一框架下只改了一个核心部件。

AC 家族每个算法都在修改同一个模板:\(\theta_{t+1} = \theta_t + \alpha_\theta \cdot [\text{IS修正}] \cdot [\text{策略梯度方向}] \cdot [\text{advantage估计}]\)。三个括号各自可以独立替换,产生不同算法组合。这正是 AC 框架模块化的魅力。

§5 常见误解 / 易混概念

很多人以为:学完 DDPG 就掌握了当前工业界用的深度 RL 算法。
实际上:DDPG 是 2015 年前后的算法,现代工业界主流已经进化到 PPO(2017)、SAC(2018)、TD3(2018,DDPG 的改进版)。这些算法在本书理论基础上各自添加了重要工程改进——PPO 的 IS clip 稳定训练、SAC 的最大熵框架鼓励探索、TD3 的双 critic 减少过估计。但数学框架都是本书已经建立的:policy gradient theorem + TD value estimation + function approximation。
因为:工业界的进步往往是"在正确方向上精细化",而本书给的是"正确方向"本身。读懂了本书再去读这些论文,数学部分都会豁然开朗。
很多人以为:Actor-critic 只是两个网络并行训练,结构很简单。
实际上:AC 的复杂性不在结构而在动态交互:actor 在改变的策略下采样,critic 追踪不断变化的目标 q_π,两者互相影响形成非平稳优化问题。这也是为什么深度 RL 比监督学习难调参——没有固定的标签,标签(q_π)随策略(actor)一起移动。理解这个动态本质,才能有效诊断训练失败的原因。
因为:RL 的样本来自当前策略,优化目标依赖当前策略,策略又在被优化——这种"自举循环"是 RL 区别于监督学习的根本特征,也是 RL 调参需要更多经验和耐心的原因。

§6 全书总结 + 下一步

Q1. 用一句话描述 REINFORCE、A2C、off-policy AC、DPG 各自的核心改动。

A. REINFORCE:用 MC 回报 \(G_t\) 估 \(q_\pi\),策略梯度的起点(无偏高方差)。A2C:用 TD 误差 \(\delta_t \approx q-v\) 替代 \(G_t\),一个 critic 网络估 \(v\),方差大幅降低。Off-policy AC:乘以 IS 权重 \(\rho = \pi/\beta\),用旧数据的样本训练当前策略。DPG/DDPG:策略改为确定性 \(\mu(s;\theta)\),梯度通过链式法则 \(\nabla_\theta\mu \cdot \nabla_a q\) 计算,适合连续动作空间。

Q2. 学完本书之后,想深入深度 RL 应该去看哪些方向?

A. On-policy 方向:PPO(Schulman 2017) — REINFORCE + IS clip + GAE,稳定高效;Off-policy 连续控制:SAC(Haarnoja 2018) — 最大熵 AC,探索性强;TD3(Fujimoto 2018) — DDPG + 双 critic,解决过估计。大规模并行:IMPALA(Espeholt 2018) — 分布式 off-policy AC,数百 worker。多智能体:MADDPG、MAPPO — AC 在多 agent 设定下的扩展。模型基础:Dreamer、MuZero — 用世界模型规划,sample efficiency 大幅提升。所有这些的数学根,都在本书 Ch.1-10 里。

Q3. 从 Ch.1 的状态/动作/策略,到 Ch.10 的 DDPG,RL 的核心逻辑链是什么?

A. 一条主线:定义问题(Ch.1:MDP)→ 给出最优解的性质(Ch.2-3:贝尔曼方程/BOE)→ 有模型时求解(Ch.4:VI/PI)→ 无模型时用数据估(Ch.5 MC + Ch.7 TD)→ 状态空间大时参数化(Ch.6 SGD + Ch.8 FA/DQN)→ 直接优化策略(Ch.9 PG)→ 同时优化策略和值函数(Ch.10 AC)。每一章都在解决前一章的局限:有模型→无模型→表格→参数化→值函数→策略→两者结合。这条逻辑链是本书的骨架。

全书结语

回望 Ch.1:那时候我们刚刚认识"状态"和"动作"这两个词。Ch.1-L1 的第一段话是——"你应该玩过类似走迷宫的游戏……直觉只能让你说小人往右走是因为右边是终点,却没办法让一台机器自动学会这件事"。现在回看这句话,你已经知道怎么让机器学会了:用贝尔曼方程定义价值,用 TD 误差在线学习,用策略梯度直接优化决策,用 critic 精确估计每个动作的优势。

Ch.4 的值迭代给了第一个"有模型"的解法。Ch.7 的 TD 让学习从离线变成在线。Ch.8 的 DQN 让 Atari 游戏成为可能。Ch.9 的策略梯度打开了策略空间的大门。Ch.10 的 actor-critic 把 value-based 与 policy-based 合为一体——这是现代深度 RL 的心脏。

下一步的三个方向:(1) 深度 RL 实践——PPO(最常用)、SAC(连续控制)、TD3(DDPG 改进版),代码实现从 stable-baselines3 或 CleanRL 开始;(2) 多智能体 RL——MADDPG、QMIX、MAPPO,多个 AC agent 同时学习,博弈论与 RL 的交汇;(3) 模型基础 RL——Dreamer(世界模型)、MuZero(无模型表示 + 规划),把环境模型与 AC 融合,样本效率大幅提升。这些都是本书逻辑链的自然延伸。

感谢一路读到这里。从第一个状态到最后一个 actor-critic,你已经走完了强化学习的数学基础。这不是终点,而是一个扎实的起跑线。
