L2: Basic Concepts (P2 - Reward, return, Markov decision process) (基本概念(P2):奖励、回报与马尔可夫决策过程)

§1 Why this lesson

上一节我们告诉了 agent "在哪里、能做什么、怎么决定做什么" —— 但还没有任何"对错"的概念。现在你给 agent 任意配一个策略,它都会很认真地按手册行动,可哪怕这个策略让它满地图乱撞、永远也走不到 target,我们也无从指责。因为我们还没给"好坏"下定义。

本节就是来补这个缺口。教材 §1.5–1.9 一口气抛出 4 个新词:reward(单步反馈)、trajectory(状态–动作–奖励链)、return(整条轨迹的累计打分)、discount factor γ(对未来的折现率),最后用 MDP 把前两节所有符号收进同一个五元组里。这一节最关键的一跳是 reward → return:为什么单步反馈不够、为什么必须看"整条路加起来",以及 γ 到底在调什么。教材中"先抛出 reward,再用 return 否定单看 reward"的写法,跟课程视频 P3 里"贪婪 vs 远视"的演示是同一个逻辑,务必跟着走完。

§2 核心定义(直觉先行)

  • 奖励 (reward):agent 在状态 \(s_t\) 下执行动作 \(a_t\) 之后,环境立刻吐出的一个标量实数 \(r_{t+1}\)(注意下标:做的是 \(a_t\),收到的奖励标作 \(r_{t+1}\),"动作打在 t、回报落在 t+1")。它可以为正、为负、为零;教材的 grid-world 约定 \(r_\text{target} = +1\)、\(r_\text{forbidden} = -1\)、\(r_\text{boundary} = -1\)、\(r_\text{other} = 0\)(p. 7)。关键认知:reward 是单步的、瞬时的,绝不是已经"加好"的总分。
  • 轨迹 (trajectory):从某个起始状态出发,agent 按策略实际走出来的一串"状态–动作–奖励"链,形式为 \((s_0, a_0, r_1, s_1, a_1, r_2, s_2, a_2, r_3, \dots)\)。教材 §1.6 用箭头表示(下面这串是教材原文 3×3 网格的索引,本系列正文一律用 5×5 / s_1..s_25):\(s_1 \xrightarrow[r=0]{a_2} s_2 \xrightarrow[r=0]{a_3} s_5 \xrightarrow[r=0]{a_3} s_8 \xrightarrow[r=1]{a_2} s_9\)。回到上一节的 grid-world 直觉,trajectory 就是橙色那条折线,只不过现在每条边上都贴了一个 reward 标签。
  • 回报 (return,又称 total reward 或 cumulative reward):把一条 trajectory 上的所有 reward 加起来,得到的总分。无折现版本:\(G = r_1 + r_2 + r_3 + \dots\)。教材 §1.6 用它来横向比较两条策略 —— "穿过 forbidden 那条 return 更小" —— 这是 RL 里"评估策略"的最原始动作。
  • 折扣因子 γ (discount factor / discount rate):γ ∈ [0, 1) 的一个实数,用来给未来的 reward 打折。本节首次出现,且贯穿后面所有公式。直觉:γ 越接近 0,agent 越只看眼前(myopic / 短视);γ 越接近 1,agent 越看长远(far-sighted)。数学上,它保证"无限长 trajectory 的折现回报"是个有限值(几何级数收敛)。
  • 折现回报 (discounted return):\(G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \dots = \sum_{k=0}^\infty \gamma^k r_{t+k+1}\)。这个量是后面"价值函数 \(v_\pi(s)\)"的内核 —— 价值就是对 \(G_t\) 取期望。
  • 马尔可夫决策过程 (Markov decision process, MDP):把 \(\mathcal{S}, \mathcal{A}, P, R, \gamma\) 这五个东西打包成一个数学对象 \(M = (\mathcal{S}, \mathcal{A}, P, R, \gamma)\)。再加一条马尔可夫性 (Markov property):下一状态/奖励只依赖当前的 \((s_t, a_t)\),不依赖更久远的历史。这条无记忆性是后面贝尔曼方程能写出来的根基。

§3 关键公式与拆解

本节有两条必须记牢的公式:折现回报的求和形式,以及 MDP 的五元组定义。

公式 1:折现回报 \(G_t\)(教材 §1.6 公式 (1.3),p. 9)

从时刻 \(t\) 起,把后续所有 reward 用 \(\gamma\) 的幂次加权求和:

Step 1. 为什么必须引入折现: 教材 §1.6 给了一个最直白的反例 —— 让 agent 走到 \(s_9\) 后停下不动并每步收 +1,无折现回报就是 \(0+0+0+1+1+1+\dots = \infty\),根本没法比较两条策略。引入 \(\gamma < 1\) 后,这条无穷 trajectory 的折现回报变成 \(\gamma^3 (1 + \gamma + \gamma^2 + \dots) = \gamma^3 / (1-\gamma)\),立刻收敛。
Step 2. γ 的"调旋钮"含义: γ 不只是技术性收敛工具,更是一个"远见参数"。γ → 0 时,\(G_t \approx r_{t+1}\),agent 只看下一步,变成贪婪策略;γ → 1 时,远期 reward 几乎不打折,agent 愿意为遥远的回报忍受短期惩罚。本节末尾我们会用 grid-world 把这件事画出来。
Step 3. 下标错位再次提醒: 写成 \(\sum_{k=0}^\infty \gamma^k r_{t+k+1}\) 是为了让"第一个 reward 的折现权重恰好是 \(\gamma^0 = 1\)"。后面写贝尔曼方程时这个错位会反复出现 —— 看到 \(r_{t+1}\) 就要想到"它是 \(a_t\) 的产物"。

公式 2:MDP 五元组(教材 §1.7,p. 11)

把上一节的状态/动作/转移、本节的 reward/discount 一并打包:

Step 1. 马尔可夫性是 MDP 的"灵魂": 教材 §1.7 公式 (1.4) 写道 \(p(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \dots, s_0, a_0) = p(s_{t+1} \mid s_t, a_t)\) 以及 reward 同理。意思是"未来只取决于当下,不取决于历史" —— 这是为什么我们可以用一个不带历史的 \(v_\pi(s)\) 来描述价值,而不需要写成 \(v_\pi(\text{whole history})\)。
Step 2. MDP vs MP: 教材 §1.8 提到"一旦策略 π 固定,MDP 就退化为马尔可夫过程 (MP)"。直觉是:MDP 的不确定性有两个源头 —— 策略 \(\pi(a\mid s)\) 和环境 \(p(s'\mid s,a)\);把策略固定后,只剩环境一个源头,系统就只是一条带状态转移的 Markov chain。
Step 3. 为什么把 γ 写进 MDP: 不同的 γ 对应不同的"什么是最优"的定义。同一个环境 \((\mathcal{S}, \mathcal{A}, P, R)\),把 γ 从 0.5 调到 0.99,最优策略可能完全不一样(下面 §4 给出实例)。所以 γ 是问题定义的一部分,不是算法选项。

§4 Grid-world 实例化

用上一节那张 5×5 网格,对比两条策略可能产出的 trajectory。设定:target = \((3, 2)\),forbidden = \(\{(1,1), (2,1), (2,2), (1,3), (3,3)\}\),起点 \((0, 0)\)。注意几何前提:从 \((0,0)\) 到 \((3,2)\) 的 Manhattan 距离 = \(3 + 2 = 5\),所以任何合法路径至少 5 步,要"更短"必须接受额外代价。下面这张图展示 Path A:走外圈,沿第 0 列下到 \((4,0)\),再沿第 4 行右到 \((4,2)\),最后向上一步到 target,共 7 步,完全避开 forbidden:

Path A 的奖励标注。沿这条 7 步 trajectory 一步步收 reward(本书 §1.5 标准约定:\(r_\text{target} = +1\),普通格 = 0,踩 forbidden = \(-1\)):

  • \((0,0) \xrightarrow{\downarrow} (1,0)\):\(r_1 = 0\)
  • \((1,0) \xrightarrow{\downarrow} (2,0)\):\(r_2 = 0\)
  • \((2,0) \xrightarrow{\downarrow} (3,0)\):\(r_3 = 0\)
  • \((3,0) \xrightarrow{\downarrow} (4,0)\):\(r_4 = 0\)
  • \((4,0) \xrightarrow{\rightarrow} (4,1)\):\(r_5 = 0\)
  • \((4,1) \xrightarrow{\rightarrow} (4,2)\):\(r_6 = 0\)
  • \((4,2) \xrightarrow{\uparrow} (3,2)\):\(r_7 = +1\)(到达 target)
  • 到达后按教材 §1.6 的"target 可继续行动"约定,留在 target 上每步 \(+1\):\(r_8 = r_9 = \dots = +1\)

Path A 的折现回报。前 6 步 reward 都是 0,从第 7 步起每步 \(+1\):

\(G_0^A = \gamma^6 \cdot 1 + \gamma^7 \cdot 1 + \gamma^8 \cdot 1 + \dots = \dfrac{\gamma^6}{1 - \gamma}\)。

Path B(假想:5 步到 target,中途踩一次 forbidden,比 A 快 2 步)。设这条 trajectory 是 \((0,0) \to (1,0) \to (2,0) \to (2,1)_\text{forb} \to (3,1) \to (3,2)\) —— 沿第 0 列下到 \((2,0)\),侧身一步进 \((2,1)\) 撞 forbidden,再下到 \((3,1)\),最后右一步到 target。教材 §1.5 明确"forbidden 可达,只是有 \(-1\) 的负 reward",所以 reward 序列:

  • \((0,0) \xrightarrow{\downarrow} (1,0)\):\(r_1 = 0\)
  • \((1,0) \xrightarrow{\downarrow} (2,0)\):\(r_2 = 0\)
  • \((2,0) \xrightarrow{\rightarrow} (2,1)_\text{forb}\):\(r_3 = -1\)(踩 forbidden)
  • \((2,1) \xrightarrow{\downarrow} (3,1)\):\(r_4 = 0\)
  • \((3,1) \xrightarrow{\rightarrow} (3,2)\):\(r_5 = +1\)(到达 target)
  • 之后 \(r_6 = r_7 = \dots = +1\)

\(G_0^B = \gamma^2 \cdot (-1) + \gamma^4 \cdot 1 + \gamma^5 \cdot 1 + \dots = -\gamma^2 + \dfrac{\gamma^4}{1 - \gamma}\)。

把数代进去 —— 真 γ flip 出现:

  • γ = 0.5(短视): \(G_0^A = 0.5^6 / 0.5 = 0.03125\);\(G_0^B = -0.25 + 0.5^4 / 0.5 = -0.25 + 0.125 = -0.125\)。A 赢(差距 0.156)—— B 在 \(t = 2\) 时拿到的 \(-1\) 被权重 \(\gamma^2 = 0.25\) 放大,把"早 2 步到 target"的优势完全吃掉。
  • γ = 0.9(远视): \(G_0^A = 0.9^6 / 0.1 \approx 5.314\);\(G_0^B = -0.81 + 0.9^4 / 0.1 \approx -0.81 + 6.561 = 5.751\)。B 反超 A(差距 0.44)—— γ 大让"早 2 步到 target"凸显:B 的 \(r_5\) 权重 \(\gamma^4 \approx 0.656\),A 的 \(r_7\) 权重只有 \(\gamma^6 \approx 0.531\),早到优势超过一次罚分。
  • γ = 0.99(极远视): \(G_0^A \approx 94.15\);\(G_0^B \approx 95.08\)。B 继续赢,差距约 0.93,且代数上随 γ → 1 持续扩大。

怎么读这组数?代数上 \(G_0^A - G_0^B = \gamma^2 \, (1 - \gamma^2 - \gamma^3)\)。所以 \(\gamma^2 + \gamma^3 < 1\)(数值上约 \(\gamma < 0.755\))时 A 赢,反之 B 赢 —— γ 一旋,最优策略真的会翻。本质是:γ 小 → 怕早期罚 → 走外圈安全路;γ 大 → 看重早到 target → 容忍中途一次罚。这正是教材 p. 9 的核心论点:"γ 接近 0 → 短视;γ 接近 1 → 远视,敢于冒短期负奖励" —— 单步 reward 永远讲不清这件事,必须升级到 return 才看得见。

§5 常见误解 / 易混概念

很多人以为: "选 reward 最大的动作就是好策略,贪婪算法就行了。"
实际上: 教材 §1.5 末尾就在防这一点 —— "能不能直接拿 reward 表挑最大值?答:不能。"reward 只是 immediate reward,真正决定策略好坏的是 total reward = return。
因为: 一个动作的"贪心 reward"高,可能把 agent 推进一条后续 reward 都很烂的死胡同;反过来,一个当下 reward = 0 甚至 -1 的动作,如果它通向高回报的远期路径,长期看反而更优。这正是为什么 §1.6 紧接着就引入 return。
很多人以为: "γ 是算法超参数,随便调一下就行。"
实际上: γ 是 MDP 五元组 \(M = \langle \mathcal{S}, \mathcal{A}, P, R, \gamma \rangle\) 本身的一部分。换 γ 等于换问题:同一个环境在 \(\gamma = 0.5\) 下的最优策略,在 \(\gamma = 0.99\) 下未必最优(§4 已经看到趋势)。
因为: γ 决定了"未来奖励折现多少",而最优策略是定义在"最大化期望折现回报"上的 —— 优化目标变了,argmax 自然变。所以选 γ 是建模决策,不是调参。
很多人以为: "Markov 性就是说环境没有随机性。"
实际上: Markov 性说的是无记忆,跟随机性正交。环境完全可以随机(\(p(s_{t+1}\mid s_t, a_t)\) 是真正的概率分布),但只要"下一状态/奖励只依赖当下 \((s_t, a_t)\)",它就是马尔可夫的。
因为: 这条性质让我们能用一个 \(s\) 的函数 \(v_\pi(s)\) 完整描述价值,而不必以整段历史为条件 —— 这是贝尔曼方程能成立的根本。
很多人以为: "做 \(a_t\) 就拿 \(r_t\),下标一致才整齐。"
实际上: 教材通篇约定是"做 \(a_t\) 之后,环境给出 \(r_{t+1}\)" —— 动作打在 t,reward 落在 t+1。
因为: reward 是动作落地后才能观测到的;这种错位让 \(G_t = \sum_{k=0}^\infty \gamma^k r_{t+k+1}\) 中"k = 0 对应 \(\gamma^0 r_{t+1}\)"自然顺手,后面贝尔曼公式都吃这个约定。

§6 自检 + 视频 + 下一节

Q1. agent 沿一条无限长 trajectory 每步都拿 \(r = +1\)。当 \(\gamma = 0.9\) 时,\(G_0\) 等于多少?当 \(\gamma = 0\) 呢?

A. 由公式 \(G_0 = \sum_{k=0}^\infty \gamma^k r_{k+1} = \sum_{k=0}^\infty \gamma^k \cdot 1 = \dfrac{1}{1-\gamma}\)。代入:\(\gamma = 0.9 \Rightarrow G_0 = 10\);\(\gamma = 0 \Rightarrow G_0 = 1\)(只看 \(r_1\),后续都被 \(0^k\) 抹平,\(0^0 = 1\) 是约定)。教学点: γ 越大、累积上限越高,这是 γ 在"放大未来"的直观体现。

Q2. 在 §4 的两条路径中,我们算出 \(\gamma = 0.5\) 时 \(G_0^A = 0.03125\)、\(G_0^B = -0.125\)。如果保持 reward 设计不变,但把 \(\gamma\) 降到 0,A 和 B 的回报分别等于多少?这给最优策略带来什么影响?

A. γ = 0 时,\(G_0 = r_1 + 0 + 0 + \dots = r_1\)。两条路径的 \(r_1\) 都是 \(0\)(从 \((0,0)\) 出发到下一格都是普通格),所以 \(G_0^A = G_0^B = 0\),两条路径无法区分。这正暴露了纯短视(γ = 0)的弊病:agent 看不到 target 在哪儿,任何"开局第一步 reward 相同"的策略都被判为等价。要真正分辨 A、B,必须 γ > 0。

Q3. 教材 §1.7 把 Markov 性写成 \(p(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \dots, s_0, a_0) = p(s_{t+1} \mid s_t, a_t)\)。如果某天我们引入"上一格的格类型也会影响下一格转移"的环境,这条性质还成立吗?该怎么修复?

A. 不成立 —— 因为下一状态显式依赖了 \(s_{t-1}\),违反了"只依赖当前 \((s_t, a_t)\)"。修复办法是扩状态:把 \((s_{t-1}, s_t)\) 重新打包成一个新状态 \(\tilde s_t\),那么 \(\tilde s_t\) 就重新满足 Markov 性。代价是状态空间从 \(|\mathcal{S}|\) 暴涨到 \(|\mathcal{S}|^2\)。这种"用扩状态换 Markov 性"的技巧,会在后面 POMDP / N-step 方法里反复出现。

下一节预告

这一节我们把"策略好坏"翻译成了一个数 —— 折现回报 \(G_t\)。但 \(G_t\) 是随机变量(策略和环境都可能有随机性),要比较两个策略,真正该看的是 \(G_t\) 的期望。下一章(Ch.2)就在这里登场:它定义状态价值函数 \(v_\pi(s) = \mathbb{E}_\pi [G_t \mid S_t = s]\),并推导出大名鼎鼎的 Bellman 方程 —— 一旦能对每个状态算出"期望折现回报",我们就能正面回答"哪个策略最优"这个问题。本章到此结束,带着 reward / return / γ / MDP / Markov 性这五个零件出门吧。
