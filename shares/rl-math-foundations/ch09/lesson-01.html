L1: Policy Gradient Methods (P1) — Basic Idea (策略梯度方法 P1:基本思想)

§1 Why this lesson

Ch.8 L8 §6 留下一句关键话:DQN 的 \(\max_{a'}\hat q(s', a'; w_T)\) 钉 住了discrete actions。连续控制 ⇒ 无 max 可言,DQN 失效。本节(教材 §9.1, pp. 191-193)给一个结构性的另一支:不再先学 \(\hat q\) 再 \(\arg\max\),而是直接 把策略 自己参数化 成 \(\pi(a|s; \theta)\),用梯度上升 优化某个标量目标 \(J(\theta)\)。这一步看似简单,实际上整个 deep RL 的另一半都从这里展开:Ch.10 actor-critic、PPO、SAC、DDPG 全是该思路的变体。

核心 takeaway:(a) 表 vs 函数:Ch.1-Ch.8 的 \(\pi(a|s)\) 是表(Table 9.1),Ch.9 换成 \(\pi(a|s; \theta)\)(NN softmax,Fig. 9.2),\(\theta \in \mathbb R^m\)、\(m \ll |\mathcal S||\mathcal A|\)。(b) 最优定义改写:表式"最优" = "逐状态价值最大"(Ch.3 BOE);函数式"最优" = "标量指标 \(J(\theta)\) 最大"。(c) 更新规则一统:一个公式 \(\theta_{t+1} = \theta_t + \alpha \nabla_\theta J(\theta_t)\),不再"策略评估 + 策略改进" 双层循环。(d) 三大优势:大/连续空间可学;参数共享 ⇒ 泛化 + 样本效率;策略天然随机(\(\pi > 0\)),自带探索。(e) 桥到 Ch.6:SGD 模板第三次应用 —— Ch.7 估 \(V/Q\),Ch.8 优化 \(\hat v\),Ch.9 优化 \(\pi_\theta\)。骨架不变,目标对象换。

§2 核心定义(直觉先行)

  • 参数化策略 (parameterized policy) \(\pi(a|s; \theta)\):输入 \(s\)(可附 \(a\)),输出动作概率,参数为 \(\theta \in \mathbb R^m\)。常见实现是 softmax(教材 (9.12)):\(\pi(a|s; \theta) = e^{h(s, a; \theta)} / \sum_{a'} e^{h(s, a'; \theta)}\),其中 \(h(s, a; \theta)\) 是偏好函数(NN 的 logits)。这保证 \(\pi > 0\)(策略恒 探索)且 \(\sum_a \pi = 1\)。
  • 策略目标函数 (policy objective) \(J(\theta)\):把"策略好坏" 浓缩成一个 标量。L2 给平均状态值 \(\bar v_\pi = \mathbb E_{s\sim d}[v_\pi(s)]\),L3 给平均奖励 \(\bar r_\pi = \mathbb E_{s\sim d_\pi}[r_\pi(s)]\)。两者在 \(\gamma < 1\) 下等价(\(\bar r_\pi = (1-\gamma)\bar v_\pi\),Lemma 9.1)。
  • 梯度上升 (gradient ascent) 模板:\(\theta_{t+1} = \theta_t + \alpha \nabla_\theta J(\theta_t)\)。\(\alpha > 0\) 是学习率,\(\nabla_\theta J\) 是本章最核心的对象(L4 给出 PG 定理)。注意是+ 不是-:\(J\) 越大越好。
  • policy-based vs value-based 的结构差:value-based(Ch.4-Ch.8)走"v / q → π" 路径(先学价值,greedy 导出策略);policy-based 走"θ → π" 直达(把策略当主优化对象)。两者输出都是 \(\pi\),但谁是 first-class object 不同。Ch.10 actor-critic 把两者合并:critic 学价值帮 actor 估梯度。
  • "表 不再可写":Ch.1-Ch.8 的 \(\pi\) 总能显式 列出每 \((s, a)\) 概率;Ch.9 的 \(\pi(a|s; \theta)\) 是函数,改 \(\theta\) 一次,所有 \((s, a)\) 概率同时 改。这是泛化能力的根(类比 Ch.8 FA),也是连续动作能跑的根(无需枚举)。
  • 策略一定 随机:softmax 给的 \(\pi(a|s) \in (0, 1)\),无纯 0 / 纯 1。这与 Ch.5 ε-greedy 类似但更彻底:ε-greedy 是"greedy + 一点随机",softmax 是"整个 概率分布由 \(\theta\) 决定"。最优策略可能是近 确定性(\(h\) 一项远大于其他)而非纯 确定。
  • 桥到 Ch.6 SGD 模板:\(\theta_{t+1} = \theta_t + \alpha \nabla_\theta J(\theta_t)\) 是 RM/SGD 应用在 \(J\) 上;真梯度未知 ⇒ 用样本估(Ch.6 L4 SGD 范式)。L4 给真梯度形式,L5 REINFORCE 把它变成样本可估。

这两条公式合在一起就是整个 Ch.9 的骨架:剩下的 4 节全部回答"\(J(\theta)\) 是什么"(L2-L3)、"\(\nabla_\theta J\) 怎么算"(L4)、"用样本怎么估"(L5)。

Step 1. "表 → 函数" 不只是形式 改。表式 \(\pi(a|s)\) 有 \(|\mathcal S|\cdot|\mathcal A|\) 个独立数,改一个不影响其他;函数式 \(\pi(a|s; \theta)\) 由 \(\theta\) 决定所有 \((s, a)\),改一个 \(\theta_i\) 同时 改所有 \(\pi\) 值。这是共享参数 的双刃剑:好处 = 泛化(在没访问的 \(s\) 上也有合理 \(\pi\)),坏处 = 互相干扰(改某 \((s, a)\) 可能反向 影响其他)。Ch.8 FA 在价值 上揭示了这点;Ch.9 在策略 上重演。
Step 2. "最优" 定义的松绑。表式下 BOE 要求每个 状态值最大(Ch.3 严苛定义);函数式下只能要求标量指标 \(J\) 最大,无法对每个 \(s\) 都最优(参数共享 ⇒ 不同 \(s\) 的最优策略可能互斥)。这是结构上的妥协:换来的是大空间可学。教材原文:"policy is defined as optimal if it can maximize certain scalar metrics",把"逐状态最优" 弱化为"整体指标最优"。
Step 3. "探索 = 随机性" 自动满足。Ch.5 MC ε-greedy 用外加 ε 保证每动作 ≥ ε/|A|,Ch.7 Q-learning 类似。Ch.9 softmax 天然 \(\pi > 0\) ⇒ 不需 ε,概率小但非零 的动作仍偶尔被选。这是 Ch.9 比 value-based 原生 优雅的一点 —— exploration 是策略表示 的副产品。但代价:不能输出纯 确定性策略(对某些问题如 chess 不利,Ch.10 deterministic AC 解决)。

§5 常见误解 / 易混概念

很多人以为:"既然 Ch.8 DQN 能用 NN 学 Q,直接 NN 学策略不就一样?把 Q 网络的输出层改成 softmax 概率不就完事了?"
实际上:"NN 学 Q" 与"NN 学 \(\pi\)" 训练目标结构 不同。DQN 的目标是"减小 TD 误差"(Ch.8 L2 MSPBE),loss 形式是回归 \((y_T - \hat q)^2\),有明确 的 supervision-style label \(y_T\);PG 的目标是"增大 \(J(\theta)\)",loss 形式是"负 \(\nabla \ln \pi \cdot q_\pi\)"(L4),没有"label \(\pi^*\)"(策略本身就是要学的东西)。两者都用 SGD,但"梯度方向从哪来" 完全不同:DQN 从预测误差 来,PG 从采样轨迹的回报 来(L5 REINFORCE)。
因为:价值学习是supervised-style(有 \(y_T\) 当 label);策略学习是direct-policy(目标 \(J\) 是策略 的函数,无 label)。这一结构差 导致 PG 比 DQN 方差大、需 baseline 减方差(Ch.10),但能 处理 DQN 不能的连续动作。
很多人以为:"\(\pi\) 既然必须随机,最优 策略肯定不是确定性 —— 与 Ch.3 BOE 矛盾。"
实际上:softmax 下 \(\pi\) 可近 确定性。当 \(h(s, a^*)\) 远大于其他时 \(\pi(a^*|s) \to 1\)。Ch.3 BOE 说"最优策略存在 确定性形式";softmax 能逼近 但不等于。实践上 \(\pi\) 仍有极小 exploration 残留(\(\theta\) 有限),通常是优点(鲁棒)。
因为:极限可达,但有限 \(\theta\) 永远 stochastic。Ch.10 DPG 用 \(\mu(s; \theta)\) 直接输出确定动作,绕开 softmax 限制。
很多人以为:"PG 看上去比 DQN 简洁(无 target net、无 replay),应该更稳定。"
实际上:PG 反而方差大、收敛慢。原因:(i) 梯度是策略期望 的样本估,episode 间方差大(L5 REINFORCE);(ii) 无 bootstrap(MC 风格 ⇒ 方差高);(iii) on-policy 严格 ⇒ 数据用一次就废。Ch.10 AC 专门缓解 —— critic 替代 \(G_t\)、A=q-v 进一步减、IS 让数据可重用。
因为:DQN 的 target net + replay 是 deadly triad 的工程缓解;PG 不 bootstrap 价值故无 deadly triad,但方差 是 PG 的主要敌人。Ch.10 把 Ch.8 critic 与 Ch.9 actor 合并,同时解决两者代价。

§6 自检 + 视频 + 下一节

Q1. 写出 softmax 策略形式,并说明为什么 Ch.9 选用 softmax 而非 sigmoid。

A. \(\pi(a|s; \theta) = e^{h(s, a; \theta)} / \sum_{a'} e^{h(s, a'; \theta)}\)(教材 (9.12))。为什么 softmax:(i) 多动作场景下,sigmoid 给每动作一个 \([0,1]\) 但和不为 1(独立伯努利),需事后归一;softmax 原生 满足 \(\sum_a \pi = 1\)。(ii) softmax 把偏好 \(h\) 与概率 \(\pi\) 分离 —— \(h\) 可任意实数(NN logits 自由),\(\pi\) 自动 (0,1)。(iii) \(\pi > 0\) ⇒ \(\ln \pi\) 有限 ⇒ L4 log-derivative trick \(\nabla \pi / \pi = \nabla \ln \pi\) 在所有 \((s, a)\) 处都成立,这是 PG 算法能存在 的形式根。换 sigmoid 或硬 0/1 ⇒ \(\ln 0 = -\infty\),PG 在某些动作上失效。

Q2. 表式策略下的"最优" 定义(BOE)与函数式策略下的"最优" 定义有何不同?

A. 表式下"\(\pi^*\) 最优" = "对每个 \(s\),\(v_{\pi^*}(s) \ge v_\pi(s)\) 对任意 \(\pi\)"(逐状态最优,Ch.3 BOE)。函数式下"\(\theta^*\) 最优" = "\(J(\theta^*) \ge J(\theta)\) 对任意 \(\theta\)"(标量指标最优)。两者差别:(a) 表式逐状态严苛,函数式只能保证整体 指标(L2 平均状态值或 L3 平均奖励)。(b) 函数式因共享 \(\theta\) ⇒ 不同状态的最优策略可能互斥(参数干扰),无法同时逐状态最优;表式则可独立调每格。(c) 实际工程接受这种"松绑",换大空间可学(连续动作、像素状态)。Ch.9 整章都建立在函数式 + 标量指标 这个新框架上。

Q3. 为什么 PG 比 DQN 原生 适合连续动作?DQN 的具体瓶颈在哪?

A. DQN 核心 update 含 \(y_T = r + \gamma \max_{a'}\hat q(s', a'; w_T)\),\(\max_{a'}\) 要求枚举 所有 \(a'\)。连续动作 \(a \in \mathbb R^d\) 无 枚举(无穷多);若用数值优化 求 max 每步训练成本爆炸(每条 transition 一次内层优化)。PG 不计算 max,而直接 把策略写成 \(\pi(a|s; \theta)\) —— 对连续动作可输出Gaussian 分布参数 \((\mu(s; \theta), \sigma(s; \theta))\),采样 \(a \sim \mathcal N(\mu, \sigma^2)\)。\(\nabla_\theta \ln \pi\) 对 Gaussian 是解析 闭形式,PG 直接套。结果:DDPG / SAC / PPO 全在连续控制(机器人、Atari 离散变种、自动驾驶)上是 SOTA,DQN 只能在 Atari 离散动作。这是 Ch.9 与 Ch.8 的能力分界线。

下一节预告

本节立了"策略 = 函数 \(\pi_\theta\),目标 = 标量 \(J(\theta)\),更新 = 梯度上升" 的骨架,但没 定义 \(J(\theta)\) 的具体形式。L2(教材 §9.2) 给第一种 指标:平均状态值 (average value) \(\bar v_\pi = \mathbb E_{s\sim d}[v_\pi(s)] = \sum_s d(s) v_\pi(s)\)。关键设计选择是"加权分布 \(d(s)\) 怎么选":可独立 于 \(\pi\)(\(d_0\),如均匀或起点 \(s_0\))得 \(\bar v_\pi^0\),也可依赖 \(\pi\)(stationary distribution \(d_\pi\))。L2 还把 \(\bar v_\pi\) 与"discounted return" \(\mathbb E[\sum_{t=0}^\infty \gamma^t R_{t+1}]\) 联系起来 —— 这两者其实是同一件事的不同写法(教材 (9.1))。
