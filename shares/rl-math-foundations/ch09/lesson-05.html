L5: PG (P5) — Gradient-based Algorithms & REINFORCE (策略梯度 P5:梯度算法与 REINFORCE)

§1 Why this lesson

L4 给了 Ch.9 的核心定理:\(\nabla_\theta J(\theta) = \mathbb E_{S\sim\eta, A\sim\pi}[\nabla_\theta\ln\pi(A|S;\theta) \cdot q_\pi(S, A)]\)(教材 (9.9))。这是真 梯度,但实际工程中无 法直接计算(\(\eta, \pi, q_\pi\) 都需遍历 状态-动作-轨迹)。本节(教材 §9.4-9.6,pp. 210-214)给第一个具体的 PG 算法:把真 梯度的期望 用样本 估,把 \(q_\pi(s_t, a_t)\) 用Monte Carlo 回报 \(G_t\)(Ch.5 复用)估 ⇒ REINFORCE(Williams 1992,Algorithm 9.1)。这是 PG 算法的开山之作,也是 Ch.10 actor-critic 的蓝本。三件事:(a) 真梯度 → 随机梯度 (9.32);(b) exploration-exploitation 的 \(\beta_t = q_t/\pi\) 解读;(c) Algorithm 9.1 完整伪码 + 方差问题 + Ch.10 桥。

核心 takeaway:(a) 真梯度 → 随机梯度:\(\theta_{t+1} = \theta_t + \alpha \nabla_\theta\ln\pi(a_t|s_t; \theta_t) \cdot q_t(s_t, a_t)\)(教材 (9.32)),其中 \(q_t\) 是 \(q_\pi(s_t, a_t)\) 的近似。(b) MC 估 \(q_t\):用 \(q_t(s_t, a_t) = \sum_{k=t+1}^T \gamma^{k-t-1} r_k = G_t\)(完整 episode 后向求和),把 PG 落地为REINFORCE。(c) \(\beta_t\) 解读(教材 (9.33)):\(\theta_{t+1} = \theta_t + \alpha \beta_t \nabla\pi(a_t|s_t)\),\(\beta_t = q_t / \pi(a_t|s_t)\)。\(\beta_t \ge 0\) ⇒ 增 \(\pi(a_t|s_t)\)(exploit q 大);\(\pi\) 小 ⇒ \(\beta_t\) 大 ⇒ 增长低概率动作(explore)。自带 exploration-exploitation 机制。(d) on-policy 性:S 应 ~ \(\eta = d_\pi\) / \(\rho_\pi\)、A 应 ~ \(\pi(\theta)\) ⇒ 严格 on-policy。Algorithm 9.1 实操放松:1 episode 用完(每条 transition 训 1 步)。(e) 高方差 → Ch.10 critic 减方差:\(G_t\) 是 MC 回报,episode 间方差极大(Ch.7 L1 MC vs TD)。Ch.10 用 critic 网络 \(\hat q_w\) 替代 \(G_t\) ⇒ actor-critic。这是 Ch.10 的动机起点。

§2 核心定义(直觉先行)

  • 真梯度上升(教材 (9.31)):\(\theta_{t+1} = \theta_t + \alpha \nabla_\theta J(\theta_t) = \theta_t + \alpha \mathbb E[\nabla_\theta\ln\pi(A|S; \theta_t) q_\pi(S, A)]\)。真 期望需遍历,工程上不可行 ⇒ 随机化为 (9.32)。
  • 随机梯度(教材 (9.32)):\(\theta_{t+1} = \theta_t + \alpha \nabla_\theta\ln\pi(a_t|s_t; \theta_t) \cdot q_t(s_t, a_t)\)。把期望 换成单样本 \((s_t, a_t)\)、把 \(q_\pi(s_t, a_t)\) 换成近似 \(q_t(s_t, a_t)\)。Ch.6 SGD 模板第三次应用。
  • REINFORCE(教材 §9.4):把 \(q_t(s_t, a_t)\) 用 Monte Carlo 回报估,即 \(q_t(s_t, a_t) = G_t = \sum_{k=t+1}^T \gamma^{k-t-1} r_k\)(同 Ch.5 first-visit MC)。这是"Monte Carlo policy gradient"(REINFORCE = REward Increment = Nonnegative Factor × Offset Reinforcement × Characteristic Eligibility,Williams 1992)。
  • \(\beta_t\) 系数(教材 (9.33)):用 log-trick 反推 \(\nabla\ln\pi = \nabla\pi/\pi\) ⇒ (9.32) 重写为 \(\theta_{t+1} = \theta_t + \alpha (q_t/\pi(a_t|s_t)) \nabla\pi(a_t|s_t) = \theta_t + \alpha \beta_t \nabla\pi(a_t|s_t)\),\(\beta_t = q_t/\pi(a_t|s_t)\)。
  • \(\beta_t\) 的双 解读:(i) exploit:\(\beta_t \propto q_t\),\(q_t\) 大 ⇒ \(\beta_t\) 大 ⇒ \(\pi(a_t|s_t)\) 增加多 ⇒ 偏好高价值动作。(ii) explore:\(\beta_t \propto 1/\pi(a_t|s_t)\),\(\pi(a_t|s_t)\) 小 ⇒ \(\beta_t\) 大 ⇒ 概率小动作被增加多 ⇒ 平衡探索。两者天然合一。
  • Algorithm 9.1 伪码:(1) Init \(\theta_0\);(2) for episode in episodes:(2a) 按 \(\pi(\theta)\) 采 \(\{s_0, a_0, r_1, ..., s_{T-1}, a_{T-1}, r_T\}\);(2b) for \(t = 0, ..., T-1\):算 \(q_t(s_t, a_t) = \sum_{k=t+1}^T \gamma^{k-t-1} r_k\);更新 \(\theta \leftarrow \theta + \alpha \nabla_\theta\ln\pi(a_t|s_t; \theta) q_t(s_t, a_t)\)。注意:1 episode 内每 \(t\) 用同一 \(\theta\) 算 \(q_t\)(off-policy 漂移在实践中容忍)。
  • 方差大 → Ch.10 baseline:\(G_t\) 是 episode 累积 reward 的无偏 估,但方差极大(整条 episode 的随机性都体现在 \(G_t\) 中)。常见缓解:baseline 减 \(b(s_t)\) ⇒ \(\nabla\ln\pi (G_t - b)\),不变期望(\(\mathbb E[\nabla\ln\pi \cdot b] = 0\))但减方差。Ch.10 把 \(b = \hat v(s_t)\) 学出来 ⇒ advantage A=G-V ⇒ A2C。

Algorithm 9.1 — REINFORCE 步骤(教材 §9.4):

REINFORCE 是history:Williams 1992 提出,DeepRL 时代之前 RL 主流;Sutton-Barto 经典案例;Ch.10 actor-critic 直接由它替换 \(G_t\) 派生。

Step 1. "真梯度 → 随机梯度" 是 Ch.6 SGD 模板第三次出现。Ch.6 L4 给"用样本估梯度,RM 步长收敛";Ch.7 拿它估 \(V/Q\)(TD);Ch.8 拿它优化 \(\hat v\)(MSPBE 半梯度);Ch.9 拿它估 \(\nabla J\)(REINFORCE)。骨架:\(\theta_{t+1} = \theta_t + \alpha \cdot (\text{single sample of } \nabla J)\)。每次具体化 不同:Ch.7 估的是价值,Ch.8 估的是FA 价值,Ch.9 估的是策略目标。同一"用一个样本估期望" 的工具用了三次,RL 数学的统一性。
Step 2. "\(G_t\) 替 \(q_\pi\)" 的无偏 + 高方差。无偏:Ch.5 L1 已证 \(\mathbb E[G_t | s_t, a_t] = q_\pi(s_t, a_t)\)(MC return 是 \(q_\pi\) 的无偏样本)。所以 \(\nabla\ln\pi(a_t|s_t) G_t\) 是 \(\nabla J\) 的无偏 单样本估,SGD 收敛(Ch.6 L3)。高方差:\(G_t\) 是整条 后续 episode reward 的累积,episode 间随机性叠加 ⇒ 方差 \(\sim T\)(episode 长度)。这与 Ch.7 L1 MC vs TD 的同 困境:MC 高方差,TD 低方差但有偏。Ch.10 actor-critic 用 critic \(\hat q_w\) 替 \(G_t\) ⇒ TD-style 估(低方差),代价是引入有偏(\(\hat q\) 还在学)。
Step 3. "策略学习的 self-improvement" 几何。REINFORCE 每 episode 后,\(\theta\) 朝"放大好动作 + 缩小坏动作" 走一步。多步后整体 \(\pi\) 漂向"常选 high-q 动作",即greedy 极限。但因 softmax 永远 \(> 0\),\(\pi\) 永远 保留极小 探索 ⇒ 既近 deterministic 又稳健(对偶环境扰动有抗)。这是 PG 与 BOE deterministic \(\pi^*\) 的微妙差异:PG 的 \(\pi_\theta\) 在极限 上逼近 deterministic,但有限训练 后保留 stochasticity。

§5 常见误解 / 易混概念

很多人以为:"REINFORCE 既然是 PG 定理直接 SGD 落地,那它和真 梯度同样有效,只是步数多一点。"
实际上:REINFORCE 方差极大,实际收敛远慢于"真梯度"。原因:(i) \(G_t\) 是整条 后续 reward 求和,episode 越长方差越大(\(\sim T\) 倍);(ii) on-policy 严格 ⇒ 老数据不能复用,采样效率低;(iii) 不同 episode 间 \(\nabla\ln\pi\) 方向无关(随机轨迹),平均 \(O(\sqrt N)\) 才收敛。对比 DQN:replay buffer 让样本利用 100×,平均梯度更稳。实测:Atari 类任务,REINFORCE 收敛需 \(10^7\) episode,DQN 需 \(10^4\) episode(约 1000× 差距)。
因为:RL 的"真" 梯度需遍历 期望;样本估必然 有方差。MC 风格(REINFORCE) ⇒ 高方差;TD 风格(actor-critic) ⇒ 低方差但有偏。Ch.10 给低方差版本是必须,REINFORCE 不是工程好 算法,只是第一 个 PG 算法。
很多人以为:"既然 \(G_t\) 方差大,直接减一个 baseline(常数 \(b\) 或 \(\hat v\))就降方差,这与无偏性 冲突吗?"
实际上:baseline 减不 改期望,只减方差。证:\(\mathbb E_{A\sim\pi(s)}[\nabla\ln\pi(A|s) \cdot b(s)] = b(s) \mathbb E_A[\nabla\ln\pi(A|s)] = b(s) \cdot \nabla \sum_a \pi(a|s) = b(s) \cdot \nabla 1 = 0\)。所以从 \(\nabla\ln\pi \cdot G_t\) 中减去 \(\nabla\ln\pi \cdot b\) 不改期望但减 \(G\) 的变动幅度。选 \(b = \hat v(s)\)(状态平均回报)⇒ \(G_t - \hat v\) 波动 比 \(G_t\) 小很多 ⇒ 方差减很多。这是 Ch.10 advantage actor-critic (A2C) 的核心:learn \(\hat v(s)\) 当 baseline,得 advantage \(A_t = G_t - \hat v(s_t)\)。
因为:"baseline 不改期望" 是 \(\sum_a \pi(a|s) \nabla\ln\pi(a|s) = \nabla \sum_a \pi = 0\) 的几何 consequence。这一性质让任意 \(b(s)\) 函数(state-only,不依赖 a)都不改期望,只要选得好(接近 \(v_\pi\))就显著减方差。Ch.10 把这一观察落地为 actor-critic 算法。
很多人以为:"REINFORCE 是 on-policy 的 ⇒ 不能用 replay buffer 复用老数据 ⇒ 严重浪费样本。"
实际上:这一限制 真实存在,但 可用importance sampling 缓解。On-policy 严格要求 \(a_t \sim \pi(\cdot|s_t; \theta_t)\)(当前 \(\theta\) 的策略采样)。老 \(\theta_{\mathrm{old}}\) 采的样本 ⇒ 期望失配,直接用偏差大。缓解:用 IS 比 \(\rho_t = \pi(a_t|s_t; \theta) / \pi(a_t|s_t; \theta_{\mathrm{old}})\) 修正,得 off-policy PG \(\nabla J = \mathbb E_{\theta_{\mathrm{old}}}[\rho_t \nabla\ln\pi q_\pi]\)。代价 = \(\rho_t\) 大时方差爆,实际工程加 clip(\(\rho_t \in [1-\epsilon, 1+\epsilon]\),即 PPO 的核心机制)。Ch.10 §10.3 讲 off-policy AC + IS。
因为:"on-policy = 数据浪费" 是结构 限制,但 RL 工程有 trick。PPO(Schulman 2017)是 REINFORCE + IS clip + GAE 的工程 集大成,deep RL 主流之一。Ch.10 给 IS 的纯 形式,后续算法各加各的工程。

§6 自检 + 视频 + 下一节

Q1. 写出 REINFORCE 的更新公式 与伪码 5 步。

A. 公式:\(\theta_{t+1} = \theta_t + \alpha \nabla_\theta\ln\pi(a_t|s_t; \theta_t) \cdot G_t\),其中 \(G_t = \sum_{k=t+1}^T \gamma^{k-t-1} r_k\) 是 MC 回报。5 步伪码:(1) Init \(\theta_0\),设 \(\gamma, \alpha\)。(2) for episode = 1, 2, ...:(2a) 按 \(\pi(\theta)\) 采完整轨迹 \((s_0, a_0, r_1, ..., s_{T-1}, a_{T-1}, r_T)\);(2b) for \(t = 0, ..., T-1\):算 \(G_t\)(后向递推 \(G_T = 0; G_{t-1} = \gamma G_t + r_t\));(2c) for \(t = 0, ..., T-1\):\(\theta \leftarrow \theta + \alpha \nabla_\theta\ln\pi(a_t|s_t; \theta) G_t\);(3) 重复直到收敛。关键:on-policy(每 episode 必须用当前 \(\theta\) 采),逐 \(t\) 后向求 \(G_t\)。

Q2. 解读 \(\beta_t = q_t(s_t, a_t) / \pi(a_t|s_t; \theta_t)\) 的exploration-exploitation 机制。

A. 由 (9.33) \(\theta_{t+1} = \theta_t + \alpha \beta_t \nabla\pi(a_t|s_t)\)。exploit:\(\beta_t \propto q_t\),\(q_t\) 大(动作好) ⇒ \(\beta_t\) 大 ⇒ \(\pi(a_t|s_t)\) 增加多 ⇒ 偏好高价值动作。explore:\(\beta_t \propto 1/\pi(a_t|s_t)\),低概率动作 \(a_t\) 一旦被偶然 选中,\(1/\pi\) 大 ⇒ \(\beta_t\) 大 ⇒ \(\pi(a_t|s_t)\) 增长被放大 ⇒ 探索新动作。两者同公式 自动出现,无需外加 ε 超参。这是 PG 比 ε-greedy 原生优雅 的根 —— exploration 是梯度公式 的副产品。

Q3. REINFORCE 的主要 缺点是什么?Ch.10 actor-critic 怎么解决?

A. 主要缺点:(i) 高方差 —— \(G_t\) 是整条后续 reward 求和,episode 间方差大(\(\sim T\));(ii) on-policy 浪费 —— 老数据不能复用;(iii) episode-update —— 必须等整 episode 跑完才能算 \(G_t\),无法在线 更新。Ch.10 解决:(a) critic 网络 \(\hat q_w\) 替 \(G_t\)(教材 §10.1):TD bootstrap 估,1-step 而非整 episode ⇒ 低方差 + 在线;(b) advantage \(A = q - v\) 进一步减方差(§10.2 A2C);(c) IS 让数据可重用(§10.3 off-policy AC,PPO 类前身);(d) deterministic policy(§10.4 DPG) 给连续动作严格 \(\arg\max\)。Ch.10 把 Ch.8 critic 与 Ch.9 actor 合一,集大成。

下一章预告:Ch.10 Actor-Critic

Ch.9 用"策略 = 函数 \(\pi_\theta\) + PG 定理 + REINFORCE" 走完policy-based 主线,但 REINFORCE 的方差大 是主要敌人。Ch.10 给整合 方案:用 Ch.8 学 \(\hat q_w\)(critic),用 Ch.9 学 \(\pi_\theta\)(actor),同时 训练。REINFORCE 用 \(G_t\) 估 \(q_\pi\),方差很大;Ch.10 用 critic 网络估 \(q_\pi\) —— 这就是 actor-critic。具体:\(\theta \leftarrow \theta + \alpha \nabla\ln\pi(a_t|s_t; \theta) \hat q_w(s_t, a_t)\)(critic 替 \(G_t\),低方差);\(w \leftarrow w + \beta \delta_t \nabla\hat q_w\)(critic 用 TD 学,Ch.8 复用)。L1-L5 渐进:简单 AC → A2C(advantage)→ off-policy AC + IS → DPG(deterministic for continuous)→ 总结。Ch.10 是 deep RL 现代算法(PPO、SAC、DDPG)的直接前身。
